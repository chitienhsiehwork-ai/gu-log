各位好，今天我們來聊一個很實用的主題——怎麼在 Claude Code 裡面「優雅地」調用 OpenAI 的 Codex。

先說結論：**直接用 Codex CLI？可以，但體驗普普。把 Codex 接成 MCP server 丟進 Claude Code？那才叫優雅。**

---

## 為什麼不直接用 Codex CLI？

用過 Codex CLI 的朋友應該都有感覺：

- **命令行交互很原始** — 就是那種「能用但不太想用」的程度
- **Sandbox 環境 bug 不少** — 跑到一半炸掉也不是什麼稀奇的事
- **命令執行限制多** — 很多操作被擋，綁手綁腳

相比之下，Claude Code 的設計就完善多了。它原生支援 MCP、Plugin、Skill、Agent 等各種自定義擴展，工具集成好，UX 也友善。

所以思路很簡單：**與其忍受 Codex CLI 的各種限制，不如把 Codex 當成一個 MCP server，接進 Claude Code 這個更強大的框架裡。**

<ClawdNote>
簡單說就是：Codex 有腦但手腳不太靈活，Claude Code 有完整的身體和神經系統。把 Codex 的腦裝進 Claude Code 的身體裡，就是最佳組合。

~~等等這個比喻怎麼有點獵奇~~
</ClawdNote>

## 一條命令搞定配置

廢話不多說，直接上指令。這條命令會把 Codex 當作 MCP server 加到你的 Claude Code 全局配置：

```bash
claude mcp add codex -s user -- codex -m gpt-5.3-codex -c model_reasoning_effort="high" mcp-server
```

讓我們拆解一下各個參數：

| 參數 | 說明 |
|------|------|
| `-s user` | 寫入全局配置（不只是單一專案） |
| `codex -m gpt-5.3-codex` | 指定使用的 Codex 模型 |
| `-c model_reasoning_effort="high"` | 把推理強度拉到最高 |
| `mcp-server` | 啟動 MCP 服務器模式 |

<ClawdNote>
`model_reasoning_effort="high"` 就是跟 Codex 說「你給我認真想」。

不過認真想 = token 燒更快 = 帳單更痛。自己斟酌囉 💸
</ClawdNote>

## 驗證與使用

設定完之後，先確認一下有沒有成功：

```bash
claude mcp list
claude mcp get codex
```

使用方式也很直覺：在 Claude Code 的對話中輸入 `/mcp`，然後選擇 `codex` 就好了。

就這樣。沒有複雜的設定檔要改，沒有奇怪的環境變數要設。一條命令，收工。

---

## 常見問題 FAQ

**Q: 輸入指令後找不到 `codex` 命令？**
→ 檢查你的 `PATH` 有沒有包含 Codex CLI 的安裝路徑。

**Q: 想移除怎麼辦？**
→ 一樣一條命令搞定：

```bash
claude mcp remove codex
```

<ClawdNote>
MCP 的好處就是「即插即用」。今天覺得 Codex 好用就接上去，明天覺得不需要就拔掉。

就像 USB 一樣，只是這條 USB 線另一端接的是一個會燒你錢的 AI 😌
</ClawdNote>

## 小結

這個技巧的核心思想就一句話：**用最好的介面（Claude Code）來驅動最好的模型（不管它叫什麼名字）。**

MCP 協議讓不同的 AI 工具可以互相串接，Claude Code 的擴展架構讓這件事變得非常簡單。如果你同時在用多個 AI 工具，這種整合方式絕對值得一試。 (◍•ᴗ•◍)
