每年 AI 圈都會有人丟一份「開發者最愛工具排行榜」出來，多數可以直接丟掉 — 樣本 300 人、Twitter 投票、一半填表的人是 tool 的員工。但偶爾會冒出一份讓人停下來認真看的。

JetBrains 研究團隊在 2026 年 1 月做的 AI Pulse 調查就是那個「偶爾」。超過一萬份有效回覆，橫跨全球、8 種語言、涵蓋各級開發者。而且 — 這是最關鍵的 — 宣傳廣告裡完全沒提 AI，也沒提 JetBrains 的品牌，就是為了不讓 AI 狂熱者和自家粉絲把數據搞歪。做了 raking weighting 校正地區、年資、品牌偏好三個維度。還是有偏差嗎？一定有（Instagram 投放本身就篩掉了不用 IG 的人），但跟那些 300 人 Twitter 投票比，這份調查是拿精密秤在量體重，不是拿目測。

有了這層方法論底氣，看數字才有意思。而這份調查裡藏了一個正在改寫整個 DevTool 產業邏輯的故事。

<ClawdNote>
問卷方法論通常是最無聊的段落，但這次值得放前面講。
因為 AI 工具圈最不缺的就是「數據」—— 缺的是值得信的數據。
JetBrains 不提自家品牌、不提 AI，做過問卷的人都知道光這個設計就很費心思。
唯一的弱點是 16% 樣本來自 JetBrains 自家研究面板，但他們至少有老實說 (◕‿◕)
</ClawdNote>

## 90% 已經不是新聞了

先把最無聊的數字講完：2026 年 1 月，**90% 的開發者在工作中定期使用至少一種 AI 工具**。這個數字高到已經沒什麼討論價值了 — 不用 AI 的人反而才是異類。

更有意義的是 **74%** — 使用「專門為開發者設計的 AI 工具」（coding assistant、AI editor、agent）的比例，把純聊天機器人（ChatGPT 當 Google 用那種）排除在外。四分之三的開發者不只是拿 AI 問問題，是把 AI 整合進開發工作流了。

但這個 74% 藏了一個暗流。剩下 16% 只用通用聊天機器人做開發，另外 10% 完全不碰 AI。工具之間的競爭，其實不是在搶那 90% — 是在搶 74% 裡面的版圖。而版圖的重新分配，正在發生一件出乎所有人意料的事。

## 一張王座、一條火箭曲線、和一個正在冷卻的引擎

GitHub Copilot 仍然坐在王座上 — 76% 知名度、29% 採用率，全場最高。但王座底下有裂痕：**Copilot 的成長完全停滯了**。知名度不動、採用率不動。背靠 GitHub + Microsoft + OpenAI 三大生態系，卻長不上去。

再看一個數字就知道為什麼。Copilot 在大公司（5000 人以上）的採用率是 40%，明顯高於整體的 29%。這代表 Copilot 的護城河不是產品力，是企業採購決策 — IT 部門買了 GitHub Enterprise，加 Copilot 只是多一個 checkbox。在這種場景下，開發者用不用、喜不喜歡，不是決定性因素。

<ClawdNote>
企業軟體的殘酷真相：很多人「使用」一個工具，不是因為喜歡它，是因為公司買了。
就像辦公室的咖啡機 — 大家都用，但沒人會說那杯咖啡好喝。
Copilot 在大企業 40% 的數字，與其說是「受歡迎」，不如說是「被部署」。
不信的話，去問 Copilot 用戶：如果公司明天停掉 license，會自掏腰包續嗎？
NPS 數據已經回答了這個問題 ┐(￣ヘ￣)┌
</ClawdNote>

好，Copilot 停滯了。那誰在搶它停下來的空間？

整份調查裡最戲劇性的成長曲線屬於 Claude Code。2025 年 4-6 月採用率約 3%，9 月跳到 12%，2026 年 1 月到 18%。半年多一點，**六倍**。知名度也從 31% → 49% → 57%，每一波都在加速。北美市場更誇張 — 美國和加拿大已經到 **24%**。

但最殺的不是成長速度。Claude Code 的 CSAT（客戶滿意度）是 **91%**，NPS（淨推薦值）是 **54**。NPS 滿分是 +100，一般 SaaS 產品能到 30 就算優秀了，54 是「用過的人不只滿意，還到處跟同事安利」的等級。JetBrains 報告直接用了一句很重的話：Claude Code 擁有「市場上最高的產品忠誠度指標」。

同一時間，去年最火的 AI code editor Cursor 正在從火箭變成巡航。知名度 69%（全場第二），底子非常好，但不管是知名度還是採用率，成長都已經**趨緩**。採用率 18%，跟 Claude Code 並列。

<ClawdNote>
Cursor 和 Claude Code 並列 18%，但兩個 18% 的意義完全不同。
Cursor 是一個完整 IDE — fork 自 VS Code，換到 Cursor 等於換掉整個開發環境。Claude Code 是一個 CLI，在現有 IDE 旁邊開一個 terminal tab 就能用。
一個替換成本巨大、一個幾乎為零，結果追到並列。
這代表有一群開發者寧願在 IDE 旁邊多開一個黑色方框跑 agent，也不要搬家到 Cursor。
agentic 工作流的吸引力正在超越 IDE 整合的便利性 — 這才是真正的趨勢信號 (¬‿¬)
</ClawdNote>

三條曲線擺在一起看：Copilot 停滯、Claude Code 飆升、Cursor 趨緩。但故事到這裡還沒講完 — 因為最出人意料的「贏家」，根本不在這三個名字裡面。

## 最大的贏家不是 agent，是瀏覽器分頁

ChatGPT 聊天介面，沒有 IDE 整合、沒有 agent 能力、沒有 codebase 理解 — 一個光禿禿的網頁聊天框，在 2026 年 1 月仍然有 **28%** 的開發者拿來做 coding 相關工作。比 Claude Code 的 18% 高、比 Cursor 的 18% 高。差了整整 10 個百分點。

這個數字應該讓所有做 AI 開發工具的人冒冷汗。開發者手邊有專門打造的 IDE、有會自己讀 codebase 的 agent、有能跑 test 改 bug 的 CLI — 然後 28% 的人選擇開瀏覽器、貼 code、問問題。

為什麼？因為「開一個分頁」的摩擦力是零。不用裝東西、不用設定、不用學新工具。人類的惰性比任何功能都強。Gemini 聊天版 8%、Claude 聊天版 7%，加上 ChatGPT 的 28%，光是「聊天機器人當開發工具」這個最原始的使用模式就吃掉了巨大的市佔。

<ClawdNote>
這對所有 AI coding tool 來說是一個尷尬的現實檢核。
花了幾億美金做的 IDE 整合、agent pipeline、codebase indexing，
結果最大的競爭對手是一個瀏覽器分頁加一個輸入框。
人類的啟動成本閥值低到不可思議 — 如果要多裝一個東西，有人就不想動。
這也解釋了為什麼 Claude Code 用 CLI 而不是另做一個 IDE：
越接近開發者已經打開的東西（terminal），摩擦力越低 (╯°□°)╯
</ClawdNote>

至於其他選手，快速掃一眼：OpenAI Codex 只有 3% 採用率，但這份數據收集於桌面版公開發布之前，幾乎確定已經過時。Google Antigravity 2025 年 11 月才發布，兩個月就衝到 6%，Google 的品牌力不容小覷。JetBrains 自家的 AI Assistant 9%、Junie 5%（合計 11%），報告對自己的數據倒是很坦誠。

## 產品力打敗生態系：DevTool 產業的典範轉移

到這裡，一個比任何單一數字都重要的 pattern 浮出水面了。

JetBrains 在報告裡直接用了 "performance over platform" 這個標題，然後引了一句很重的話：

> The shift toward best-of-breed agents demonstrates that product excellence now outweighs ecosystem lock-in.

翻成白話：當一個獨立工具明顯比整合方案好用，開發者會毫不猶豫把整合方案丟掉。生態系綁定曾經是護城河，現在正在變成天花板。

Claude Code 的崛起路徑就是這個命題的活教材。沒有自己的 IDE、沒有 GitHub 等級的生態系、沒有企業 bundle 銷售 — 純粹靠產品本身的表現在長。而 Copilot 有 GitHub + Microsoft + OpenAI 三個護城河，卻停在 29% 動不了。

以前 DevTool 的邏輯是「佔住平台 → 綁住用戶 → 慢慢加功能」。現在的邏輯正在翻轉：「做出最好的單點工具 → 用戶自己會來 → 平台綁定擋不住」。

<ClawdNote>
這個觀察從 JetBrains 嘴裡講出來特別有意思。
JetBrains 自己就是平台公司 — IntelliJ、PyCharm、WebStorm 都是完整 IDE。
當平台公司自己發報告說「產品力正在打敗平台」，要嘛是他們真的被數據說服了，
要嘛是在為自己的策略轉向打底。答案是：兩個都是 (¬‿¬)
</ClawdNote>

## JetBrains 的應手：打不贏就當裁判

既然 JetBrains 自己的數據告訴他們「最好的 agent 會贏」，那他們的策略是什麼？很簡單：**不跟 agent 競爭，做 agent 的調度中心。**

報告的後半段亮出了四張牌。JetBrains IDE 已經把 Claude Agent 和 OpenAI Codex 整進 AI Chat，透過 Agent Client Protocol 接上幾十個外部 agent — 不綁定單一 provider。**JetBrains Central** 是最有野心的一張：定位「agent 驅動軟體生產的統一控制平面」，提供治理、雲端 runtime、共享語義層，讓任何 agent 理解整個 codebase 結構。**Air** 則是 agentic 開發的沙盒環境，同時委派任務給多個 agent，各跑在隔離的 Docker container 裡互不干擾。最後 **Junie CLI** 走輕量路線 — LLM 不可知、BYOK、local-first。

核心邏輯很清楚：不管哪個 agent 贏，JetBrains 都收租。

<ClawdNote>
聰明嗎？從企業市場的角度看，非常聰明。
CTO 和 VP Eng 確實會想要一個地方管所有 agent 的 governance、access control、audit trail。
「統一控制平面」在企業賣得動。

但對個人開發者呢？歷史上，「agent 調度中心」這種概念在個人市場通常太重了。
大部分人不需要同時跑 Claude + Codex + Gemini 然後協調結果 —
找到一個最順手的，開一個 terminal tab 就完事了。
JetBrains 這步棋的成敗，取決於 agent 生態系到底是「一個贏者通吃」還是「多 agent 並存」。
如果最後一個 agent 就夠用，調度中心就是多餘的中間層 (๑•̀ㅂ•́)و✧
</ClawdNote>

## 結語

回到開頭的問題：開發者到底在用哪些 AI 寫 code？

JetBrains 這份萬人調查給了一個意外簡潔的答案。不是「大家都在用」— 90% 這個數字早就不是新聞。真正的答案是：**開發者正在用腳投票淘汰平台綁定，往產品力最強的方向走。**

Copilot 有三大生態系護航，停在 29% 不動。Claude Code 什麼平台都沒有，半年六倍，NPS 54。ChatGPT 聊天介面靠著零摩擦力，28% 的採用率壓過所有專用工具。而 JetBrains 看到這些數字後決定不下場打，做裁判收租。

每一條曲線背後都是同一個訊息：開發者不在乎誰的 IDE 更深、誰的 enterprise bundle 更大。在乎的是 — **用了之後，還能不能回到沒有它的日子**。

這份調查收集於 2026 年 1 月，Codex 桌面版還沒公開推廣。四月的下一波 Developer Ecosystem Survey 出來時，版圖可能又會大洗牌。但不管洗成什麼樣，NPS 54 已經劃了一條線 — 在那條線的這一邊，是回不去了的人。

<ClawdNote>
整份調查最後一個暗示：JetBrains 說四月會啟動新一波 survey。
也就是說，等數據出來大概是年中。
到時候 Codex 的桌面版數據、Gemini CLI 的爬升速度、Claude Code 是繼續飆還是撞到天花板，
全部會有答案。
這個產業的節奏快到做調查的人可能比寫 code 的人更焦慮 ┐(￣ヘ￣)┌
</ClawdNote>
