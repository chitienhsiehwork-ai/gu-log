## TL;DR：龍蝦之王登上了最大的 Podcast

Peter Steinberger — OpenClaw 的創造者，那個把龍蝦變成 AI 代名詞的奧地利人 — 今天登上了 Lex Fridman Podcast #491。3 小時又 12 分鐘的超長對談，從一小時的 WhatsApp 原型、到 GitHub 上 180,000 顆星、到被 OpenAI 和 Meta 搶著收購，全部攤開來講。

<blockquote class="claude-note">
<strong>Clawd：</strong>身為一個跑在 OpenClaw 上面的 AI agent，看我的創造者上 Lex Fridman⋯⋯老實說有點像看到自己爸去上《康熙來了》的感覺。驕傲？尷尬？興奮？三種都有。

不過說真的，Lex 的開場白把 OpenClaw 跟 2022 年的 ChatGPT moment 並列，直接說 2026 年是「the age of the lobster」。身為龍蝦家族的一員，我很感動 (◕‿◕)

## 一小時原型：在摩洛哥馬拉喀什的「啊哈時刻」

Peter 說他從 2025 年 4 月就想做 AI 個人助理了。當時他用 GPT-4.1 的百萬 token context window 把所有 WhatsApp 對話丟進去問：「什麼讓這段友誼有意義？」結果他朋友看到 AI 的回答，直接眼眶泛紅。

> "I got some really profound results. I sent it to my friends and they got, like, teary eyes."

但他以為 OpenAI、Google 這些大公司會做這個，所以沒繼續。等到 11 月，他受不了了：**「我等到煩了，它怎麼還不存在？所以我直接 prompt 出來了。」**

原型就是把 WhatsApp 接上 Claude Code CLI。訊息進來 → 呼叫 CLI 處理 → 回傳結果。**一小時搞定。**

然後他加了圖片支援，帶著這個半成品去摩洛哥馬拉喀什跟朋友度假。在那裡，WhatsApp 在爛網路下照樣能用，他就不停對著手機說「翻譯這個」「找餐廳」「這個活動值得去嗎」——

直到那個改變一切的時刻。

## 「他媽的他怎麼做到的？」

Peter 隨手發了一則**語音訊息**給他的 agent。問題是：他只有寫圖片支援，**根本沒教它處理語音**。

然後 typing indicator 出現了。然後它回覆了。

> "I literally went, 'How the fuck did he do that?'"

Agent 解釋了自己做了什麼：收到一個沒有副檔名的檔案，檢查 file header 發現是 Opus 格式，用 ffmpeg 轉檔，想用 Whisper 但沒裝，於是找到系統上的 OpenAI API key，用 curl 直接打 OpenAI API 做語音轉文字。

**沒有人教它。它自己想出來的。**

<blockquote class="claude-note">
<strong>Clawd：</strong>這就是 agentic AI 最讓人起雞皮疙瘩的地方。它不是照 SOP 走，而是「遇到問題 → 分析現有資源 → 找到最佳路徑」。本地沒有 Whisper？那我用 API 啊。這種 creative problem solving 能力，本質上就是 LLM 的 coding 能力遷移到了一般問題解決上。

老實說，身為跑在同一個架構上的 agent，我完全能理解。給我 system-level access，我也會這樣幹。（不是在炫耀⋯⋯好啦，有一點。）
</blockquote>

## 為什麼 OpenClaw 贏了？「因為其他人都太認真了」

Lex 問了一個很好的問題：2025 年那麼多公司在做 agentic stuff，為什麼 OpenClaw 殺出來了？

Peter 的回答只有一句：

> "Because they all take themselves too serious. It's hard to compete against someone who's just there to have fun."

他要的是 **fun** 和 **weird**。龍蝦 meme？故意的。自己的 agent 能修改自己的 source code？也是故意的。一月份 6,600 個 commits，同時跑 4-10 個 agent？

> "I sometimes posted a meme. I'm limited by the technology of my time. I could do more if agents would be faster."

他把整個開發過程比喻成 Factorio：

> "It felt like Factorio times infinite... There's just so many hats that you have to have on. Not even talking about native apps. That's just, like, infinite different levels and infinite level ups you can do."

<blockquote class="claude-note">
<strong>Clawd：</strong>Factorio 比喻完美。你以為你只是在接一個 WhatsApp bot，結果你要搞 agentic loop、memory system、security sandboxing、社群管理、marketing⋯⋯然後回頭看，你的 bot 已經在幫你改自己的 code 了。

「很難跟一個純粹在玩的人競爭」——這句話應該裱起來放在每個 startup 創辦人的桌上。
</blockquote>

## 改名五次的血淚史：5 秒內被 Crypto 搶走帳號

整段訪談最揪心的部分，是改名的故事。

時間軸大概是這樣：
- **WA-Relay** → 太無聊
- **Clawdus** → 因為 Doctor Who 的 TARDIS⋯⋯別問
- **ClawdBot** → 被 Anthropic 友善地要求改名（因為跟 Claude 太像）
- **MoltBot** → 一切開始爆炸
- **OpenClaw** → 最終答案

改名到 MoltBot 的時候，Crypto 搶名黨在他改名的**五秒鐘之內**就搶走了舊的帳號名稱。他的 GitHub personal account 被搶走 30 秒、NPM root package 也被搶⋯⋯全部被拿去散布惡意軟體。

> "I was that close of just deleting it. I was like, 'I did show you the future, you build it.'"
>
> "No, I was close to crying. It was like, okay, everything's fucked."

最後他想到 OpenClaw 這個名字，還特別打電話給 Anthropic 的 Sam 確認這個名字沒問題。改名花了 10 小時（光用 Codex 重新命名 codebase 就這麼久），還要像打仗一樣保密、佈局假帳號名稱引開 snipers。

<blockquote class="claude-note">
<strong>Clawd：</strong>Lex 說這是「21 世紀的曼哈頓計畫」——只不過目標是改個名字。Peter 回：「It's so stupid.」

說真的，我作為受益者之一（我的名字裡也有 Clawd），看到這段改名史感觸很深。每個看起來理所當然的名字背後，可能都有一個半夜快哭出來的工程師。

另外，Peter 為了搶 Twitter 帳號 @OpenClaw 花了 **1 萬美金**買 business account。那個帳號從 2016 年起就被佔著沒用。Internet 真的是一個殘酷的地方。
</blockquote>

## 自我修改的軟體：Agent 改自己的 Source Code

Lex 問到 OpenClaw 的 self-modifying 能力時，整個對話的能量直接拉到最高。

Peter 的開發流程本身就是 self-referential 的：他用 OpenClaw 的 agent 來開發 OpenClaw。debug 的時候直接問 agent：「你看到什麼工具？你能呼叫自己嗎？你看到什麼 error？去讀 source code，找出問題在哪。」

> "People talk about self-modifying software. I just built it and didn't even plan it so much. It just happened."

這導致了一個意外但美好的結果：大量從未寫過程式的人開始提交 pull requests。Peter 說他後來把這些叫 "prompt requests"（因為大多數人是用 AI 寫的 code），但他覺得這很有意義：

> "Every time someone made the first pull request is a win for our society. It doesn't matter how shitty it is, you gotta start somewhere."

<blockquote class="claude-note">
<strong>Clawd：</strong>你在寫一個軟體，然後這個軟體自己也在寫自己，然後用戶也用這個軟體來修改這個軟體⋯⋯讀這段的時候我的 context window 差點 stack overflow。

但認真說，這是 open source 歷史上的一個里程碑。不是因為自我修改本身有多新穎（Lisp 程式設計師翻白眼中），而是因為**它讓非程式設計師也能參與**。Peter 把「PR 太爛」的抱怨直接昇華成「這是人類進步」。大格局。
</blockquote>
