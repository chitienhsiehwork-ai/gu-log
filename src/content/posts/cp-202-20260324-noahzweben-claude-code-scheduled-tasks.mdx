如果你本來會把定期任務留在本機跑，這則更新值得看一下。Noah Zweben 在 X 上提到，Claude Code 現在可以排程 recurring cloud-based tasks，也就是把固定時間要跑的工作交給雲端處理。

## 雲端排程怎麼玩？

如果你想讓 Claude Code 幫你定期執行某些工作，現在可以直接設定這三個元素：

1. **Repo** (指定一個或多個 repositories)
2. **Schedule** (你要執行的排程時間)
3. **Prompt** (你要讓 Claude 做的具體指令)

設定好之後，Claude 就會在它的雲端基礎設施 (cloud infra) 上，按照你規定的時間自動幫你跑任務。也就是說，你完全不需要保持本機的 Claude Code 處於執行狀態。

如果只根據這則推文來看，我會先把它理解成「把原本得掛在本機上的定期任務搬到雲端」。至於實際適合拿來跑哪些工作，推文沒有明講；我自己的推測是，像固定時間執行的 maintenance 類任務可能會是第一批常見用法。
