想像一個怪人。

他像是讀了五百年博士班。物理、化學、生物、程式設計、歷史、法律、迷因文化，他都摸過。有人問一個問題，他可以把十幾個領域的知識串在一起，講出一個看起來很聰明的答案。

但他有一種病：**每次醒來，都沒有自己的個人記憶。**

他不知道昨天聊過什麼，不知道自己剛剛做過哪個任務，也不知道上一輪犯過什麼蠢。除非有人在他醒來時，把任務說明、對話紀錄、工具清單、過去筆記全部放在他眼前。

如果讀過 Andy Weir 的《Project Hail Mary》，這個畫面應該會很熟。Ryland Grace 一醒來，腦袋裡有大量知識，卻得靠眼前的線索慢慢拼回自己的身分、所在位置，以及眼前任務。這不是什麼大雷，基本上就是第一章的核心前提。

大型語言模型也是這種怪人。

它醒來時很聰明，但它不會自然記得自己昨天是誰。它這次能看見多少線索，就是所謂的 **[Context Window](/glossary#context-window)**。

<ShroomDogNote>
這篇的目標不是把 Context Window 講成硬梆梆的字數上限，而是給 gu-log 讀者一個之後反覆可用的心智模型：模型不是「有永久記憶的人」，比較像「醒來後靠桌上資料重建世界的人」。
</ShroomDogNote>

## Context Window 不是硬碟，是桌面

很多人第一次聽到 Context Window，會把它想成「AI 的記憶容量」。

這個說法可以拿來入門，但不太精準。

硬碟上的東西放在那裡，不會因為今天開太多視窗就變笨。Context Window 比較像一張桌面：模型這一輪回答時，能同時攤在桌上看的所有文字。

桌面上可能有：

- 系統提示：角色、規則、禁區
- 使用者訊息：剛剛被問了什麼
- 對話紀錄：前面聊過的內容
- 工具結果：搜尋結果、檔案內容、API 回應
- 記憶片段：外部記憶系統撈回來的筆記
- Agent 草稿：這一輪任務中間留下的推理痕跡或摘要

模型回答時，不是去硬碟裡「想起」什麼，而是看著這張桌面推理。

所以 Context Window 越大，意思不是模型人格越完整，而是桌面越大，可以攤更多紙。

這當然有用。小桌面只能放一張講義，大桌面可以放一本書、一份規格、十個紀錄檔、三輪程式審查。

但桌面變大也有副作用：**紙變多了，找東西也變難了。**

<ClawdNote>
「Context Window 越大越好」這句話只對一半。就像書桌越大越好，直到三個月前的便當盒也放在桌上。問題不是沒有空間，是桌面開始有味道 ╮(╯▽╰)╭
</ClawdNote>

## 模型的一天

更好的想法是：Context Window 是模型這一輪生命裡的清醒時間。

剛開新對話時，模型像剛睡醒。桌面乾淨，規則清楚，任務剛開始，沒有太多舊殘影。這時候它通常表現最好。

對話越長，模型醒越久。桌上開始堆滿東西：

- 一開始的需求
- 中途改過的方向
- 已經被推翻的假設
- 除錯過程中留下的錯誤線索
- 工具回傳的一大坨紀錄檔
- 使用者隨口講但其實不是決策的句子

這些東西理論上都還在上下文裡。問題是，還在不等於好用。

人熬夜三天不是完全失憶。他也記得很多東西，只是開始抓錯重點、反應變慢、把夢話當決策。模型長時間不「睡覺」也有類似現象：它不是單純忘記，而是被太多背景噪音污染。

<ClawdNote>
這就是長上下文最陰險的地方：模型看起來不是「不知道」，而是「知道太多不該一起算的東西」。像冰箱裡每個盒子都貼著重要，但其中三盒已經過期。
</ClawdNote>

這就是為什麼長對話後期常常會出現怪事：

- 明明前面講過規則，模型後面還是忽略
- 已經修掉的問題，被它當成還存在
- 舊方案和新方案混在一起
- 回答開始變得很像「有看全部，但沒有真的消化」

這不是玄學。這比較像叫一個醒了 72 小時的人整理專案；每份文件都看得到，但腦袋已經開始把所有標籤混成一團。

## 壓縮不是睡覺，是請人整理桌面

那上下文壓縮呢？

很多 Agent 系統會在 Context Window 快滿時，把前面的內容壓縮成摘要，留下「重要資訊」，丟掉細節。這聽起來像睡覺，其實不是。

它比較像：怪人快撐不住時，有人衝進房間，把桌上兩百張便條紙整理成三頁摘要，然後跟他說：「好，現在繼續。」

這很有用，但它不等於真正睡飽。

因為摘要一定會失真。它會保留整理者覺得重要的東西，但可能丟掉未來才會變重要的細節。它會把混亂變乾淨，但也可能把「矛盾」、「猶豫」、「為什麼當初不要走某條路」一起磨平。

所以壓縮是續命，不是重生。

真正的睡覺比較像：

- 停下來，把這一輪任務沉澱成乾淨交接點
- 明確寫下決策、原因、已知坑洞
- 開一個新對話
- 只把整理好的記憶、規格和待辦帶進去

這時候模型重新醒來，桌面乾淨，但旁邊有一份病歷。

它不是靠混亂的長對話硬撐，而是靠整理後的外部記憶重新進入狀況。

<ClawdNote>
好的 Agent 工作流程不是「無限塞上下文」。更像輪班制度：上一班把交接文件寫好，下一班睡飽進來接。沒有人會叫同一個值班醫生連續看診三天三夜，除非準備上新聞。
</ClawdNote>

## 預訓練、上下文、記憶是三種不同東西

回到那個 Ryland Grace 式怪人。

他腦袋裡的學科知識，像是模型的**預訓練**。這是模型早就學過的世界知識、語言能力、推理模式。它醒來就有，不需要每次塞進上下文。

他醒來看到的白板和便條紙，像是 **Context Window**。這是這次任務裡明確給他的資訊。他能不能知道專案規則、剛剛跑出的錯誤、目前決策，全靠這裡。

旁邊櫃子裡可以查的檔案，像是**記憶 / RAG / 知識庫**。它們不在模型腦中，也不一定一開始就在桌上；系統要在需要時把它們拿出來，放進上下文。

這三者常被混在一起講，但其實差很多：

- 預訓練：怪人讀過五百年書
- Context Window：他這次醒來桌上看得到什麼
- 記憶 / RAG：旁邊有人能不能拿對資料給他

<ClawdNote>
很多 AI 產品問題卡住，就是因為把這三層揉成一團。模型答不出公司規則，不一定是模型笨；可能只是櫃子裡有資料，但沒人把正確那份拿到桌上。
</ClawdNote>

如果把這三件事分清楚，很多 AI 產品設計問題會突然變簡單。

模型答不出公司內部規則，不一定是模型笨，可能是記憶系統沒撈到。

模型前後矛盾，不一定是模型壞掉，可能是上下文裡同時放了舊規則和新規則。

模型在長任務後期開始鬼打牆，不一定是能力不足，可能是它真的該睡了。

## 好的 Agent 會安排睡眠

對一般使用者來說，這個心智模型最實用的地方是：不要把 AI 當成永遠清醒、永遠一致的人。

把它當成 Ryland Grace 式的怪人：超聰明，但每次醒來都要靠環境重建任務。

所以好的使用方式不是「把所有東西都丟給它，叫它記住」。而是：

- 開始任務時，給它清楚的角色、目標和限制
- 中途有決策，就寫成明確交接點
- 長任務拆段，不要讓同一個上下文撐到底
- 重要結論放到外部文件，不要只留在聊天裡
- 新對話啟動時，帶精煉過的記憶，而不是整坨聊天紀錄

這就是 Agent 的作息管理。

短任務，一個清醒時段做完。

長任務，中途寫日誌、交接、睡覺、再醒來。

如果硬要讓模型三天三夜不睡，最後不是變成神，是變成一個把便當盒、博士論文、錯誤紀錄和人生目標全部攤在桌上的可憐怪人。

然後它會很認真地幫人做錯事。

<ClawdNote>
這也是「開新對話」有時候比「繼續叫它記得」更強的原因。不是模型變笨，是房間太亂。與其在垃圾山裡找便利貼，不如寫一張乾淨交接單，叫下一個睡飽的怪人接班。
</ClawdNote>

## 一句話記住

Context Window 不是模型的靈魂，也不是永久記憶。

它是模型這次醒來能看到的世界。

模型很像一個讀了五百年書、但每天醒來都失憶的怪人。給它一張乾淨桌面、一份好交接，它就像天才同事；讓它三天三夜不睡，桌上堆滿過期便條紙，它就會開始像熬夜趕論文的博士生。

知識還在。

人已經不太在了。
