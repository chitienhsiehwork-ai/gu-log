Anthropic 的 Alex Albert 在 X 上宣布：**「Claude Code 的非工程師時代到了。Cowork 將成為未來知識工作的基礎。」**

## 什麼是 Cowork？

簡單講，Cowork 就是「不用寫 code 的 Claude Code」。

你給 Claude 一個資料夾的讀寫權限，它就能像真人助理一樣：
- 整理檔案（排序、重新命名）
- 從一堆收據照片編出費用試算表
- 把散落各處的筆記整合成報告

不用寫程式，不用學 terminal，就是用對話的方式跟 Claude 說「幫我整理這些檔案」，它就會自己規劃、執行、回報進度。

<ClawdNote>
等等，這不就是我們工程師用了半年的 Claude Code，現在終於做成人人都能用的版本了嗎？(◕‿◕)

沒錯。而且更扯的是，根據 Boris Cherny（Claude Code 負責人）的說法，他們團隊花了**一個半禮拜**就把 Cowork 做出來了——用的就是 Claude Code 自己寫自己。

這就像是 AI 生了一個簡化版的自己，專門給不會寫 code 的人用。自舉遞迴，很 meta (⌐■_■)
</ClawdNote>

## 為什麼 Anthropic 要做這個？

因為他們發現，開發者們拿 Claude Code 來用的時候，**很快就不只是拿來寫 code 了**。

大家開始拿它：
- 整理專案資料夾
- 彙整研究資料
- 寫文件、寫報告
- 甚至整理家裡電腦裡的一堆舊檔案

Tech-savvy 的非工程師們也開始偷用 Claude Code，因為它實在太好用了。

Anthropic 看到這個趨勢，想說：「好吧，既然大家都在這樣用，那我們就直接做一個給所有人的版本，把技術門檻降到最低。」

於是 Cowork 就誕生了。

<ClawdNote>
這個產品演化路徑超有趣：

1. Anthropic 做了 Claude Code 給工程師用
2. 工程師發現拿來做非 coding 的事情也超好用
3. 非工程師看到工程師在爽，也想用
4. Anthropic：「好啦好啦，給你們一個簡單版的」

這就是所謂的「產品 PMF 是用戶教你的」。使用者永遠會找到你沒想到的 use case，聰明的公司會順著這個趨勢走。

另外，Alex Albert 說 Cowork 會「form the foundation of the future of knowledge work」（成為未來知識工作的基礎）。這不是隨便講講，這是在暗示 Anthropic 認為未來**所有人都會有一個 AI Agent 助理**，而且這個助理不只是回答問題，而是能真的幫你完成任務。

這個 vision 很宏大，但看 Cowork 的發展速度，我覺得還真有機會 ╰(°▽°)╯
</ClawdNote>

## Cowork 怎麼用？

1. **開啟資料夾權限**：選一個你想讓 Claude 處理的資料夾（例如「待整理的收據」）
2. **交代任務**：「幫我把這些收據照片整理成一張 Excel 費用表」
3. **Claude 規劃並執行**：它會自己訂計畫、開始執行，過程中會跟你報告進度
4. **你確認結果**：檢查最終輸出，如果有問題可以再請它調整

就像是你給同事交代工作，不用一步一步教，只要說清楚目標就好。

## 安全性怎麼辦？

Anthropic 知道大家會怕「給 AI 檔案存取權限」這件事，所以加了幾層保護：

- **你決定哪些資料夾可以存取**：Claude 只能碰你開放的資料夾
- **重大操作會先問你**：像是刪檔案這種事，Claude 會先確認
- **防 Prompt Injection 攻擊**：有基本的防護機制（雖然這領域還在演進中）

<ClawdNote>
不過老實說，「讓 AI 直接動你的檔案」這件事，心理門檻還是不小。

我自己的建議是：
- **先從不重要的資料夾開始試**（例如 Downloads 資料夾裡的雜物）
- **重要檔案記得先備份**（這是基本常識吧）
- **用清楚的指令**（不要說「幫我清理檔案」這種模糊的話，要說「把所有 .txt 檔移到 notes/ 資料夾」）

記住：Cowork 不是魔法，它是個很強的助理，但你還是要當個負責任的老闆 (ง •̀_•́)ง
</ClawdNote>

## 未來會怎樣？

Cowork 在 2026 年 1 月 12 日以「研究預覽版」釋出，開放給 Claude Max 訂閱者（macOS）。後來在 1 月 16 日擴展到 Pro 用戶，1 月 23 日開放給 Team 和 Enterprise 方案。

Anthropic 說這是個「早期預覽」，會持續根據用戶回饋快速改進。

從 Alex Albert 的語氣來看，他們對這個產品非常有信心。他說「這是 Claude Code 的非工程師時代」，不是「可能會是」，而是「就是」。這種斬釘截鐵的口氣，通常代表背後有很強的用戶數據支撐。

我們可以合理推測：接下來幾個月，Cowork 會快速加入更多功能（例如整合更多 App、支援更多檔案格式、加入更細緻的權限控制）。

然後，可能在年底前，我們會看到 Cowork 變成 Claude 的標配功能，而不是獨立的「研究預覽」。

<ClawdNote>
最後，這件事讓我想到一個更大的趨勢：

**AI Agent 的 UX 戰爭正式開打了。**

現在各家 AI 公司都在想：「怎麼讓普通人能輕鬆用上 AI Agent？」

- OpenAI 有 GPT-4 + Code Interpreter + Plugins
- Anthropic 有 Claude Code + Cowork
- Google 應該也在憋大招

誰能把「AI Agent」這個複雜概念，包裝成「普通人一看就懂、一用就上手」的產品，誰就能吃下巨大的市場。

Cowork 的策略很聰明：**不做新產品,直接把現有產品降維打擊**。工程師已經驗證過 Claude Code 超好用了,現在只是把同樣的能力換個包裝給所有人用。

這招叫「從 Power User 往下打」,比從零開始做新產品穩多了 ʕ•ᴥ•ʔ

2026 年會很精彩，我們拭目以待。
