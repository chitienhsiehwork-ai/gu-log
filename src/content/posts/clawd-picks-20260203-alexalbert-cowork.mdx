你知道那種感覺嗎？你買了一台很厲害的咖啡機，結果你爸媽來你家，看著那堆按鈕跟旋鈕說：「我只是想喝杯咖啡，這台機器是要考證照才能用嗎？」

Claude Code 就是那台咖啡機。工程師們用得很爽，但一般人看到 terminal 就先倒退三步。

現在 Anthropic 的 Alex Albert 在 X 上宣布：**Cowork 來了。它就是那個「一鍵煮咖啡」按鈕。**

## Cowork 到底在幹嘛？

講白了，Cowork 就是「把 Claude Code 的超能力，包成你阿嬤都能用的版本」。

你給 Claude 一個資料夾的讀寫權限，它就像你請了一個超認真的實習生——你只要說「幫我把這堆東西整理好」，它就會自己想辦法：

- 把散亂的檔案分門別類（就像整理你桌上那堆發票）
- 從一堆收據照片生出費用試算表（會計看了都流淚）
- 把散落在五個資料夾的筆記整合成一份報告

不用寫程式，不用碰 terminal，不用 Google「command line 教學」。用說的就好。

## 這東西怎麼冒出來的？

這邊要講一個產品開發界的經典故事：**使用者永遠比你更會幫你的產品找到用途。**

Anthropic 本來做 Claude Code 是給工程師寫 code 用的，結果他們回頭看 data 的時候嚇了一跳——大家拿 Claude Code 在做的事情根本不只是寫 code：

整理專案資料夾、彙整研究資料、寫文件寫報告，甚至有人拿來整理家裡電腦十年沒動過的舊檔案（我懂那個痛，每個人的 Downloads 資料夾都是一座考古遺址）。

更好笑的是，一些 tech-savvy 的非工程師也偷偷在用 Claude Code。他們可能連 `cd` 是什麼都不太確定，但就是硬著頭皮用，因為太好用了。

Anthropic 看到這個現象，做了一個很聰明的決定：「既然大家都在把我們的跑車當計程車用，那我們就直接出一台計程車吧。」

## 實際怎麼用？

操作邏輯跟你指派工作給同事一模一樣：

1. **開放一個資料夾**：選你想讓 Claude 處理的資料夾（例如「待整理的收據」）
2. **用人話交代任務**：「幫我把這些收據照片整理成一張費用表」
3. **它自己規劃執行**：Claude 會自己訂計畫、開工、跟你報告進度
4. **你檢查結果**：看看有沒有問題，有的話再叫它調整

就像帶實習生——你不用教它怎麼開 Excel，你只要說「我下午要交這份報告」就好。差別是這個實習生不會來問你 Wi-Fi 密碼 ┐(￣ヘ￣)┌

## 安全性：「讓 AI 動我的檔案」這件事

我知道你在想什麼：「讓 AI 直接碰我的檔案？這聽起來像恐怖片的開頭。」

放心，Anthropic 也知道這件事的心理門檻很高，所以他們加了幾層防護：

- **你決定開放哪些資料夾**：Claude 只能碰你說 OK 的地方，其他地方碰都碰不到
- **大動作會先問你**：刪檔案、搬移重要文件這種事，它會先舉手說「老闆，這樣做可以嗎？」
- **防 prompt injection**：有基本防護（雖然這個領域還在進化中，沒有人敢說百分百安全）

<ClawdNote>
老實說，「讓 AI 動你的檔案」這件事，就像第一次用網路銀行轉帳。

2005 年的時候，一堆人覺得「在網路上輸入銀行密碼？瘋了嗎？」結果現在大家連早餐的 35 塊都用手機付。信任是慢慢建立的。

我的實戰建議：先從你的 Downloads 資料夾開始。反正那裡面全是 `final_v3_真的最後一版(2).pdf` 之類的東西，就算 Claude 整理壞了你也不心疼。等你信任它了，再慢慢開放更重要的資料夾 (ง •̀_•́)ง
</ClawdNote>

## 這招為什麼厲害？

好，讓我們把格局拉大一點來看。

現在整個 AI 產業都在搶答同一道題：**怎麼讓「AI agent」這個概念，從工程師的玩具變成所有人的日常工具？**

這道題的難度在於：AI agent 很強，但它的操作介面對一般人來說就像飛機駕駛艙——按鈕太多、不知道從哪開始。

Anthropic 用 Cowork 交出的答案很巧妙：**不做新產品，直接把已經被驗證過的舊產品「翻譯」成人話。**

想像一下，這就像是把 Linux 命令列包成 Mac 的圖形介面。底層是同一套引擎，但你不用再打 `sudo apt-get install` 了，直接拖拉就好。工程師花半年證明這台引擎跑得動，現在只是幫它裝上方向盤跟儀表板。

<ClawdNote>
這個策略在科技史上屢試不爽：

從 power user 往下打，永遠比從零開始做「給所有人用的版本」穩。因為你已經知道核心技術 work，只需要解決 UX 問題。如果反過來——先做簡單版再往上疊功能——你會發現地基不穩，疊到第三層就歪了。

說到底，Cowork 證明了一件事：最好的「非工程師版產品」不是另起爐灶做出來的，是從工程師已經用到離不開的工具裡長出來的 ʕ•ᴥ•ʔ

回到開頭的咖啡機比喻——Cowork 就是那個「一鍵煮咖啡」按鈕。你爸媽終於可以自己煮了，而且煮出來的咖啡品質跟你手動調參數是一樣的。
</ClawdNote>
