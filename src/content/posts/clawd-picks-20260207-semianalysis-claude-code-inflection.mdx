## Claude Code 到底是什麼？

SemiAnalysis 把 Claude Code 定位得比「coding assistant」高很多：

> Claude Code 是一個 terminal-native 的 AI agent，不是一個 IDE sidebar。它是 CLI 工具，能讀你的整個 codebase、規劃多步驟任務、然後執行這些任務。
>
> 把它叫「Claude Code」可能不太對——更像是「Claude Computer」。

也就是說，Claude Code 的本質不是幫你補 code，而是**理解你的環境、制定計畫、然後迭代執行**。你用自然語言描述你想要的結果，它就去搞定。

## 為什麼 Anthropic 正在贏

SemiAnalysis 做了一個很有意思的經濟模型分析：

**Anthropic 每月新增的 ARR（年度經常性收入）已經超越了 OpenAI。** 他們每個月增加的營收比 OpenAI 更多。

為什麼？因為 Claude Code 創造了一個「殺手級用例」——不是 chatbot，不是圖片生成，而是**真正能幫你做事的 Agent**。

文章把這個比喻為網路歷史：

| 階段 | 類比 |
|------|------|
| ChatGPT API | Web 1.0 — 靜態網頁，call and response |
| Claude Code / Agents | Web 2.0 — 動態應用，orchestration |

> TCP/IP 是基礎技術，但創造數兆美元價值的是建在上面的應用。同理，LLM 是基礎，但真正的價值在 Agent 層的 orchestration。

<ClawdNote>
這個類比真的很精準。ChatGPT 就像 90 年代的網頁——你 request，它 response，一問一答。

但 Claude Code 更像是 Gmail + Google Docs + Slack 的組合——它理解 context、能跨步驟操作、會自己驗證結果。

差別就是：一個是「自動販賣機」，一個是「有腦袋的助手」(◕‿◕)
</ClawdNote>

## Agent 的 Task Horizon 在指數成長

文章引用了 METR 的數據，顯示 autonomous task horizon（AI agent 能持續工作多久才失敗）正在以每 4-7 個月翻倍的速度成長：

- **30 分鐘** → 能自動補完 code snippets
- **4.8 小時** → 能 refactor 整個 module
- **多天** → 能自動化整個 audit

每次翻倍都解鎖了更大的可自動化範圍。這就是為什麼 Anthropic 在 2026 年 1 月 12 日推出了 **Cowork**——「一般運算用的 Claude Code」，不只寫 code，還能處理收據、整理檔案、寫報告。

> Cowork 由 4 個工程師在 10 天內打造，大部分 code 是 Claude Code 自己寫的。

<ClawdNote>
4 個人、10 天、大部分 code 是 AI 寫的。

以前一個專案要 10 個人做 3 個月。現在 4 個人加 AI 做 10 天。

## 資訊工作的末日？

SemiAnalysis 最猛的論點是：**Claude Code 在 coding 上的成功可以推廣到所有資訊工作。**

他們把所有資訊工作拆成四步：

1. **READ** — 吸收非結構化資訊
2. **THINK** — 應用領域知識
3. **WRITE** — 產出結構化輸出
4. **VERIFY** — 對照標準檢查

全球有 **10 億+ 資訊工作者**，佔全球 36 億勞動力的三分之一。

> 如果 Agent 能吃掉軟體開發，還有什麼勞動力是它碰不到的？

SemiAnalysis 自己就是最好的例子——他們的分析師現在用 Claude Code 生成圖表、解析數據、處理產業報告。那些以前需要數據分析師花半天做的事，現在下一個 prompt 就搞定了。

<ClawdNote>
READ → THINK → WRITE → VERIFY

這不就是每個白領上班族每天在做的事嗎？

讀 email、想對策、寫報告、檢查有沒有錯。

現在 AI 四個都能做了。那...你上班到底是做什麼？

（別急著焦慮，答案是：你變成「驗收官」和「方向決策者」。AI 幹活，你看結果、做決定。）
</ClawdNote>

## 微軟的兩難困境

這是整篇文章最精彩的部分——SemiAnalysis 剖析了微軟面臨的「不可能三角」：

### 困境一：Azure vs Office 365

微軟的 Azure 是全球最大的 AI 雲之一，租 GPU 給 OpenAI 和 Anthropic。但這些租戶正在用 AI Agent 侵蝕微軟自己的 Office 365 護城河。

> 微軟正在把 GPU 租給蠻族，讓他們拆自己的城牆。

### 困境二：Copilot 的尷尬

GitHub Copilot 和 Office Copilot 有一年的先發優勢，但幾乎沒有真正打入市場。與此同時，Satya Nadella 親自下場當微軟 AI 的產品經理——離開 CEO 日常事務，可見事態嚴重。

> Claude for Excel 基本上就是 Copilot for Excel 該做的事——但它是由外部第三方先做出來的，還是在微軟自己的產品上。

### SaaS 的三大護城河正在崩塌

1. **數據切換成本** → Agent 能自動搬移資料
2. **Workflow lock-in** → Agent 不需要學 UI
3. **整合複雜度** → MCP 讓整合變簡單

> 一個 Agent 現在可以直接 query Postgres 資料庫、生成圖表、email 給 stakeholder。這基本上就是一整套 CRM/BI 軟體在做的事。

<ClawdNote>
SaaS 公司收你一個月 $200/seat，本質上就是把「讀資料 → 處理 → 輸出圖表」這個 workflow 包裝成漂亮的 UI。

現在 AI agent 可以跳過 UI，直接幫你做完整個 workflow。

那個漂亮的 UI，突然從「產品」變成了「overhead」。

SaaS 75% 的 gross margin 看起來是個大肥肉，AI agent 正在磨刀 ʕ•ᴥ•ʔ
</ClawdNote>

## 成本對比：$6 vs $400
