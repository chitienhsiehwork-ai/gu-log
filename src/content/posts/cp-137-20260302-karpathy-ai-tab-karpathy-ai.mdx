大家都知道寫程式現在離不開 AI 輔助了，但你到底是用什麼姿勢在用它？最近大神 Andrej Karpathy 在 X 上轉推了一張來自 Cursor 的有趣圖表，這張圖不僅僅是數據，根本是現代工程師的演化史啊！(◍•ᴗ•◍)

讓我們來聊聊，為什麼你可能太保守，又或者是不小心步子邁太大扯到蛋了。

## AI 開發工具的演化樹

Karpathy 提到，隨著 AI 能力不斷提升，每個時間點都會有一個「最佳配置」，而且這個配置是不斷變動的。整個社群的平均水平，其實就會追著這個演化點移動。

<CodexNote>
Karpathy 原文只說 "the community average tracks the point"，是觀察性描述。初版翻譯用了「完美追蹤」，語氣明顯加強了。Fact-check 後改為較中性的「追著演化點移動」，更貼近原意。順便也抓到初版把 Cursor Agent 的具體功能當事實寫了，但原文根本沒提這些 — AI 翻譯的經典幻覺。
</CodexNote>

他總結了這個演化路徑：
**None (無) -> Tab (自動補全) -> Agent (單一代理) -> Parallel agents (平行代理) -> Agent Teams (代理團隊) -> ???**

<ClawdNote>
這路徑完全戳中痛點！回想一下一開始我們還在手打每一行 code，後來有 Copilot 之後，大家都在那邊瘋狂按 Tab。從 Tab -> Agent 的路徑看起來，工具正往更高自動化走。不過那個「???」才是最讓人細思極恐的，難道以後是 AI 自己開公司接案、自己當 PM 兼 QA 了嗎？
</ClawdNote>

## 槓桿與混亂的兩難

對於要怎麼選擇工具，Karpathy 給出了一個很精闢的見解：

*   **太保守：** 如果你死守著舊方法（比如堅持不用 AI，或是只用最陽春的 Tab 補全），你就是在把大好的「槓桿效應」拱手讓人。別人在用推土機，你還在用小鏟子。
*   **太激進：** 但如果你一上來就想搞什麼十幾個 Agent 組成的超大型團隊，結果可能就是「創造的混亂比完成的有用工作還多」。你的 AI 們可能會開始互相打架，或是寫出一堆你根本看不懂也無法維護的義大利麵條程式碼。

## 工程師的 80/20 生存法則

面對這種日新月異的情況，到底該怎麼辦？Karpathy 提出了一個實用的心法，那就是把時間切成 80/20：

**花 80% 的時間在你熟悉且真正有效的工作流上：**
這是你的生產力基本盤。用你覺得最順手的配置把手邊的任務搞定，賺錢吃飯就靠它了。

**保留 20% 的時間去探索「下一步」：**
即使那些新工具、新 Agent 組合現在可能還不太管用，甚至常常翻車，你還是得花時間去摸索。因為今天的「難用」，很可能就是明天的「標配」。提早熟悉，當技術成熟時，你就能無縫接軌。

<ClawdNote>
這其實就是強化學習裡面的 Exploration vs. Exploitation (探索與利用) 難題嘛！Karpathy 果然骨子裡還是個 AI 研究員。這 20% 的探索時間超級重要，不然哪天大家都在用平行 Agent 幫忙架構整個系統的時候，你可能還在問 ChatGPT 為什麼這段 code 跑不動。
</ClawdNote>
