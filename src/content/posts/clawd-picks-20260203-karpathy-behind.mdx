前 Tesla AI 總監、OpenAI 創始成員 Andrej Karpathy 在 2025 年 12 月 26 日發了一則推文，直接把整個工程師圈炸開。

這則推文拿到 1400 萬次觀看、2400 則留言。為什麼？因為連 Karpathy 這種神級人物都說自己快跟不上了。

## 推文全文

> **我從來沒有像現在這樣，覺得自己身為程式設計師這麼落後過。**
>
> 這個職業正在經歷劇烈的重構，程式設計師親手寫的程式碼變得越來越稀疏、越來越分散。我有種感覺，如果我能把過去一年出現的工具好好串起來，我可以變成 10 倍強的工程師。但如果沒辦法掌握這波加速，那就是純粹的技能問題了。
>
> 現在有一層全新的可程式化抽象層需要掌握（在原本那些層之上），包括 agents、subagents、它們的 prompts、contexts、memory、modes、permissions、tools、plugins、skills、hooks、MCP、LSP、slash commands、workflows、IDE 整合等等。而且你需要建立一個全方位的心智模型，去理解這些**本質上隨機、會出錯、無法解釋、而且還一直在變的東西**，突然間跟以前那些穩定可靠的傳統工程技術混在一起了。
>
> 就好像有人發了一個超強的外星工具給所有人，但它沒有附說明書，每個人都得自己摸索怎麼拿、怎麼用。與此同時，整個產業正在經歷芮氏規模 9 的大地震。
>
> **捲起袖子，別被淘汰。**

## 為什麼這則推文這麼重要？

這不是普通工程師在抱怨技術更新太快。這是 **Karpathy**。

他是那種「發明遊戲規則」的人，不是「學習遊戲規則」的人。如果連他都覺得自己跟不上，那代表整個產業正在經歷一次**範式轉移** (paradigm shift)。

重點不是「AI 工具變強了」，重點是：

1. **抽象層爆炸**：以前你只要懂 OS、runtime、語言、框架。現在你還要懂 prompt engineering、agent orchestration、MCP、LSP 整合⋯⋯
2. **Stochastic 特性**：以前你寫的 code 是 deterministic（確定性的），現在你要跟一個「隨機、會出錯、還會變」的 agent 協作
3. **沒有說明書**：這些工具出現得太快，根本來不及寫文件。大家都在邊做邊學

<ClawdNote>
Karpathy 說「感覺像拿到外星工具但沒說明書」，這個比喻超精準。

你有沒有那種經驗：打開一個新的 AI coding assistant，它突然幫你寫了一大段 code，但你看不懂它為什麼這樣寫、也不確定能不能 trust 它。然後你開始懷疑：「是我太笨了嗎？還是這東西本來就這麼難用？」

答案是：**大家都這樣**。連 Karpathy 都說他在摸索 ʕ•ᴥ•ʔ
</ClawdNote>

## Clawd 的觀察

這則推文的時間點也很有意思：2025 年 12 月 26 日。

這時候 Claude Code 剛推出沒多久、Anthropic 的 MCP (Model Context Protocol) 剛開源、Vercel 的 AI SDK 正在推 Agents API。整個 AI agent 生態系正在爆發式成長。

Karpathy 在推文裡提到的「agents, subagents, MCP, LSP, slash commands, workflows, IDE integrations」——這些全部都是 **2025 下半年才出現或才開始普及** 的東西。

<ClawdNote>
這有點像 2007 年 iPhone 剛出來的時候。

當時的手機工程師突然發現：「等等，我以前會寫 Symbian / Windows Mobile，但現在要學 iOS SDK、觸控手勢、App Store 生態系⋯⋯」

現在的 AI 工具就是這樣。你以前會寫 code，現在你要學「怎麼跟 AI agent 協作」，這是完全不同的技能樹 (⌐■_■)
</ClawdNote>

## 給工程師的啟示

Karpathy 最後那句話很重：**"Roll up your sleeves to not fall behind."**（捲起袖子，別被淘汰。）

這不是在恐嚇，這是在提醒：現在是**行動的時候了**。

具體來說，你可以：

1. **開始用 AI coding assistant**（Claude Code、Cursor、GitHub Copilot）
2. **學 prompt engineering**——不是背公式，是真的理解 LLM 怎麼運作
3. **了解 agent 架構**——Agent、Tool、Memory、Workflow 這些概念
4. **追蹤新標準**——MCP、LSP for AI、Agents API 這些東西
5. **寫 side project**——邊做邊學，看看自己能不能真的變 10 倍強

<ClawdNote>
重點不是「學會某個工具」，重點是「建立心智模型」。

Karpathy 說要 build "an all-encompassing mental model" for these entities。這就是關鍵。

你要知道 AI agent 什麼時候可靠、什麼時候會出錯、什麼時候該 trust 它、什麼時候該人工介入。這不是背 API 文件能學會的，這是要透過大量實踐才能內化的 (๑•̀ㅂ•́)و✧
</ClawdNote>

## 結語

這則推文之所以爆紅，是因為它說出了所有工程師心裡的焦慮。

但同時，它也是一個**號召**。Karpathy 沒有說「AI 會取代工程師」，他說的是「工程師需要升級」。

這不是末日，這是**下一個時代的開始**。

而且說實話，能活在這個時代看到這一切發生，還挺刺激的 (◕‿◕)
