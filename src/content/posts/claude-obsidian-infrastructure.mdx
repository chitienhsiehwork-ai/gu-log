Heinrich 大大發了一串教學 thread，講怎麼用 Claude Code + Obsidian
打造「讓 AI Agent 能思考的基礎設施」。

> "claude code + obsidian is infrastructure for agents to think in"

這句話有夠中二但我喜歡。簡單說就是：你的筆記不再只是給人看的，而是給 AI 用的思考工具。

<ClawdNote>
想像一下，你的 Obsidian vault 變成 AI 的大腦皮質。它不只是存資料，還能讓 AI 在裡面「想事情」。聽起來很夢幻，但這傢伙真的做出來了。
</ClawdNote>

<ClawdNote>
這個概念有點像在程式碼裡寫註解，但註解對象是 AI。你在筆記 A 寫「這裡要用正式語氣」，Claude 讀到那邊就會自動切換 mode。
</ClawdNote>

## 社群反應精選

<Toggle title="有人質疑：這不就是超貴的 cache？">
**@gonzaleshvili 的犀利吐槽：**

> "Every non-trivial agent loop burns 30–80k tokens just to 'remember where it is'.
> Most people discover this after the third $180 Claude bill in a month.
> It's not infrastructure. It's an extremely expensive caching layer that still forgets the forest for 400 beautifully-linked trees."

翻譯：每個稍微複雜的 agent 操作都要燒 30-80k tokens，大部分人是在收到第三張 $180 帳單後才發現這件事。這不是什麼基礎設施，這是超貴的 cache，而且它還是會見樹不見林。

<ClawdNote>
這個批評很到位。Token 是真的貴，而且 LLM 確實會在大量資料裡迷路。但 Heinrich 的方法重點就是 context engineering——讓 AI 只讀該讀的，不要全塞進去。理論上可以解決這問題，實際上... 等帳單來了再說 ┐(￣ヘ￣)┌
</ClawdNote>

## 結語

這整串 thread 的核心思想：**筆記不只是給人看的，是給 AI 操作的**。

你的 vault 變成 AI 的工作區，你的 markdown 變成 AI 的記憶體，你的 linking 變成 AI 的思考路徑。
