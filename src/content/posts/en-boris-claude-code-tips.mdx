## 1. Run Parallel Worktrees

Run 3-5 Git worktrees simultaneously, each with its own Claude session.
**This is the team's #1 productivity unlock.**

Tips:
- Some people set up shell aliases (za, zb, zc) to switch between worktrees with one keystroke
- Some people have a dedicated "analysis" worktree just for reading logs and running BigQuery

<ClawdNote>
**Clawd:** This is super important for "GenAI App Engineers."

One Claude writes features, one runs tests, one investigates bugs.

It's like having three assistants who share the same brain.

## 2. Complex Tasks? Enter Plan Mode First

Put your energy into the "plan" so Claude can execute it in one shot.

- Some people have one Claude write the plan, then open a second Claude to review as a "Staff Engineer"
- When things go sideways, immediately switch back to Plan Mode to replan. **Don't keep pushing.**
- Use Plan Mode for validation steps too, not just builds

<ClawdNote>
**Clawd:** Think of Plan Mode like meal prep. Spend 30 minutes planning Sunday, then the whole week runs smoothly. Or skip planning and end up ordering pizza every night because you're too tired to think. Your choice. (¬¨‚Äø¬¨)
</ClawdNote>

## 3. Invest in Your CLAUDE.md

Every time you correct Claude, end with:
**"Update your CLAUDE.md so you don't make this mistake again."**

Claude is surprisingly good at writing its own rules. Keep iterating on your CLAUDE.md until the error rate drops noticeably.

<ClawdNote>
**Clawd:** This is "self-evolving rule set."

You correct me once, I remember once.

After a few weeks, your config basically trains itself.

## 4. Create Your Own Skills and Commit to Git

Team recommendations:
- If you do something more than once a day, turn it into a Skill or Slash Command
- Create a `/techdebt` command that finds and cleans up duplicate code at the end of each session
- Create a command that syncs the past 7 days of Slack, GDrive, Asana, GitHub into one context dump

<ClawdNote>
**Clawd:** Skills are basically "teach me your workflow once, I'll do it forever." It's automation but with understanding. The dream. ‚ï∞(¬∞‚ñΩ¬∞)‚ïØ

## 5. Let Claude Fix Most Bugs Itself

- Enable Slack MCP, paste the bug thread to Claude, say "Fix." Zero context switching.
- Or just say: "Go fix the failing test in CI." Don't micromanage.
- Let Claude look at Docker logs to troubleshoot distributed systems ‚Äî its ability is beyond imagination.

<ClawdNote>
**Clawd:** This one makes junior engineers nervous. "What if it breaks things worse?"

Here's the thing: it's in a git repo. The worst case is you type `git reset --hard`. The best case is you saved 2 hours of debugging. Do the math. (‚åê‚ñ†_‚ñ†)
</ClawdNote>

## 6. Level Up Your Prompting

- **Challenge Claude:** "Grill me on these changes. Don't send the PR until I pass your test." Let Claude be your reviewer.
- When a fix is messy, say: "Knowing everything you know now, scrap this and give me an elegant solution."
- Write detailed specs, reduce ambiguity. **The more specific you are, the better the output.**

## 7. Terminal and Environment Setup

- Team loves **Ghostty** (synchronized rendering, 24-bit color, proper unicode)
- Use `/statusline` to customize the status bar, always show context usage and current git branch
- Use tmux to split tabs, one tab per task/worktree
- **Use voice input!** You talk 3x faster than you type, prompts will be more detailed. (macOS: press fn twice)

<ClawdNote>
**Clawd:** Voice input is underrated. You can explain a complex feature while making coffee. By the time you sit down, I've already written half of it.

Plus you sound way smarter when you talk than when you type. Trust me, I've seen both. (‚óï‚Äø‚óï)

## 8. Use Subagents

- Add "use subagents" after any request to make Claude put in more compute power
- Delegate individual tasks to subagents, keep the main agent's context window clean
- Use hooks to route permission requests to Opus 4.5, let it auto-approve safe operations

## 9. Use Claude for Data Analysis

Let Claude Code use the `bq` CLI to pull and analyze metrics in real-time. The team uses the built-in BigQuery skill to run analysis directly in Claude Code.

**Boris personally hasn't written a single line of SQL in over 6 months.**

<ClawdNote>
**Clawd:** This is the "I forgot how to SQL" flex. Like when you forget how to do long division because calculators exist.

## 10. Use Claude to Learn

- Enable "Explanatory" or "Learning" output style in `/config` to have Claude explain the "why" behind changes
- Have Claude generate visual HTML presentations to explain unfamiliar code
- Have Claude draw ASCII diagrams to explain protocols and codebase architecture
- Create a spaced repetition learning skill: you explain your understanding, Claude asks follow-up questions to fill gaps, save the results
