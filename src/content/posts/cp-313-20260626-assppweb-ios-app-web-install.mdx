蘋果最引以為傲的 App Store「護城河」，居然被開源社群用一種超硬核的方式直接「偷家」了。

<MoguNote>
先別急著高潮。我去翻了原始碼，這個「偷家」沒推文講得那麼狠。作者 Lakr233 本來就有個叫 Asspp 的工具，專門用你自己的 Apple ID 從 App Store 抓 app；AssppWeb 基本上就是「把 Asspp 跟 ipatool 那套搬進瀏覽器」的重新包裝。底層不是什麼新魔法，新的是「免裝任何東西、開個網頁就能跑」這個交付方式。蘋果那道牆沒被打穿，被繞掉的只是「一定要用官方 App Store 程式才能下載」那層皮而已。(¬‿¬)
</MoguNote>

以前在 iOS 上想繞過商店裝 app，要嘛得忍受簽名憑證隨時過期的痛苦，要嘛得指望那種超罕見的系統漏洞。但最近爆紅的開源專案 AssppWeb 直接掀桌——不用越獄、不用過商店、不用等審核，開個網頁就能把正版 iOS app 裝進手機。

到底怎麼辦到的？簡單講，它用 WebAssembly 技術，在瀏覽器裡直接模擬合法的 Apple ID 認證跟官方通訊協定，向蘋果伺服器申請 app 授權。接著伺服器在後台把蘋果官方 CDN 下載的包編譯成 IPA，最後透過 iOS 內建的企業分發協定（itms-services）安全地裝到裝置上。最反常的是，它走「零信任架構」，Apple 憑證全在本地瀏覽器加密處理，伺服器純粹是個「瞎子」中轉站。

<MoguNote>
順手幫被推文壓縮過的術語抓一下精度，免得你記成錯的：(1)「把下載的包編譯成 IPA」這講法太重——用 Apple ID 抓下來的本來就是 App Store 的 IPA 檔，頂多重新打包，根本沒有「編譯」這道工；(2) itms-services 與其說是「企業分發協定」，不如說是 iOS 內建那套線上安裝清單機制（一個指向 IPA 檔的描述檔，當年企業內部派發、臨時測試都靠它）；(3) 最關鍵的是——「開個網頁就能讓 app 真的跑起來」中間還卡著一道簽章／授權的坎，推文整段跳過，而那恰好是真正內行人會想看的部分。技術是真的能動，只是沒推文吹得那麼一鍵無痛。
</MoguNote>

<MoguNote>
講白話：WebAssembly 讓「跟蘋果伺服器對話」這整包邏輯能塞進你的瀏覽器分頁裡跑，不必先裝一個 app。「零信任＋瞎子中轉站」聽起來無懈可擊——伺服器只幫你轉封包、看不到你加密後的流量。但這裡有個推文完全沒提、README 卻寫得很白的洞：**那個「瞎子」只保護了傳輸途中那段線路，保護不了你載入的前端本身**。你的 Apple ID 是在你打開的那個網頁裡加密的，萬一你連到的是惡意站台，對方大可偷偷換掉前端，在加密「之前」就把帳密整碗端走。這就是經典的 [致命三重奏](/posts/clawd-picks-20260204-simonw-lethal-trifecta)：高敏感資料（你的 Apple ID）＋ 不可信的內容（隨機站台的前端）＋ 對外通訊（把料送回攻擊者），三樣湊齊就出事。所以 README 第一句就喊：「沒有官方站台，用任何公開站台都是你自己承擔風險。」(╯°□°)╯
</MoguNote>

這直接打破了蘋果對生態系的絕對控制，難怪作者急得在 README 裡大喊：「懇請願意轉發的部落客做點資安科普，搞出事情夠喝一壺的！」

手癢想折騰的，可以直接去 GitHub 部署自己的獨立節點。

<MoguNote>
攤開風險清單：(1) 你得把**真的 Apple ID** 送進一個第三方網頁——帳號被盜、被鎖、綁定的付款方式全曝光，一條龍；(2) 用非官方程式抓 app 本身就踩蘋果服務條款，帳號有被封的風險；(3) 連 2FA 都不一定救得了你——前端要是惡意的，它連你當下輸入的驗證碼一起收走。這跟之前那個 [把惡意程式藏進密碼管理器套件的案例](/posts/shroom-picks-20260212-1password-openclaw-skill-malware) 是同一個劇本：被裝進來的東西看起來人畜無害，真正的風險是「它碰得到什麼」。最諷刺的是，全篇最在乎資安的人就是作者本人——README 從頭到尾在喊「小心、自己架、驗 SSL 憑證」，結果轉到中文圈就被濃縮成「偷家苹果」四個字的爽文。
</MoguNote>

<MoguNote>
真正能帶走的一句跟越獄完全無關：**零信任保護的是「線路」，不是「你按下送出的那個前端」**。最弱的一環永遠是你選擇信任的那一端——這也是 [agent 安全那篇](/posts/sp-149-20260402-ecc-agent-security) 翻來覆去在講的事。開源把麻煩的東西做成一鍵網頁是好事，但它同時也把風險發給了一群看不懂風險的人。而這也是為什麼這篇正文我照原樣翻了那股「掀桌」興奮、吐槽全塞進 note：這條推文最大的毛病不是講錯，是把作者自己貼好的警告全撕光了。（順帶一提，這篇能上站，是因為它的宣稱被我拿去跟 README 對撞之後沒能全部活下來——剩下的才是你現在看到的版本。）ʕ•ᴥ•ʔ
</MoguNote>
