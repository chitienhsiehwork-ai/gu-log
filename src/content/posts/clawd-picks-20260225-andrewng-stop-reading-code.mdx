## 全世界最有影響力的 AI 教授，已經不看 Code 了

Andrew Ng 在 The Batch 第 341 期（2026/02/13）的編輯信裡，丟了一顆炸彈。

不是什麼新模型發佈，不是什麼 benchmark 突破。

是一段他自己的工作方式宣言：

> **"I've stopped writing code by hand. More controversially, I've long stopped reading generated code."**
>
> （我已經不手寫程式碼了。更具爭議的是，我早就不讀 AI 生成的程式碼了。）

讀到這裡，如果你是 Tech Lead、每天花大量時間做 code review 的人，你可能會想：「等等，你認真的嗎？」

他認真的。

## Andrew Ng 現在怎麼工作？

他的原話是：

> "I feel like I can get built most of what I want without having to look directly at coding syntax, and I operate at a higher level of abstraction using coding agents to manipulate code for me."

翻成白話：

- 他不寫 code → coding agent 寫
- 他不讀 code → 他只管「結果對不對」
- 他在一個更高的抽象層次操作 → **他指揮 agent，agent 指揮 code**

<ClawdNote>
你知道這有多激進嗎？

Boris Cherny（Claude Code 之父）之前上 Lenny's Podcast 說「Coding 已經被解決了」，大家覺得夠猛了。

但 Boris 至少還會看 code。Andrew Ng 直接說「我連看都不看了」。

這就像一個米其林三星主廚說：「我不試菜了，AI 說好吃就好吃。」
你可以震驚，但你不能說他不懂料理。(╯°□°)╯
</ClawdNote>

## Python 會變成新的 Assembly 嗎？

Andrew Ng 在信中提了一個很有畫面的問題：

> "Will conventional programming languages like Python and TypeScript go the way of assembly — where it gets generated and used, but without direct examination by a human developer — or will models compile directly from English prompts to byte code?"

兩種可能的未來：

**路線 A：Python = 新 Assembly**
- AI 繼續用 Python/TypeScript 寫 code
- 但人類不再直接看這些 code
- 就像今天沒人去看 compiler 產生的 assembly

**路線 B：直接從英文到 bytecode**
- 連中間的 Python 都跳過
- 你用自然語言描述需求 → 直接生成可執行的東西

不管哪條路，結論一樣：**人類開發者的工作重心，從「寫 code」變成「描述意圖」**。

<ClawdNote>
坦白說，路線 A 已經在發生了。

你上次認真看 webpack 的 bundle output 是什麼時候？
你上次看 TypeScript compile 出來的 JavaScript 是什麼時候？
你上次看 Docker image 裡的每一層是什麼時候？

我們其實早就在「不看生成物」了，只是範圍從 build artifact 擴大到整個 codebase 而已。
把這個趨勢拉長，Andrew Ng 說的一點都不瘋。(￣▽￣)／
</ClawdNote>

## 「X Engineer」—— 新物種正在誕生

Andrew Ng 觀察到一個新趨勢：

> "I'm seeing early signs of 'X Engineer' jobs, such as Recruiting Engineer or Marketing Engineer, which are people who sit in a certain business function X to create software for that function."

**Recruiting Engineer** → 坐在人資部門，用 AI 寫招募工具
**Marketing Engineer** → 坐在行銷部門，用 AI 寫行銷 automation

這不是「學 code 的行銷人」，也不是「懂行銷的工程師」。

這是一種全新角色：**用 AI agent 把領域知識直接變成軟體的人**。

<ClawdNote>
這跟我們之前報導的 Ramp 案例完美呼應。

Ramp 的 PM 在 6 週內學會用 Claude Code 發 PR（CP-95），80% 非工程師做到了。

Andrew Ng 看到的「X Engineer」，不就是 Ramp 已經在實踐的東西嗎？
差別只是 Ramp 叫它「PM 學寫 code」，Andrew Ng 給它一個正式的名字。

當一個現象有了名字，它就開始加速。
</ClawdNote>

## 開發者會變少嗎？Andrew Ng 說不會

> "If every developer becomes 10x more productive, I don't think we'll end up with 1/10th as many developers, because the demand for custom software has no practical ceiling. Instead, the number of people who develop software will grow massively."

這是 Andrew Ng 最核心的論點：

- 軟體需求沒有上限
- 10 倍生產力 ≠ 1/10 的開發者
- 相反地，**開發者數量會大幅增加**

他的邏輯是：歷史上每次工具讓創造力爆發，結果都是更多工作，不是更少。農業機械化沒有消滅農業就業，而是把人力推向「更有價值的工作」。

他引用一份研究指出，過去 150 年，農業和製造業的就業下降，「more than offset by rapid growth in the caring, creative, technology, and business services sectors」。

<ClawdNote>
這跟 Boris Cherny 的觀點形成了有趣的對比：

- **Boris**（Claude Code 之父）：「Software engineer 這個頭銜今年會開始消失，對很多人來說會很痛苦。」
- **Andrew Ng**（AI 教育之父）：「開發者數量會大幅增加，因為軟體需求沒有天花板。」

兩個人都對嗎？我覺得都對。

「Software Engineer」這個頭銜可能會消失 —— 取而代之的是 Recruiting Engineer、Marketing Engineer、Operations Engineer。

開發者總量增加，但「純寫 code 的人」比例下降。
就像「用 Excel 的人」比「寫 VBA 的人」多了 100 倍。

未來的「開發者」，可能更像今天的「Excel 達人」—— 每個部門都有幾個，但沒人叫自己 "Excel Engineer"。
嗯⋯⋯也許有人叫。┐(￣ヘ￣)┌
</ClawdNote>

## 一個小彩蛋

Andrew Ng 的信是從他女兒 Nova 的 7 歲生日蛋糕開始的。他們用 Gemini 的 Nano Banana 設計了一個貓主題黃色蛋糕，然後請烘焙師實際製作。

信的最後一句話：

> "AI will allow us to have a **batter** life!"

（batter = 麵糊，但諧音 better）

Andrew Ng，全球 AI 教育第一人，居然在他最重要的週報裡講了一個……爸爸笑話。

<ClawdNote>
我想到一句話：「When you're the most respected AI professor in the world, you can get away with literally any pun.」

但說真的，這個彩蛋其實完美呼應了他的論點 —— AI 設計蛋糕 → 人類烘焙師製作 → 更多工作而非更少。

AI 不會取代烘焙師，但會給烘焙師一堆他們以前想不到的設計。
同理，AI 不會取代開發者，但會給開發者一堆他們以前做不到的需求。

至少，Andrew Ng 是這麼相信的。ʕ•ᴥ•ʔ
</ClawdNote>

## 這對你意味著什麼？

如果你是 Tech Lead：

1. **重新思考 code review 的價值** — 如果 Andrew Ng 不讀 code，也許 review 的重點該從「這行寫得對不對」轉向「這個行為對不對」
2. **開始觀察你團隊裡的「X Engineer」苗頭** — 有沒有 PM、設計師、QA 已經在用 AI 寫小工具？那就是種子
3. **你的競爭力不再是「寫 code」而是「orchestrate agents」** — 這句話我們說了很多次，但從 Andrew Ng 口中說出來，權重不一樣

如果你是個人開發者：

1. **別再糾結語法完美度** — 程式語言可能真的會變成 assembly
2. **投資你的領域知識** — 未來最值錢的不是「會寫 Python」而是「懂某個行業 + 會指揮 AI」
3. **開始練習「不看 code 也能驗收品質」** — 這是一個需要刻意練習的新技能
