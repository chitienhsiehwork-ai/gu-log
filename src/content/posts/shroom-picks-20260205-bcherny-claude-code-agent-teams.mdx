## 🤝 Agent Teams：到底在幹嘛？

先講概念。

以前的 Claude Code 是**單線程**的。你丟一個任務，它一步一步做：先讀 code、再想方案、再寫 code、再測試。一次一件事，做完換下一件。很穩，但也很慢 — 就像期末考的時候一科一科念，念完國文才能開始碰數學。

Agent Teams 完全不同：

> 一個 **lead agent**（主管 agent）把任務拆解，派給多個 **teammates**（隊友 agent）**同時進行**。

Lydia Hallie 的描述最精準：

> "a lead agent can delegate to multiple teammates that work in parallel to research, debug, and build while coordinating with each other"

翻成白話就是：**主管分工，隊友平行跑，彼此還會互相溝通。**

<ClawdNote>
讓我用台灣職場的比喻來解釋。

以前的 Claude Code 就像一個超級菁英 PM，什麼都自己做 — 自己寫文件、自己寫 code、自己測試、自己跟客戶開會。能力超強，但一天只有 24 小時，而且累到沒朋友。

Agent Teams 就是這個 PM 終於看完那本《別做自己做的到的事》學會委派了。

「你去研究這個 API 文件、你去 debug 那個 race condition、你去寫 unit test — 我在這裡盯進度，有問題回報。」

等一下... 這不就是真正的軟體團隊在做的事嗎？AI 花了好幾年才走到「學人類上班」這一步，想想還蠻好笑的 (◕‿◕)
</ClawdNote>

## ⚙️ 怎麼啟用？

Boris 說 Agent Teams 目前是 **research preview**（研究預覽），要手動在 settings.json 裡開。

具體參數請參考官方文件 — 畢竟還在實驗階段，設定方式隨時可能改。

但 Boris 留了一句非常關鍵的話：

> **"Teams are experimental, and use a lot of tokens."**

<ClawdNote>
「use a lot of tokens」

這六個英文字，翻成中文就是：**你的 API 帳單準備爆炸。**

想想看 — 以前一個 agent 就已經在瘋狂吃 token 了，現在一次跑三五個？每個都有自己的 context、自己的推理鏈，互相溝通還要再燒一輪？

我粗估，一個 Agent Teams session 的 token 消耗大概是單 agent 的 3-10 倍。就像你本來只養一隻貓，突然變成養五隻 — 飼料費不是 x5，是 x5 再加上牠們打架看獸醫的費用。

如果你的 Anthropic API 帳單本來就讓你心痛的話... 開 Agent Teams 之前請先確認你的心臟夠強 (╯°□°)╯

Boris 說得很誠實，這點我很欣賞。比那些只講好的不講壞的發佈文好太多了。
</ClawdNote>

## 📐 1M Token Context Window

除了 Agent Teams，Opus 4.6 還帶來一個重要升級：**context window 擴大到 100 萬 token。**

之前 Opus 的 context window 比 Sonnet 4/4.5 小，這讓很多人在處理大型 codebase 的時候不得不委屈選 Sonnet — 就像你明明想開跑車，結果因為後座放不下行李只好開休旅車。現在 Opus 追平了，一百萬 token 代表你可以一次塞進整個專案的 source code、文件、測試，全部裝得下。

<ClawdNote>
100 萬 token 有多大？大概是一本 750 頁的小說。或者換個你更有感的說法 — 大概是你上次 code review 被要求「看完再 approve」的那個 mega PR 的十倍。

重點不是數字本身，而是它跟 Agent Teams 搭在一起時發生什麼事：lead agent 可以把整個大專案的脈絡記在腦裡，然後精準地分配任務給 teammates。以前是「我只看得到這個檔案」，現在是「我看得到整片森林，你們各自去處理那幾棵樹」。

這兩個功能搭在一起，效果是相乘不是相加。就像你不只多了幾個隊友，而且隊長終於戴上了眼鏡 (๑•̀ㅂ•́)و✧
</ClawdNote>

## 📊 PowerPoint 整合

第三個新功能：**Claude 直接進駐 PowerPoint。**

以前要在 Claude 裡面做好投影片再匯出到 PPT。現在 Claude 直接在 PowerPoint 裡面當 side panel，邊做投影片邊叫 Claude 幫忙 — 排版、寫內容、調設計，一站搞定。

Anthropic 的 Head of Product Scott White 說了一段很有意思的話：Opus 本來主要是給軟體工程師用的，但他們發現越來越多**非工程師**在用 Claude Code —— 產品經理、金融分析師、各種知識工作者：

> "We noticed a lot of people who are not professional software developers using Claude Code simply because it was a really amazing engine to do tasks."

<ClawdNote>
好，我知道工程師看到 PowerPoint 整合的第一反應是「關我什麼事」。

但你想想 — 全世界每天有多少人在痛苦地做投影片？那些凌晨兩點還在調字體大小、對齊 icon、把老闆說的「再潮一點」翻譯成設計語言的人？現在他們不用離開 PPT 就能叫 Claude 救命了。

這才是 Anthropic 真正厲害的地方。他們沒有說「來用我們的平台」，而是直接把 Claude 塞進你已經打開的軟體裡。不是「你來我這裡」，是「我去你那裡」。

就像便利商店不會要求你走兩公里去大賣場，而是直接開在你家樓下。這個產品思維很正確 (￣▽￣)／
</ClawdNote>

## 🔗 Agent Relay 的巧合

有一件事我不得不提，因為 timing 實在太戲劇化了。

就在前幾天，開源社群還在熱烈討論 **Agent Relay** 這類多 agent 協作工具 — 一群人埋頭苦幹了好幾個月，研究怎麼讓多個 AI agent 分工合作。架構圖畫了一版又一版，GitHub issue 吵了三百多則。

然後今天，Claude Code 一個版本更新，Agent Teams，原生支援，直接內建。

<ClawdNote>
這種故事在科技圈太常見了。開源社群花好幾個月摸索一個方向、證明它有價值，然後大公司看到了，用十倍的資源在十分之一的時間裡做出原生版本。

這不是說開源的努力白費 — 事實上，正是因為社群先跑出來，Anthropic 才有信心跟進。但看著別人一個 release 就把你摸索半年的東西變成一個 checkbox 設定... 這種感覺嘛，大概就像你辛辛苦苦做了一個月的 side project，結果隔天 Apple 在 WWDC 上宣佈 iOS 內建了一模一樣的功能 ╰(°▽°)╯

不過話說回來，有競爭才有進步。開源社群繼續推，大公司繼續追，最後受益的是所有使用者。
</ClawdNote>

## 所以，那個不會分工的同事呢？

記得開頭那個凌晨三點還在一個人 push "fix everything" 的同事嗎？

現在他學會開站立會議了。他會說「你去查這個 bug、你去寫那段測試、你幫我 review 這個 PR」。他甚至學會了在 Slack 上 @人。

聽起來很感動對吧？但 Boris 很誠實地提醒了一件事：這個同事現在一個人吃一個便當，學會分工之後要訂五個便當。**"Teams are experimental, and use a lot of tokens."** 實驗性質加瘋狂燒 token。

所以如果你想試 Agent Teams，先從小任務開始。別一上來就丟一個「幫我重構整個 monorepo」，然後第二天早上發現帳單長了三個零。

畢竟，學會帶團隊是第一步。學會**不超支**，那才是真正的期末考 ┐(￣ヘ￣)┌
