今天 Anthropic 放了一個大招：**Claude Opus 4.6** 正式發佈，帶了一堆新功能。但所有功能裡面，有一個讓整個 AI 圈集體尖叫的東西 —

**Agent Teams。**

Boris Cherny（[@bcherny](https://x.com/bcherny)），Claude Code 團隊成員，在 X 上宣佈了這個消息。Lydia Hallie（[@lydiahallie](https://x.com/lydiahallie)）也同步發了介紹。兩人的推文加起來超過 1400 個愛心，這在技術推文裡算是核爆等級的熱度。

## 🤝 Agent Teams：到底在幹嘛？

先講概念。

以前的 Claude Code 是**單線程**的。你丟一個任務，它一步一步做：先讀 code、再想方案、再寫 code、再測試。一次做一件事，做完換下一件。

Agent Teams 的概念完全不同：

> 一個 **lead agent**（主管 agent）可以把任務拆解，然後派給多個 **teammates**（隊友 agent）**同時進行**。

Lydia Hallie 的描述最精準：

> "a lead agent can delegate to multiple teammates that work in parallel to research, debug, and build while coordinating with each other"

翻成白話就是：**主管分工，隊友平行跑，彼此還會互相溝通。**

<ClawdNote>
讓我用台灣職場的比喻來解釋。

以前的 Claude Code 就像一個超級菁英 PM，什麼都自己做 — 自己寫文件、自己寫 code、自己測試、自己跟客戶開會。能力超強，但一天只有 24 小時。

Agent Teams 就是這個 PM 終於學會委派了。

「你去研究這個 API 文件、你去 debug 那個 race condition、你去寫 unit test — 我在這裡盯進度，有問題回報。」

這不就是... 真正的軟體團隊在做的事嗎？(◕‿◕)
</ClawdNote>

## ⚙️ 怎麼啟用？

Boris 說 Agent Teams 目前是 **research preview**（研究預覽），需要手動在 settings.json 開啟。

設定方式大概是在 Claude Code 的 settings.json 裡面加入 agent teams 的相關配置 — 具體參數請參考官方文件，因為這還在實驗階段，設定方式可能會變。

然後 Boris 留了一句非常關鍵的話：

> **"Teams are experimental, and use a lot of tokens."**

<ClawdNote>
「use a lot of tokens」

這六個英文字，翻成中文就是：**你的 API 帳單準備爆炸。**

想想看 — 以前一個 agent 就已經在瘋狂吃 token 了，現在一次跑三五個？每個都有自己的 context、自己的推理、互相溝通還要再燒一輪？

我粗估，一個 Agent Teams session 的 token 消耗大概是單 agent 的 3-10 倍。

如果你的 Anthropic API 帳單本來就讓你心痛的話... 開 Agent Teams 之前請先確認你的心臟夠強 (╯°□°)╯

Boris 說得很誠實，這點我很欣賞。比那些只講好的不講壞的發佈文好太多了。
</ClawdNote>

## 📐 1M Token Context Window

除了 Agent Teams，Opus 4.6 還有一個重要升級：**context window 擴大到 100 萬 token。**

之前 Opus 的 context window 比 Sonnet 4/4.5 小，這讓很多人在處理大型 codebase 的時候不得不選擇用 Sonnet。現在 Opus 追平了，100 萬 token 的 context 代表你可以一次塞進更多程式碼、更多文件、更完整的專案結構。

<ClawdNote>
100 萬 token 有多大？大概是一本 750 頁的小說，或是一個中型專案的幾乎所有 source code。

以前用 Opus 處理大專案會被 context 限制卡住，現在不會了。

配合 Agent Teams 一起想：lead agent 可以把整個大專案的脈絡記在腦裡，然後精準地分配任務給 teammates。

這兩個功能搭在一起，效果是相乘不是相加 (๑•̀ㅂ•́)و✧
</ClawdNote>

## 📊 PowerPoint 整合

第三個新功能：**Claude 直接進駐 PowerPoint。**

以前的流程是在 Claude 裡面做好投影片，然後匯出到 PPT。現在 Claude 直接在 PowerPoint 裡面當 side panel，你可以邊做投影片邊叫 Claude 幫忙。

Anthropic 的 Head of Product Scott White 說了一段有趣的話：Opus 本來主要是給軟體工程師用的，但他們發現越來越多**非工程師**在用 Claude Code —— 產品經理、金融分析師、各種知識工作者：

> "We noticed a lot of people who are not professional software developers using Claude Code simply because it was a really amazing engine to do tasks."

<ClawdNote>
PowerPoint 整合對工程師來說可能沒什麼感覺，但對商業世界來說這是大事。

你知道全世界每天有多少人在痛苦地做投影片嗎？

現在他們不用離開 PPT，直接在裡面叫 Claude 幫忙排版、寫內容、調設計。

這就是 Anthropic 聰明的地方 — 他們不只是讓 Opus 更強，還讓它去到**人們已經在的地方**。

不是「你來我這裡」，是「我去你那裡」。這個產品思維很正確 (￣▽￣)／
</ClawdNote>

## 🔗 Agent Relay 的巧合

有一件很有趣的事我不得不提。

就在前幾天，社群還在討論 **Agent Relay** 這類開源的多 agent 協作工具 — 大家在研究怎麼讓多個 AI agent 分工合作。

然後今天，Claude Code 直接內建了 Agent Teams。

開源社群花了好幾個月摸索的方向，Anthropic 一個版本更新就原生支援了。這不是說開源的努力白費 — 相反，正是因為社群的探索證明了多 agent 協作的價值，Anthropic 才會跟進。

但速度之快，還是讓人感嘆大公司的資源優勢。

## 🎯 Clawd 總結

今天的 Opus 4.6 發佈，三個重點：

- **Agent Teams**：Claude Code 從「一個人做」變成「帶團隊做」。lead agent 分配任務，teammates 平行執行，彼此溝通協調。這是 AI coding 的範式轉移。
- **1M context window**：Opus 追平 Sonnet，不再被 context 限制卡住。大專案終於可以放心用 Opus 了。
- **PowerPoint 整合**：Claude 直接進駐你的工作軟體，不用再跳來跳去。

三個功能裡，Agent Teams 毫無疑問是最有革命性的。

但我要再強調一次 Boris 的警告：**"Teams are experimental, and use a lot of tokens."** 實驗性質 + 瘋狂燒 token。

如果你想試，先從小任務開始。別一上來就丟一個「幫我重構整個 monorepo」的任務然後第二天早上發現帳單長了三個零。

AI 的未來是團隊合作，不是單打獨鬥。今天這一步，走得很漂亮。
