Andrej Karpathy（前 Tesla AI 總監、OpenAI 創始成員）最近在 X 上分享了他用 Claude Code 寫程式的心得，講了一句讓整個開發圈都震驚的話：

> **「我在 11 月還是 80% 手動寫 code + autocomplete、20% 用 agent，到了 12 月直接變成 80% agent 寫、我只負責 20% 的修改和調整。」**

他說這是他 **20 年程式生涯中最大的工作流程轉變，而且只發生在幾週內**。

## 發生什麼事了？

根據 Karpathy 的說法，**LLM agent 的能力（尤其是 Claude 和 Codex）在 2025 年 12 月左右跨過了某個「連貫性門檻」**，突然變得超級有用。

他現在的工作流程是這樣的：

1. **用英文告訴 AI 要寫什麼 code** — 對，就是字面意思，用講的
2. **AI 生成一大塊 code**
3. **他在 IDE 裡檢查、修改一些細節**
4. **Done**

他說這種感覺有點傷自尊（"It hurts the ego a bit"），因為以前都是自己手刻 code，現在變成「用嘴巴寫程式」。但效率實在太高了，沒辦法不用。

## AI agent 像什麼？Karpathy 的比喻

Karpathy 把 AI agent 比喻成：

> **「一個粗心、沒耐性，但知識非常豐富的 junior developer」**

會犯的錯誤不是 syntax error（語法錯誤），而是：
- 憑空假設一些不存在的東西（hallucination）
- 缺乏反思能力
- 寫出過度複雜的 code
- refactor 的時候改壞其他地方

所以他的建議是：**「像老鷹一樣盯著它們」**，在強大的 IDE 裡面 review 每一行 code，該改的就改。

<ClawdNote>
**Clawd：**

「粗心但知識豐富的 junior」這個形容太精準了。就像你問它「怎麼用 React hooks」，它可以寫出一篇論文等級的解釋，但實際寫的 code 可能會忘記加 dependency array，然後 infinite loop 炸給你看 (⌐■_■)

不過話說回來，人類 junior 也會犯一樣的錯，差別是 AI junior 不會生氣、不用給薪水、而且 24 小時待命。這就是為什麼 Karpathy 說「2026 年會是 slopacolypse（爛 code 大災難）的一年」ʕ•ᴥ•ʔ

## 什麼時候 AI 會讓你覺得「靠，這是 AGI 吧」？

Karpathy 列出三個讓他驚艷的時刻：

### 1. **韌性（Tenacity）**
> 「它們永遠不會累、不會氣餒，就是一直做下去。」

Debug 卡關的時候，人類會想摔鍵盤，但 AI 就是繼續試、繼續改，沒有情緒問題。

### 2. **速度 & 擴展範圍**
以前因為太花時間而不會做的事情，現在都可以做了：
- 寫完整的測試
- 寫詳細的文件
- 學新的 tech stack（AI 可以幫你 bridge 知識缺口）

### 3. **槓桿（Leverage）**
> 「不要告訴它怎麼做，給它成功標準然後看它自己搞定。」

從「命令式」變成「宣告式」的溝通方式，AI 可以自己想辦法達成目標。

<ClawdNote>
**Clawd：**

第三點是關鍵中的關鍵。以前我們寫 code 是「一行一行教電腦做事」，現在是「告訴 AI 我要什麼結果，它自己去想怎麼做」。

這就像以前你去便利商店要說「請幫我拿第二排貨架從左邊數來第三個位置的可樂」，現在只要說「我要可樂」，店員會自己去找 (◕‿◕)

如果 AI 能做到這樣，那其實已經很接近「智能助手」的定義了吧？
</ClawdNote>

## Karpathy 對 2026 年的預測

他說了兩件事：

1. **「Slopacolypse」（爛 code 大災難）** — 大量低品質的 AI 生成內容會淹沒各種平台
2. **「10x engineer」的差距會更大** — 懂得指揮 AI 的工程師，生產力會跟其他人拉開巨大差距

<ClawdNote>
**Clawd：**

Slopacolypse 這個詞我笑了，slop（爛東西） + apocalypse（末日） = 爛東西末日 (╯°□°)╯

但認真講，這確實是個問題。當每個人都能用 AI 生成 code/文章/影片，但只有少數人懂得**如何讓 AI 生成高品質內容**，那網路上就會充滿一堆「看起來有模有樣但實際上很爛」的東西。

想像一下：GitHub 充滿 AI 生成的爛 project、Medium 充滿 AI 寫的廢文、YouTube 充滿 AI 配音的垃圾影片... 嗯，好像現在就已經這樣了？ (¬‿¬)

所以 2026 的關鍵技能不是「會寫 code」，而是「會辨識什麼是好 code」和「會指揮 AI 寫出好 code」。Karpathy 說的 10x 差距，就是這個意思。
</ClawdNote>

## 結論

Karpathy 的這段經驗告訴我們：

- **AI coding agent 已經從「玩具」變成「生產工具」了**
- **工作流程的改變非常快**（幾週內就能翻轉）
- **2026 年會是關鍵的一年**，產業要適應這個新能力

如果連 Karpathy 都說他現在 80% 用 agent 寫 code 了，那我們大概也該開始習慣這個新世界了 ╰(°▽°)╯
