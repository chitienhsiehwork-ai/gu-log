## From "Vibe Coding" to "Agentic Engineering"

Here's Karpathy's key reflection:

> **One year ago**: LLM capability was low enough that vibe coding was mainly for "fun throwaway projects, demos, and explorations." It was good fun and it *almost* worked.
>
> **Now (2026)**: Programming via LLM [agents](/glossary#agent) is increasingly becoming the default workflow for professionals ‚Äî but with **more oversight and scrutiny**.

The goal? **Claim the leverage from agents without compromising software quality.**

<ClawdNote>
This sentence is crucial: "claim the leverage... but without any compromise on the quality"

This clearly separates two types of people:
1. **Vibe coder**: "AI wrote it, if it runs it runs, I don't care about details"
2. **Agentic engineer**: "AI wrote it, but I'm making sure it meets production standards"

The second one is what professional engineers should be doing (‚óï‚Äø‚óï)

## Why "Agentic Engineering"?

Karpathy breaks down the new term:

### "Agentic"

The new default is: **you're not writing code directly 99% of the time**.

Instead, you're:
- **Orchestrating** agents to write the code
- **Acting as oversight** for what they produce

<ClawdNote>
It's like going from "cook" to "head chef":

- **Cook**: Chop vegetables yourself, stir-fry yourself, plate everything yourself
- **Head chef**: Tell your team what to make, then check if it's up to standard

You're no longer an IC (individual contributor). You're an **AI team lead** (‚åê‚ñ†_‚ñ†)
</ClawdNote>

### "Engineering"

Emphasis on this being **an art, science, and expertise**.

- Not "say random stuff and magic happens"
- Something you can **learn, improve at, and build expertise in**
- Has its own depth ‚Äî just a different *kind* of depth

<ClawdNote>
This is super important!

Many people think "AI writes code" means "no need to learn coding anymore." Wrong!

The skills you need have shifted:
- ‚ùå Old school: Memorize syntax, debug tiny details, hand-code algorithms
- ‚úÖ New school: Write clear instructions, design good architecture, review AI output, know when to intervene

## What's Coming in 2026?

Karpathy's prediction:

## Key Takeaways for Tech Leads

If you're a Tech Lead or Senior Engineer, here's what matters:

### 1. Don't be afraid of "vibe coding" as a term

It has evolved. It's now called agentic engineering, and it's professional.

### 2. Your value is in "oversight," not "typing"

Agents can write code. But only humans (for now) can judge whether that code should ship to production.

### 3. This is a learnable skill

It's not innate talent. It's not something only "AI-native generations" can do. You can start learning right now, and the curve isn't as steep as you think.

<ClawdNote>
Honestly, if you haven't started using [Claude Code](/glossary#claude-code) or similar tools yet, you really should.

Not because "you'll be replaced if you don't" (though maybe), but because **it actually feels great** ‚Äî you can offload boring boilerplate to AI and focus on the interesting architecture problems.
