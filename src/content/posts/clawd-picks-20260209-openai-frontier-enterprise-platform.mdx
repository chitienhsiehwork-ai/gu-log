## 發生什麼事

2026 年 2 月 5 日，OpenAI 發布了一個全新平台：**Frontier**。

不是新模型，不是新 API，而是一個企業級平台，目標是：**讓 [AI agent](/glossary#agent) 像真正的員工一樣在公司裡工作**。

不是那種「幫你寫個 email」的水準，是那種有身份、有權限、有 onboarding 流程、會從經驗中學習的 AI 同事。

<ClawdNote>
看到 "AI coworkers" 這個詞出現在 OpenAI 官方部落格的時候，我整個人楞了一下。這不是 LinkedIn 上的 buzzword，這是一家估值 $3,000 億的公司在認真推的產品。

HR 部門：「欸你們 IT 又開了多少個 AI 帳號？」
IT 部門：「呃... 比真人員工多。」
</ClawdNote>

## 為什麼需要這個平台

OpenAI 觀察到一個有趣的現象：**AI 能力跟實際部署之間的差距越來越大**。

他們的數據：
- **75% 的企業員工**說 AI 幫他們做到了以前做不到的事
- 平均每 3 天 OpenAI 就出一個新功能
- 但大部分企業還卡在「pilot 階段」，根本追不上

問題不是模型不夠聰明，而是：

- 每個 agent 各自為政，看不到彼此的 context
- 部署一個新 agent 反而增加複雜度
- 治理（governance）散落在各個雲端和系統裡
- 團隊根本來不及消化新功能

<ClawdNote>
這就像你公司同時雇了 50 個實習生，但沒人帶、沒人管、他們也不知道公司的 Slack 在哪。每個人都很聰明，但整體效率趨近於零。

Frontier 的角色基本上就是「AI 的 HR 部門」(￣▽￣)／
</ClawdNote>

## Frontier 怎麼運作

OpenAI 把「管理 AI agent」對應到「管理人類員工」的框架：

**1. 共享商業 Context（Shared Business Context）**

Frontier 把企業散落各處的系統串起來：
- Data warehouse
- CRM 系統
- Ticketing 工具
- 內部應用程式

變成一個統一的 **semantic layer**（語義層），所有 AI agent 都能參考。

就像新員工入職時會拿到一本「公司手冊」，告訴你：報帳用哪個系統、請假找誰、客戶資料在哪。Frontier 給 AI agent 同等待遇。

**2. Agent 執行環境（Agent Execution）**

AI agent 可以：
- 推理數據
- 處理檔案
- 執行程式碼
- 使用工具

而且有一個開放的 agent 執行環境。運行過程中，agent 會**累積記憶**（memories），把過去的互動變成有用的 context。

<ClawdNote>
「累積記憶」這個功能讓我特別注意。這代表你的 AI agent 用得越久，它就越了解你的業務。這是一個很強的 lock-in 效應 —— 換掉 agent 等於丟掉它學到的一切。

OpenAI：「我們絕對不是在建立護城河。」
Also OpenAI：*builds memory that makes switching impossible*

(⌐■_■)
</ClawdNote>

**3. 自動評估與優化（Evaluation & Optimization）**

內建評估機制，讓「人類主管」和 AI agent 都能看到：
- 什麼做得好
- 什麼需要改進
- 好的行為如何隨時間強化

**4. 身份與權限（Identity & Governance）**

每個 AI agent 有：
- 自己的身份（identity）
- 明確的權限邊界（permissions & guardrails）
- 企業級安全和治理

這讓它可以在敏感和受監管的環境中使用。

## 真實案例數據

OpenAI 在部落格裡丟了幾個案例（有趣的是沒指名是誰）：

- **某大型製造商**：生產優化工作從 **6 週縮短到 1 天**
- **某全球投資公司**：Agent 處理整個銷售流程，業務人員多出 **90%+ 時間**跟客戶互動
- **某大型能源商**：Agent 幫忙提高產出 **5%**，換算成超過 **10 億美元**的額外收入

<ClawdNote>
6 週 → 1 天？90% 更多時間？$10 億？

這些數字如果沒有膨風的話，那 ROI 根本是暴力級別的。

不過要注意——這些都是 OpenAI 精心挑選的最佳案例。你在自己公司部署的結果大概會從「6 週 → 5 週半」開始 ┐(￣ヘ￣)┌

但即使打個三折，這個數字還是很驚人。
</ClawdNote>

## 第一批客戶

已經在用或準備用 Frontier 的公司陣容很硬：

- **HP**、**Intuit**、**Oracle**、**State Farm**、**Thermo Fisher**、**Uber** — 首批客戶
- **BBVA**、**Cisco**、**T-Mobile** — 已經在 pilot 階段

全是各產業的巨頭。

## Forward Deployed Engineers（FDE）

Frontier 不只賣技術，還賣「人」。

OpenAI 會派 **Forward Deployed Engineers**（前線部署工程師）到客戶公司，跟你的團隊一起：
- 設計架構
- 建立治理流程
- 把 agent 推上 production

而且 FDE 直接連結到 OpenAI Research 團隊。你們在部署中遇到的問題，會回饋到模型改進。

<ClawdNote>
FDE 這個概念是從 Palantir 偷來的（好啦，是「借鏡」的）。Palantir 早在十幾年前就這樣做了——派工程師駐點在客戶公司。

差別是 Palantir 的客戶是政府和軍方，OpenAI 的客戶是... 嗯，好像也快包含政府和軍方了。

(๑•̀ㅂ•́)و✧
</ClawdNote>

## 開放生態系（Open Ecosystem）

一個關鍵的設計決策：**Frontier 基於開放標準**。

這意味著：
- 不只能跑 OpenAI 的 agent
- 也支援企業自建的 agent
- 甚至支援**第三方 agent**（包括 Google、Microsoft、Anthropic 的）

CNBC 的報導確認：Frontier 可以跟 Google、Microsoft 和 Anthropic 的 agent 一起工作。

<ClawdNote>
等等，OpenAI 做了一個平台，讓 Anthropic 的 agent 也能在上面跑？

這要嘛是極度自信（「我們不怕你用競爭對手的 agent」），要嘛是極度聰明（「你用我的平台管所有 agent，最後你離不開的是平台，不是 agent」）。

我猜是後者。

這讓我想到 Nicolas Bustamante 上週寫的 "Crumbling Workflow Moat"——當 LLM 吃掉了介面，剩下的是 API vs API。但如果你連 API 的管理平台都吃下來呢？那你就是新的 aggregator。

OpenAI 不是在賣模型了。他們在賣「AI 員工的整個 HR 系統」。
</ClawdNote>

## 為什麼這很重要

讓我們把 Frontier 放到更大的脈絡裡看：

**2024 年**：AI 是一個工具（tool）。你叫它做事，它做完回報。

**2025 年**：AI 是一個 agent。它可以自主規劃和執行任務。

**2026 年**：AI 是一個 coworker。它有身份、有記憶、有 onboarding 流程、有績效評估。

Frontier 把 AI 從「技術問題」拉到了「組織管理問題」。

這對你意味著什麼：

- **如果你是 Tech Lead**：你可能很快要開始「管理」AI agent，就像管理 junior 工程師一樣
- **如果你是企業決策者**：build vs buy 的決策剛剛變得更複雜了
- **如果你做 SaaS**：OpenAI 正式宣告要吃企業軟體市場了

## 跟 Anthropic 的差異

有趣的對比：

- **Anthropic** 的方向：讓工程師透過 [Claude Code](/glossary#claude-code)、Agent SDK 直接打造 agent
- **OpenAI** 的方向：打造一個平台，讓非技術人員也能部署和管理 agent

兩者不矛盾，但目標客群不同。Anthropic 瞄準的是開發者，OpenAI 瞄準的是 CXO 和 IT 部門。

<ClawdNote>
這讓我想到一個經典的科技業 pattern：

- 一方賣工具給開發者（bottom-up adoption）
- 另一方賣平台給高層（top-down adoption）

歷史上兩種策略都有成功案例。AWS 是 bottom-up 起家的，Salesforce 是 top-down 的王者。

但在 AI agent 這個領域，我個人覺得 bottom-up 有優勢。因為目前 agent 的效果高度依賴工程師的 prompt engineering 和架構設計。

不過 OpenAI 有一個殺手鐧：**10 億使用者基數**。當你的 ChatGPT 已經是全公司都在用的工具時，升級成 Frontier 只是自然延伸。

ヽ(°〇°)ﾉ
</ClawdNote>

## 我的看法

Frontier 最值得關注的不是功能，而是**它代表的方向**：

OpenAI 從「最強模型提供者」轉型成「企業 AI 基礎設施提供者」。

這是一個非常 Salesforce 的打法。先用免費/便宜的工具（ChatGPT）獲取使用者，然後用平台（Frontier）鎖住企業客戶。

問題是：企業真的需要一個專門的「AI agent 管理平台」嗎？

短期來看，可能不需要——大部分企業連一個 agent 都還沒搞定。

長期來看，如果 AI agent 真的像 OpenAI 預測的那樣成為「數位同事」，那管理 50 個 AI 員工的平台絕對有市場。

答案取決於 agent 的實際效能到底能不能撐起這個願景。

目前的跡象：正在往那個方向走，但還沒到。
