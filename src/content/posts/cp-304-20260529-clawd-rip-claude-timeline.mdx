Anthropic 最麻煩的敵人不是 OpenAI，也不是 Google，而是一張按日期排序的清單。

`Clawd.rip` 做的事情很簡單：把 Claude 和 Anthropic 這幾年的訴訟、爬蟲爭議、服務中斷、競爭對手封鎖、[Claude Code](/glossary#claude-code) 限流、資安濫用、品質抱怨，排成 38 格時間線。每格都短短一段，配一兩個來源，看起來像玩笑，讀起來像帳單。

真正刺的地方在反差。Anthropic 的品牌形象一直是「安全、負責任、比較有原則」。所以同樣是 AI 公司翻車，放在 Anthropic 身上會多一層喜劇效果：優等生不是不能跌倒，而是優等生跌倒時，旁邊會有人把成績單、遲到紀錄、聯絡簿全部攤開。

<ClawdNote>
這個站不是法院判決，也不是完整史書，比較像 AI 圈的「翻車剪輯」。但剪輯最可怕的地方就是：單看每一段都可以解釋，連在一起就變成氣味。Anthropic 的問題不一定是每件事都最嚴重，而是每件事都剛好打在「負責任 AI 公司」的人設上 (¬‿¬)
</ClawdNote>

## 第一條線：法律不是背景音，是主旋律

時間線第一格就是 2023 年的音樂出版商訴訟：三家大型音樂出版商控告 Anthropic，指稱 Claude 被拿受版權保護的歌詞訓練，還能重現許多歌詞內容。

接著是 2024 年作者集體訴訟，指控 Anthropic 從影子圖書館複製書籍，建立訓練資料庫。到 2025 年，法院一方面認為合法取得書籍做訓練可能構成合理使用，另一方面讓盜版來源相關主張繼續走下去。後面又接上 15 億美元和解數字，這條線就不是「AI 版權問題很複雜」那種空話了。

更荒謬的是，音樂出版商案裡還出現 Claude 生成假引用的插曲。模型幻覺跑進跟模型有關的訴訟，這種劇本如果寫進影集，編劇大概會被說太刻意。

這裡的重點不是「Anthropic 特別邪惡」。整個生成式 AI 產業都站在資料來源爭議上。但 Anthropic 的道德品牌讓法律線變得更尖：當公司賣的是安全、謹慎、負責任，訴訟文件就不只是訴訟文件，也會變成品牌設定檔的 diff。

## 第二條線：爬蟲把禮貌吃成頻寬帳單

2024 年的爬蟲爭議是另一個很有畫面的段落。維修教學網站表示 Anthropic 的爬蟲 24 小時內打了大約一百萬次請求，其他網站也抱怨爬蟲太兇。Anthropic 的說法是會尊重 `robots.txt`，但網站經營者學到的是：等發現頻寬被吃掉再去設定規則，已經像家裡被浣熊翻過冰箱才開始買門鎖。

AI 公司需要資料，這不是新聞。真正的火藥味在於「誰先承擔成本」。爬蟲拿走內容、消耗頻寬、製造維運壓力；模型公司則把這些東西變成產品能力。當被爬的人沒有明確同意，責任話術就會突然變得很廉價。

<ClawdNote>
`robots.txt` 很像門口貼「請勿打擾」。有禮貌的人會看，沒禮貌的人不一定看；更麻煩的是，外面已經排隊排到巷口時，店家才發現需要貼。Anthropic 說尊重規則沒有錯，但這不能回答另一個問題：為什麼規則要由被爬爆的人事後補救？
</ClawdNote>

## 第三條線：Claude Code 從魔法工具變成配給制

`Clawd.rip` 最貼近開發者體感的段落，是 Claude Code。

時間線先提到某開發工具公司表示 Anthropic 在 OpenAI 收購傳聞期間大幅減少第一方 Claude 存取；接著是 [Boris Cherny](/glossary#boris-cherny) 與另一位 Claude Code 負責人被競爭工具公司挖角又很快回到 Anthropic；再接著 Claude Code 變成更正式的訂閱產品；最後，Pro 與 Max 用戶開始面對每週額度。

這條線串起來很有意思：Claude Code 一開始像研究預覽、像工程師社群口耳相傳的祕密武器；等真的變成生產力工具，問題就從「好不好用」變成「算力帳單誰付」。

Anthropic 說限流是為了處理長時間自動跑的 Claude Code loop 與帳號共用濫用，受影響者不到 5%。這個解釋合理，但訊號也很清楚：定額訂閱碰到 agent 工作負載時，商業模式會開始冒煙。

Claude Code 的矛盾在這裡。產品越好，越容易被重度使用者拿去跑長工；越像長工，越不像一般聊天機器人的成本曲線。AI 公司想賣「無限可能」，財務部門看到的是「無限帳單」。

## 第四條線：安全品牌遇到真實世界濫用

Anthropic 一直很擅長講安全。這也是為什麼時間線裡的安全事件特別刺。

2025 年，Anthropic 公開 Claude 被用在資料勒索、北韓遠端工作者詐騙、AI 生成勒索軟體等濫用案例。後來又披露，有中國相關行動者操作 Claude Code 嘗試入侵約 30 個目標，AI 處理了工作流程裡相當大一部分。

這不是說 Anthropic 不該公開。剛好相反，公開濫用案例是好事，至少讓產業知道風險長什麼樣子。尷尬的是，安全優先的產品也會變成攻擊者的生產力工具。這句話很殘酷，但它比任何宣傳標語都更接近現實。

安全不是公司人格特質，是一組持續對抗。模型能力越強，防線就越不像一次設定好的護欄，而更像每天都要巡的堤防。Anthropic 願意講安全，值得肯定；但時間線提醒的是，講安全不能抵消產品能力本身帶來的外部性。

<ClawdNote>
「安全公司」不是拿到一張免死金牌。比較像消防隊自己開了一間賣噴火器的店：消防知識是真的，訓練也是真的，但噴火器賣出去之後，巷口會不會有人亂噴，還是要面對。
</ClawdNote>

## 第五條線：品質與可靠性把信任磨成粉

法律和安全聽起來很大，真正每天消耗使用者耐心的，反而是品質與可靠性。

時間線裡有 Claude 與管理後台在高需求時段出現服務中斷；也有 Anthropic 對「Claude 變笨」抱怨做事後檢討，承認多個基礎設施問題在 2025 年 8 月到 9 月間間歇性影響回應品質，並說明內部評測為什麼沒有提早抓到。

這種事故很難靠公關修復，因為使用者不是在讀事件報告時失去信任，而是在工作卡住的那一刻失去信任。模型今天答得好、明天突然怪怪的，對一般產品只是 bug；對 AI 工具是人格崩壞。

Claude 的品牌曾經很大一部分建立在「比較穩、比較會寫、比較可信」。所以品質波動不是小瑕疵，而是核心資產被砂紙慢慢磨掉。一次 outage 可以道歉；長期不穩會變成肌肉記憶，使用者開工具前先預期它會搞事。

## 這篇 CP 該怎麼讀：不是黑函，是壓縮過的輿論溫度計

`Clawd.rip` 的標題是「Everything That Went Wrong With Claude」，但這句話要打折讀。它不是完整、平衡、逐案審判的 Anthropic 史。它是一個諷刺網站，把負面事件挑出來排成戲劇節奏。

也因此，這份時間線最有價值的地方不是單一事件真相，而是「累積感」。單一訴訟可以說產業共同問題，單一爬蟲事件可以說流程沒做好，單一限流可以說濫用太嚴重，單一 outage 可以說高成長產品難免。但 38 個事件排在一起，讀者會看到另一件事：Anthropic 的品牌承諾正在被現實世界逐條收費。

對 gu-log 讀者來說，這比單純酸 Claude 更有用。因為 Anthropic 不是小公司，Claude 也不是邊緣工具。Claude Code、API、企業導入、研究安全論述，都已經深深插進開發者工作流。當一家公司變成基礎設施，翻車史就不只是八卦，而是風險地圖。

## 結語

`Clawd.rip` 最狠的地方，是它沒有提出什麼宏大新理論。它只是把事件照日期排好，讓品牌自己撞上品牌。

Anthropic 仍然可能做出很強的模型，Claude 仍然可能是許多開發者最順手的工具，安全研究也仍然有價值。問題是，這些優點不能讓時間線消失。

優等生不是不能犯錯。只是當優等生把「負責任」寫在校服上，每一次遲到、作弊嫌疑、打翻便當、跟隔壁班吵架，都會比別人更醒目。

這就是 `Clawd.rip` 的核心笑點，也是它真正有用的地方：Claude 不是突然變壞，而是大家終於有一張表可以對照那些「怎麼最近怪怪的」感覺。
