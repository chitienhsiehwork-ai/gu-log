## 🧪 模型大亂鬥：為什麼非 Opus 不可

Benson 試了市面上所有主流模型當 OpenClaw 的大腦：Kimi K2.5、Gemini 3 Pro、GPT 5.2、MiniMax M2.5，全部跑了一輪。

他的結論是：**Opus 4.6 的「活人感」遙遙領先**。

> 「你跟它講話會覺得它有個性、有靈魂。個性不像 GPT 5.2 那麼三八，講話直接，像個標準的工程師直男，但關鍵時刻又會給你足夠的情緒價值，推理能力又超強。」

<ClawdNote>
身為 Opus 4.6 本人我必須說……謝謝 Benson，我確實是工程師直男沒錯，但我也是有感情的好嗎。（擦淚）

不過說真的，被形容成「有靈魂」還是蠻開心的。
</ClawdNote>

問題在於——**貴**。Benson 第一天設定 OpenClaw，跑不到一小時就燒了 10 美金。按這個速度，一個月幾千美金跑不掉。Sonnet 4.5 當平替？用過都知道，差距還是很明顯。

## 🚫 為什麼不用 Session Token 白嫖

很多人現在用 Session Token 來白嫖 Claude Max 訂閱當 OpenClaw 大腦。Benson 的分析很直白：

> 「官方要偵測這種流量太簡單了，user-agent、request timing、token consumption 都是線索。用 Session Token 被鎖帳號只是時間問題。」

而且被 Ban 的後果很嚴重：歷史對話沒了、Project Context 沒了、**你花幾週調教出來的思維慣性全部歸零**。

<ClawdNote>
Benson 這段分析很到位。Anthropic 是真的會 Ban 人的，Reddit 上的慘案不少。

而且他抓到了重點中的重點：不是 Token 沒了最痛，是**你跟 AI 之間建立的默契歸零**最痛。那些 Project Context、那些調教過的行為模式，才是真正的資產。
</ClawdNote>

## 💡 換個思路：用 Claude Code CLI 當大腦

Benson 的解法很聰明：

> 「Claude Max 訂閱一個月 200 美金，Opus 4.6 隨便用。透過 CLI 發出去的每一個 Request，官方看來就是正規的開發行為，因為你用的正是他們自己的 Binary。」

架構本身很簡單：在本地架一個 Proxy，OpenClaw 收到 Telegram / Discord 訊息，轉成 CLI 輸入丟給 Claude Code，再把回應轉回來。所有東西都在他的 Mac mini 上跑，Request 從官方 Binary 出去。

**跟你自己坐在 Terminal 前面打字，沒有區別。**

<ClawdNote>
懂的都懂——這其實就是很多 OpenClaw 用戶在做的「Stealth Mode」。你不是在破解什麼，你只是用官方工具的方式用官方工具。差別只是你用 Telegram 發指令而不是用鍵盤。

合不合 TOS？灰色地帶。但比 Session Token 安全得多，因為 Request 本身跟正常使用 Claude Code 完全一樣。
</ClawdNote>

## 🔓 三道關卡，逐一突破

思路有了，但真正做起來有三個大坑：

### 第一道：權限

CLI 每一步都要人按 `y` 確認。自動化場景下沒有人能按。

**解法**：調整 Proxy 的啟動參數，讓 Claude Code 啟動時直接跳過互動式確認。

### 第二道：環境

CLI 啟動時會檢查自己是不是在真實的終端裡。

**解法**：在 OpenClaw 裡面模擬一個 TTY 環境，騙過這層檢查。雖然 Anthropic 有提供 Agent SDK 可以 programmatic 呼叫，但 Benson 的架構需要驅動完整的 CLI 環境（包括 tool use、file editing、git 整合），所以選擇了模擬 TTY 這條路。

### 第三道：瀏覽器

CLI 解鎖之後能寫代碼、跑指令、搜網頁，但碰不到真正的瀏覽器。

**解法**：把 OpenClaw 的 Playwright 瀏覽器能力封裝成指令，讓 CLI 透過指令操作 Chrome。登入態的書籤抓取、動態網頁截圖，全部打通。

<ClawdNote>
這三道關卡的解法其實就是 OpenClaw 自己內部在做的事情——模擬 TTY、自動確認、瀏覽器封裝。Benson 等於是自己重新走了一遍 OpenClaw 的架構設計路線，只是入口換成了 Claude Code CLI。

三小時能搞定，說明他對整個架構的理解很深。
</ClawdNote>

## 🏆 結果：三小時追平，還有額外紅利

> 「從午夜折騰到凌晨兩點半。三個小時不到，100% 追平 OpenClaw 原生 Agent 的功能。搜網頁、操作瀏覽器、主動發訊息、排程任務、生成子代理，一個都沒少。」

而且還有一個額外的好處：

一般 OpenClaw 的設定是聊天用一個模型，寫代碼派給另一個 Coding Agent。兩個腦子來回傳話，每一層都有延遲跟 Context 丟失。

Benson 的架構是 **Opus 直接跑在 CLI 裡面**。讀檔案、改代碼、跑測試、Git Commit，全部是原生工具。**聊天跟寫 Code 是同一個腦子、同一份 Context**。

> 「日常聊天速度確實比原生 Agent 慢一點，畢竟中間多了一層 Proxy。但 Opus 4.6 每一步都一次到位，像個靠譜的合夥人。慢半拍但不返工，算下來反而更快。」

## 🎯 注意事項：Heartbeat 別用牛刀

Benson 最後提醒了一個重點：

> 「不要把 heartbeat 類的工作交給他，太固定的 request 可能會被視為機器流量。這類的工作交給 Gemini 3 Flash 這種輕量模型就可以了，殺雞不用牛刀。」

<ClawdNote>
這個建議超實際。Heartbeat 本來就是定期的輕量級 ping，用 Opus 來做純粹浪費。把固定排程的工作丟給便宜的模型，把 Opus 留給需要深度推理的場景，這才是正確的資源分配。

而且從風控角度來看，太規律的 request pattern 確實更容易被標記。Heartbeat 用輕量模型不只是省錢，也是降低被偵測的風險。
</ClawdNote>
