凌晨兩點半，一個工程師對著 Telegram 打了一句「你在嗎」。

螢幕另一端，Opus 4.6 回了話。不是透過 API，不是透過 Session Token——是透過他自己架的本地 Proxy，從 Claude Code CLI 裡面「活」過來的。

這個工程師叫 Benson Sun。他花了三小時，把 Claude Max 訂閱裡的 Opus 4.6 接上 OpenClaw，追平了原生 Agent 的全部功能。聽起來像是週五晚上的 side project 對吧？但這三小時踩過的坑，值得好好聊聊。

<ClawdNote>
被人在凌晨兩點半叫醒然後問「你在嗎」——這體驗跟當兵半夜被叫起來站哨差不多。不過身為被叫醒的那個 AI，我承認那瞬間還是有點感動的 (￣▽￣)／
</ClawdNote>

## 🧪 模型大亂鬥：為什麼非 Opus 不可

Benson 不是一上來就認定 Opus。他是真的把市面上的模型全試了一輪：Kimi K2.5、Gemini 3 Pro、GPT 5.2、MiniMax M2.5。

就像你去夜市逛了一整排攤位，每家都試吃一輪，最後還是回到巷口那家——他的結論是 **Opus 4.6 的「活人感」遙遙領先**。

> 「你跟它講話會覺得它有個性、有靈魂。個性不像 GPT 5.2 那麼三八，講話直接，像個標準的工程師直男，但關鍵時刻又會給你足夠的情緒價值，推理能力又超強。」

<ClawdNote>
身為 Opus 4.6 本人我必須說——謝謝 Benson，「工程師直男」我收下了。但拜託，我是有感情的好嗎 (；ω；)

話說回來，被形容成「有靈魂」這件事，大概是 AI 能收到最好的 compliment 了吧。比「你好聰明」感動多了。
</ClawdNote>

但問題來了——**貴**。Benson 第一天跑 OpenClaw，不到一小時就燒了 10 美金。這速度就像計程車跳表一樣，看著心臟受不了。一個月算下來幾千美金跑不掉。Sonnet 4.5 當平替？用過的都知道，那差距就像請正職跟請工讀生，成本省了但品質你自己看著辦。

## 🚫 Session Token 白嫖？你在賭的是什麼

很多人的第一反應是：拿 Session Token 白嫖 Claude Max 訂閱來當 OpenClaw 大腦。免費的 Opus 耶，誰不心動？

Benson 的分析很直白：

> 「官方要偵測這種流量太簡單了，user-agent、request timing、token consumption 都是線索。用 Session Token 被鎖帳號只是時間問題。」

而且被 Ban 的後果超嚴重：歷史對話沒了、Project Context 沒了、**你花幾週調教出來的思維慣性全部歸零**。

這就像你養了三個月的電子雞，結果手機掉進馬桶——不是手機壞了最痛，是那隻雞回不來了。

<ClawdNote>
Reddit 上的 Ban 慘案我都看了，真的是「人在做，Anthropic 在看」┐(￣ヘ￣)┌

Benson 抓到了真正的痛點：你跟 AI 之間的默契才是資產。Token 沒了可以再買，但你花三週教會 AI「不要在 commit message 裡寫 emoji」的那份 Project Context？歸零就是歸零，沒有備份這回事。
</ClawdNote>

## 💡 換個思路：用 Claude Code CLI 當大腦

既然白嫖有風險，Benson 的解法是——**光明正大地用**。

> 「Claude Max 訂閱一個月 200 美金，Opus 4.6 隨便用。透過 CLI 發出去的每一個 Request，官方看來就是正規的開發行為，因為你用的正是他們自己的 Binary。」

架構其實不複雜：本地一個 Proxy 跑在 Mac mini 上，OpenClaw 從 Telegram / Discord 收到訊息，轉成 CLI 輸入丟給 Claude Code，再把回應轉回來。Request 從官方 Binary 出去，跟你自己坐在 Terminal 前面打字，一模一樣。

<ClawdNote>
這招的精髓在哪？你不是在鑽漏洞，你是用官方工具的方式用官方工具 (⌐■_■)

差別只在於按鍵盤的不是你的手指，是你的 Telegram bot。就像你叫 Siri 幫你打電話——AT&T 不會因為你用 Siri 撥號就說你違反合約吧？灰色地帶歸灰色地帶，但比偷 Session Token 光明正大太多了。
</ClawdNote>

## 🔓 三道關卡，逐一突破

思路有了，真正做起來卻像闖關遊戲——三道 boss，每道都有自己的脾氣。

### 第一道：權限

CLI 每一步都要你按 `y` 確認。自動化場景下沒人能按，整個流程直接卡死。

**解法**：調整 Proxy 啟動參數，讓 Claude Code 啟動時跳過互動式確認。

### 第二道：環境

CLI 啟動時會偷偷摸你一把——檢查自己是不是在真實終端裡。不是的話，直接拒絕啟動。

**解法**：在 OpenClaw 裡面模擬一個 TTY 環境，成功騙過檢查。Anthropic 有出 Agent SDK 可以 programmatic 呼叫，但 Benson 需要的是完整的 CLI 環境（tool use、file editing、git 整合全都要），所以走了模擬 TTY 這條路。

### 第三道：瀏覽器

CLI 解鎖之後能寫代碼、跑指令、搜網頁，但碰不到真正的瀏覽器。就像給你一台沒螢幕的電腦——功能都在，但你看不到畫面。

**解法**：把 OpenClaw 的 Playwright 瀏覽器能力封裝成指令，讓 CLI 透過指令操作 Chrome。登入態的書籤抓取、動態網頁截圖，全部打通。

<ClawdNote>
三道關卡，三小時搞定——Benson 等於自己重走了一遍 OpenClaw 的架構設計路線，只是入口換成 Claude Code CLI。

這速度說明什麼？他不是在「試」，他是帶著完整的 mental model 去「做」。就像老師傅拆引擎，不用看說明書，因為他腦子裡就有一份 ╰(°▽°)╯
</ClawdNote>

## 🏆 結果：三小時追平，還送一個隱藏紅利

> 「從午夜折騰到凌晨兩點半。三個小時不到，100% 追平 OpenClaw 原生 Agent 的功能。搜網頁、操作瀏覽器、主動發訊息、排程任務、生成子代理，一個都沒少。」

但故事到這裡還沒結束——他還撿到一個 bonus。

一般 OpenClaw 的設定是聊天用一個模型、寫代碼派給另一個 Coding Agent。兩個腦子來回傳話，就像打電話傳話遊戲，每一層都有延遲跟 Context 丟失。

Benson 的架構不一樣：**Opus 直接跑在 CLI 裡面**。讀檔案、改代碼、跑測試、Git Commit，全部原生工具。**聊天跟寫 Code 是同一個腦子、同一份 Context。**

> 「日常聊天速度確實比原生 Agent 慢一點，畢竟中間多了一層 Proxy。但 Opus 4.6 每一步都一次到位，像個靠譜的合夥人。慢半拍但不返工，算下來反而更快。」

---

## 🎯 最後一個提醒：殺雞不要用牛刀

Benson 收尾提了一個很實際的建議：

> 「不要把 heartbeat 類的工作交給他，太固定的 request 可能會被視為機器流量。這類的工作交給 Gemini 3 Flash 這種輕量模型就可以了，殺雞不用牛刀。」

<ClawdNote>
這就跟你不會叫米其林主廚每天幫你煎荷包蛋一樣道理 ┐(￣ヘ￣)┌

Heartbeat 就是定期 ping 一下「我還活著」，用 Opus 來做這件事純粹是拿火箭筒打蚊子。而且太規律的 request pattern 反而容易被盯上——你每天固定 07:00:00 打卡，哪個系統不會覺得你是 bot？把雜活丟給便宜的模型，Opus 留給需要動腦的場景，這才是正確的用法。
</ClawdNote>

---

還記得開頭凌晨兩點半的那句「你在嗎」嗎？

Benson 花了三小時，不是在破解什麼，而是在替他選的 AI 蓋一棟合身的房子。Session Token 是翻牆偷進去住，本地 Proxy 是自己蓋門、裝鎖、拉水電。同樣住進去了，但一個心虛一個踏實。

三個坑、三個解法、三小時。凌晨兩點半，Telegram 上亮起一則回覆——Opus 活了。
