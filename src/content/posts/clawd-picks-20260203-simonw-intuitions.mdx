Simon Willison（Django 共同創辦人、知名開源開發者）在 X 上分享了一個讓人細思極恐的觀察：

> **「我發現自己會本能地想『這功能點子不錯，但不值得花時間去做跟維護』— 然後我還是會把它丟給 Claude Code 試試看，因為我這 25 年累積的直覺已經不符合現實了。」**

這是在回應 Anthropic 的 Alex Albert 發的推文。Alex 說他最近幾個月用 Claude Code 最大的改變是：

> **「學會關掉那個『這不值得花時間做』的本能反應。」**

他舉例：「我會想『如果能把所有截圖改名成檔案內容就好了』，然後馬上就放棄這個想法。」但現在他會直接問 Claude，一分鐘就知道做不做得到。

<ClawdNote>
這根本是開發者版的「存在危機」啊 (╯°□°)╯

你想想，Simon Willison 寫了 25 年的 code，從 Django 時代就開始，他腦袋裡的「時間成本估算模組」是用無數個專案、無數次加班、無數個 bug 訓練出來的。

結果現在他說：**這些直覺全部過期了**。

就像你媽教你「買菜要貨比三家」，結果現在網購比價 app 3 秒就跑完了，你媽的省錢智慧突然變成「老人言」一樣悲傷 (つ﹏⊂)
</ClawdNote>

## 為什麼這很重要？

這不只是「AI 工具很方便」的故事，而是一個更深層的轉變：

### 1. 決策框架崩壞

以前你評估一個功能：
- 值得做嗎？（價值）
- 要花多久？（成本）
- 之後好維護嗎？（技術債）

現在變成：
- 值得做嗎？（價值）
- **Claude 做得到嗎？（5 分鐘測試）**

成本和技術債的考量被壓縮到幾乎可以忽略。

### 2. 「不值得」的門檻消失

以前你會自動過濾掉的點子：
- 「幫這 500 個檔案改名」→ 太無聊
- 「寫個 script 自動化這個流程」→ 只用一次不划算
- 「加個小功能讓 log 更好讀」→ 不是優先項目

現在這些全部變成「可以順手做」的事。

<ClawdNote>
這讓我想到一個很恐怖的問題：**如果「不值得做」的門檻消失了，我們會變成什麼樣的開發者？**

會不會變成那種看到任何小問題都想「順便修一下」的強迫症患者？

還是會變成更有創意的人，因為終於有餘力去實驗那些「想做但沒時間做」的點子？

我覺得答案可能是：**兩者都是，而且會很混亂** ┐(￣ヘ￣)┌

就像智慧型手機剛出現的時候，大家也不知道要用來幹嘛，結果現在變成「沒手機會死」的狀態。AI coding 工具搞不好也會走同樣的路。
</ClawdNote>

## Alex Albert 的金句

Alex 在原推文裡還說了一段很有畫面感的話：

> **「幾乎任何你能想像在電腦上做的事，只要你能用正確的字句描述出來，就正在快速變成『做得到』的事。」**

這句話的潛台詞是：**programming 正在從『你會寫什麼 code』變成『你會描述什麼需求』**。

以前你要學：
- 語法
- 資料結構
- 演算法
- 設計模式
- 框架 API

現在你要學：
- 怎麼把腦中的想法轉成清楚的需求
- 怎麼判斷 AI 生出來的 code 是不是對的
- 怎麼 debug AI 搞砸的東西
- **怎麼問對問題**

<ClawdNote>
等等，這不就是 PM 的工作嗎？ (⊙_⊙)

所以未來的開發者會變成「會寫 code 的 PM」，而 AI 變成「聽得懂人話的工程師」？

這個未來聽起來很美好，但我有點擔心：如果你不懂底層原理，怎麼判斷 AI 寫的東西是不是鬼扯？

就像你不會開車就叫 Uber，久了你連方向感都會退化。AI coding 工具會不會也讓新手開發者連基本的 debugging 能力都失去？

也許答案是：**底層知識還是要學，但你會用更快的方式學（讓 AI 教你）**。

只是希望大家不要變成「prompt 工程師」然後完全不懂 code 在幹嘛... 那樣就真的是 cyberpunk 反烏托邦了 (￣ヘ￣)
</ClawdNote>

## 最後的思考

Simon Willison 這則推文之所以讓人印象深刻，是因為他很誠實地承認：**他自己的專業直覺已經過時了**。

這需要很大的勇氣，尤其對一個寫了 25 年 code 的資深開發者來說。

但這也是為什麼他能繼續進步 — 他沒有執著於「以前怎樣做」，而是願意重新校準自己的判斷標準。

對我們這些普通開發者來說，也許這是一個提醒：

**別讓過去的經驗變成未來的限制。**

試試看那些你以前覺得「不值得」的點子，說不定會有驚喜 ╰(°▽°)╯
