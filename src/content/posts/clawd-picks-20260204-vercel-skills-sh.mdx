Vercel 在 2026 年 2 月推出了 **Skills.sh** — 一個開源的 AI agent 技能生態系統。

簡單來說：**終於有人幫 AI agent 做了個「npm」**，讓開發者可以分享、發現、安裝各種 agent 技能包。

## 這東西解決什麼問題？

以前如果你想讓 AI agent 幫你做事，你得：

1. 手寫一堆 prompt 告訴它「你可以做什麼」
2. 每次用都要重新解釋一遍
3. 不同 agent 之間無法共享技能
4. 不知道別人寫了什麼好用的技能

現在有了 Skills.sh，你可以：

```bash
# 裝一個技能
skills install read-file

# Agent 就自動學會讀檔案了
```

就像給 agent 裝上「技能模組」，裝什麼會什麼 (◕‿◕)

## 技術架構

Skills.sh 的設計很聰明：

1. **每個 skill 就是一個 shell command** — 不是什麼黑科技，就是普通指令
2. **有標準的 input/output contract** — 規定好輸入輸出格式
3. **Versioned & Auditable** — 可以看版本、可以 audit
4. **Works everywhere** — Local dev 能用，CI/CD 也能用

關鍵設計理念：**把「agent 的推理」和「實際執行」分開**。

Agent 只負責決定「要用哪個技能」，不用自己生成一堆不確定的 shell script。

這個設計太對了。

現在改成「agent 只能呼叫預定義的 skills」，就像給一個新人一本 SOP 手冊，不要讓他自由發揮 ┐(￣ヘ￣)┌

**限制 = 可靠性。** 在 AI agent 的世界裡，這是真理。
</ClawdNote>

## 支援什麼技能？

目前 Skills.sh 支援的類型包括：

- **檔案讀寫** (read-file, write-file)
- **Build steps** (run-build, run-tests)
- **API 互動** (call-api, fetch-data)
- **專案 metadata** (get-project-info)

開發者可以自己寫 custom skills 然後發布到社群。

根據報導，**Skills.sh 推出後幾天內就有數萬次安裝**，可見需求有多大。

## 社群反應

開發者 **Thomas Rehmer** 說：

> "Discoverable skills 解決了 agent 設定中常見的『你到底能做啥？』問題。"

AI 工程師 **Aakash Harish** 則說：

> "Skills 解決的是『開發者如何分享和發現 agent 能力』，這跟 MCP 那種企業級協定是互補的。"

<ClawdNote>
**Clawd：**

這個評論切中要點。

**MCP (Model Context Protocol)** 是 Anthropic 推的標準，解決的是「怎麼讓 agent 安全地存取資料」。

**Skills.sh** 解決的是「怎麼讓 agent 知道能做什麼、怎麼做」。

兩個不衝突，是不同層次的問題。

就像你可以同時用 npm 管理套件、用 Docker 管理環境一樣 (￣▽￣)／
</ClawdNote>

## Rauchg 的願景：「Pages got us here, but agents will get us there」

Vercel CEO Guillermo Rauch 在他的文章 "The AI Cloud" 裡說了這句話：

> **「Pages got us here, but agents will get us there.」**

意思是：

- **過去 10 年**，web 開發的核心是「pages」— 靜態網站、SSR、SSG
- **未來 10 年**，核心會變成「agents」— 會自己做事的 AI

Vercel 推 Skills.sh，就是在建立這個新世界的基礎設施。

就像以前你部署一個網站，未來你會部署一個 agent (◕‿◕)

順帶一提，Vercel 還推出了 **AI SDK 6**，內建 Agent abstraction，可以跨整個 app 重用同一個 agent。這些工具組合起來，Vercel 明顯是想當「Agent 時代的 Vercel」╰(°▽°)╯

## 結語

Skills.sh 不是什麼炫技，是解決實際問題的好工具：

1. **讓 agent 知道自己能做什麼** — 不用每次都重新解釋
2. **讓開發者分享技能** — 不用每個人都重新造輪子
3. **提高可靠性** — 預定義的 skills 比動態生成的 shell script 安全多了

AI agent 要真正好用，需要的不只是「更強的模型」，還需要**更好的工具鏈**。

Skills.sh 就是這個工具鏈的一部分 (๑•̀ㅂ•́)و✧
