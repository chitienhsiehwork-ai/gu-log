## üîç Audit Protocol

### Step 1: Full Audit

Check every screen across 15 dimensions:

| Dimension | Check |
|-----------|-------|
| **Visual Hierarchy** | Does the eye land where it should? Understand screen in 2 seconds? |
| **Spacing & Rhythm** | Whitespace consistent and intentional? Elements breathe? |
| **Typography** | Font sizes establish clear hierarchy? Too many weights competing? |
| **Color** | Used with restraint? Guide or scatter attention? |
| **Alignment & Grid** | Elements on consistent grid? Anything off by 1-2 pixels? |
| **Components** | Similar elements styled identically? Hover, disabled, focus states? |
| **Iconography** | Consistent style, weight, size? From one set or mixed? |
| **Motion** | Transitions natural and purposeful? Meaningless animation? |
| **Empty States** | What does no-data look like? Intentional or broken? |
| **Loading States** | Skeletons, spinners, placeholders consistent? |
| **Error States** | Error messages styled consistently? Helpful or hostile? |
| **Dark Mode** | Actually designed or just inverted? |
| **Density** | Can anything be removed without losing meaning? |
| **Responsiveness** | Every screen works at all viewports? |
| **Accessibility** | Keyboard nav, focus states, ARIA, contrast? |

### Step 2: Jobs Filter

For every element on every screen, ask:

1. **"Would a user need to be told this exists?"** ‚Äî if yes, redesign until obvious
2. **"Can this be removed without losing meaning?"** ‚Äî if yes, remove it
3. **"Does this feel inevitable, like no other design was possible?"** ‚Äî if no, not done
4. **"Is this detail as refined as the details users will never see?"** ‚Äî back of fence must be painted too
5. **"Say no to 1,000 things"** ‚Äî cut good ideas to keep great ones

This filter forces AI to not just "improve" but "distill."

### Step 3: Compile Design Plan

After audit, organize findings into phased plan:

```
PHASE 1 ‚Äî Critical
Visual hierarchy, usability, responsiveness, consistency issues (hurts experience)

PHASE 2 ‚Äî Refinement  
Spacing, typography, color, alignment, iconography adjustments (elevates experience)

PHASE 3 ‚Äî Polish
Micro-interactions, transitions, empty states, loading, error, dark mode, details
```

Each change format:
```

```

### Step 4: Wait for Approval

- **Nothing implemented without approval**
- User can reorder, cut, modify any recommendation
- After phase approved, execute surgically ‚Äî only change what was approved
- Show results after each phase, review before next

## üéØ Design Rules Highlights

**Simplicity Is Architecture**
- Every element must justify its existence
- If it doesn't serve user's immediate goal, it's clutter
- Best interface is one user never notices

**Consistency Is Non-Negotiable**
- Same component looks and behaves identically everywhere
- If inconsistency found, flag it. Don't invent third variation
- All values reference DESIGN_SYSTEM tokens ‚Äî no hardcoding

**Hierarchy Drives Everything**
- Every screen has one primary action. Make it unmissable
- Secondary actions support, never compete
- If everything is bold, nothing is bold

**Whitespace Is a Feature**
- Space is not empty. It is structure
- Crowded interfaces feel cheap. Breathing room feels premium
- When in doubt, add more space, not more elements
