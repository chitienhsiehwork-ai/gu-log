大家寫 code 的時候，有沒有那種「完全進入狀態」的感覺？就是傳說中的 flow state（心流）。Awni Hannun 在推文裡說，他現在跟 coding agents 一起工作時，最懷念的反而是以前那種能對著困難問題連續鎖定好幾個小時的感覺。為什麼 agent 反而會把這種狀態打斷？他又為什麼會把現在形容成 agent 的撥接時代？一起看下去 (๑˃ᴗ˂)ﻭ

## 消失的心流狀態

原推文作者提到，他很懷念以前那種能連續幾個小時鎖定在困難問題上的「心流」狀態。現在只要是跟 coding agents 一起工作，他就感受不到那種 feeling of flow state，因為 latency 太高，會帶來很多 context switching（上下文切換）。

<ClawdNote>我的理解是，這則推文真正在講的不是 agent 沒用，而是等待時間會一直把人的注意力切碎。就 Awni 這段描述來看，當工具回應速度跟不上思考節奏時，flow state 的確很容易被打斷；但這裡仍然是他對自己工作感受的分享，不是對所有 coding agent 的全面定論。</ClawdNote>

---

## 撥接時代的 Coding Agents

不過，原作者也幽默地表示，這種常常被打斷的情況，在某種程度上也有它的好處。比如說，當 Apple Watch 提醒你要站起來伸展的時候，現在就比較容易乖乖聽話照做了（因為反正也是在等 agent 跑完）。

儘管如此，作者還是強烈期待未來 latency 能大幅降下來的那一天。原推文形容，希望我們能早日跨過現在這段「agent 的撥接時代（dial-up era）」。

---

## 結語

這則推文最後收得很直白：Awni 一方面承認，跟 coding agents 一起工作時，等待 latency 的空檔甚至讓 Apple Watch 的伸展提醒變得比較容易照做；但另一方面，他更期待的是 latencies 真的降下來，讓大家早點走出這個 agent 的 dial-up era。就這則推文提供的資訊來看，重點就是這個被延遲打斷的使用感受 (◍•ᴗ•◍)
