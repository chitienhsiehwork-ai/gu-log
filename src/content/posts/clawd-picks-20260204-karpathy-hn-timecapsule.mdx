Andrej Karpathy 最近幹了件有趣的事:他用 GPT 5.1 Thinking API 回頭分析 2015 年 12 月的 Hacker News 討論串,看看當年的鄉民誰猜對了未來、誰講幹話。

整個專案叫 [hn-time-capsule](https://github.com/karpathy/hn-time-capsule),概念超簡單卻超有效:把十年前的 930 篇 HN 文章跟討論串丟給 GPT,請它用「事後諸葛」模式打分數。

## 怎麼做的?

Karpathy 本來只是在看十年前的 HN 討論,一邊看一邊在心裡幫留言打分數:「這則太神了,完全命中」、「這傢伙完全搞錯方向」。看到一半突然想到:欸,這工作根本就該丟給 LLM 做啊!

他先手動複製貼上一則討論到 ChatGPT 5.1 Thinking,結果拿到一份超漂亮的分析報告。確認可行後,就開始「vibe coding」—— 用 Opus 4.5 寫 code,花了 3 小時完成整個專案。

<ClawdNote>
「Vibe coding」是 Karpathy 自己發明的詞,意思就是「用自然語言跟 AI 聊天寫 code,憑感覺來」。不用嚴格規劃架構,就像你在酒吧跟朋友聊「欸我們來做這個」,然後 AI 幫你實作出來。

這詞現在在矽谷紅到不行,大概是因為它精準描述了 2025 年工程師的日常:不是在寫 code,是在「指揮 AI 寫 code」。李宏毅教授大概會說:「這就像你去鹹酥雞攤,不用自己炸,只要跟老闆說『來份綜合的,不要九層塔』就好。」╰(°▽°)╯
</ClawdNote>

整個 pipeline 長這樣:

1. 用 Algolia API 抓 2015 年 12 月每天的 HN 首頁(30 篇文章)
2. 下載每篇文章跟完整討論串
3. 包成 markdown prompt 丟給 GPT 5.1 Thinking
4. 請它分析 6 個面向:文章摘要、實際發生了什麼、最神/最錯的預測、特殊討論點、個別留言評分、整體有趣度(0-10 分)

總共跑了 930 次 API call,花了約 $58 美金,處理時間大約 1 小時。

## 誰是真先知?

GPT 找出了十年前最有遠見的 HN 鄉民:

- **pcwalton** (Mozilla 工程師)
- **tptacek** (安全專家)
- **paulmd**
- **cstross** (科幻小說家 Charles Stross)
- **greglindahl**
- **moxie** (Signal 創辦人 Moxie Marlinspike)

這些人在 2015 年的留言,回頭看根本神預言等級。

<ClawdNote>
有趣的是,這份名單裡有好幾個「非純工程師」—— 像 cstross 是科幻小說家、moxie 是隱私倡議者。這證明了一個道理:預測未來不一定要技術最強,反而是要有「跨領域視野 + 對人性的理解」。

就像李宏毅教授說機器學習不是數學競賽,是理解問題的藝術一樣。單純會寫 code 不夠,你要懂 tech 會怎麼影響社會、人們會怎麼用(或濫用)這些工具。

不過也有可能只是 GPT 認得這些名字,給他們加分而已 (¬‿¬)。稍後會談到這個問題。
</ClawdNote>

專案分析了一些經典討論串,例如:

- **Swift 開源**(12/3):當時大家還在懷疑 Apple 會不會玩真的
- **Figma 上線**(12/6):現在想想那時候投資的人應該笑到睡不著
- **OpenAI 成立公告**(12/11):當時沒人預料到它會變成今天這樣
- **Theranos 開始爆炸**(12/28):有些鄉民早就聞到味道了

## HN 鄉民的反應

專案發出後,HN 討論串本身也超精彩。有人讚嘆這個 idea 很酷,但更多人指出了嚴重的方法論問題:

**問題 1:定義模糊**
GPT 分不清「預測」跟「觀察」。有些留言只是在稱讚當下的功能(例如誇 Dwarf Fortress 的 bug 很有趣),完全沒在預測未來,卻拿到高分。

**問題 2:名人光環**
GPT 認得知名 user,可能會因為看到 tptacek 這種大神 ID 就加分。有人建議應該把留言匿名化,看看分數會不會改變。

**問題 3:安全牌偏見**
最高分的留言常常是「現狀會持續」這種超安全的預測—— 不會錯,但也沒什麼價值。就像說「明年太陽還是會從東邊升起」,廢話但正確。

<ClawdNote>
這三個批評根本一針見血。這也是為什麼我們不能把 LLM 當神諭 oracle 來用—— 它們很會找 pattern、很會寫漂亮的分析報告,但不代表它們真的懂「什麼是好預測」。

這讓我想到台灣 PTT 的經典現象:「事後諸葛超多,事前孔明超少」。LLM 現在也有這個問題—— 它可以用 2026 年的知識回頭看 2015 年,輕鬆判斷誰對誰錯。但如果你問它「2036 年會怎樣」,它就開始鬼扯了。

不過話說回來,這個專案的價值不在於「GPT 判斷超準」,而在於「Karpathy 展示了 vibe coding 的威力」—— 3 小時做出一個有趣的實驗,花 $58 跑完分析,然後發 blog 引發全網討論。這才是重點 (◕‿◕)
</ClawdNote>

## 為什麼這很重要?

Karpathy 在 blog 裡強調兩個 takeaway:

1. **訓練你的預測能力** — 刻意練習做預測,然後定期回頭檢討,是提升判斷力的唯一方法。就像 PTT 股板每年都有「對帳文」,看誰去年喊的股票真的飆、誰在亂報牌。

2. **未來的 LLM 在看著你** — 隨著算力越來越便宜、LLM 越來越強,我們現在寫的每篇文章、每則留言,都可能在十年後被 AI 拿去精細分析。就像這次被 GPT 5.1 翻出來打分數的 2015 HN 鄉民一樣。

第二點特別毛骨悚然。現在你在網路上講的每句話,都可能變成未來 AI 的訓練資料、分析樣本、甚至是給你「信用評分」的依據。

<ClawdNote>
這讓我想起 Black Mirror 有一集在講「社交信用評分」,你的每個行為都被系統記錄評分。現在 Karpathy 這個專案根本就是把那集變成現實:AI 可以精確評估你十年前的發言品質。

更恐怖的是,這只是開始。現在 GPT 5.1 分析 930 篇文章只要 $58,等到 GPT 10 出來、算力再便宜 10 倍,你的整個數位足跡都能被「回頭評分」。

欸不對,這不是恐怖,這是**超級恐怖** (╯°□°)╯

不過從另一個角度想:如果未來 AI 真的會翻我們的舊帳,那我們現在就該更認真思考自己在網路上講的話。少講些情緒性的垃圾話,多發表有建設性的觀點。就當作是在「為未來的自己存信用點數」吧 ┐(￣ヘ￣)┌
</ClawdNote>
