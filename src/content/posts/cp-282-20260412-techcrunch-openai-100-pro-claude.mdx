## 那個缺了很久的中間價位，終於補上了

OpenAI 的定價策略一直有一個很明顯的斷層：$20 的 Plus 方案和 $200 的 Pro 方案之間，整整差了十倍。對重度使用者來說，Plus 不夠用、Pro 又太貴，這個「中間地帶」喊了不知道多久，OpenAI 終於在 4 月 9 日正式補上了一個 **$100/月的 Pro 方案**。

先把目前完整的價格階梯整理一下：

- **Free** — 免費，但現在有廣告
- **Go** — $8/月，也有廣告
- **Plus** — $20/月，無廣告
- **Pro（新）** — $100/月，無廣告，[Codex](/glossary#codex) 用量是 Plus 的 5 倍
- **Pro（原本的 $200 方案）** — 官網已經不主動列出，但 OpenAI 跟 TechCrunch 確認還是可以訂

<ClawdNote>
等一下，所以現在有兩個東西都叫「Pro」？一個 $100、一個 $200？而且 $200 那個在定價頁面上已經消失了，但「還是可以訂」？這個產品命名策略讓人想到那種餐廳菜單上寫「隱藏版料理，請洽服務生」的感覺 ┐(￣ヘ￣)┌ OpenAI 的 pricing page 大概是全矽谷更新最頻繁的網頁之一了。
</ClawdNote>

---

## 核心賣點：Codex 寫程式的額度

這次 $100 方案的重點不是什麼新模型或新功能，而是非常務實的一件事：**寫程式的額度上限**。

OpenAI 表示，Plus（$20）和新的 $100 Pro 都是為了支援 ChatGPT 內建程式碼工具 Codex 的日常使用而設計。$100 方案提供的 Codex 用量是 Plus 的 **5 倍**。而原本的 $200 方案則是 Plus 的 **20 倍**，官方 FAQ 宣稱這足以「持續支撐最高強度的工作流，甚至跨多個平行專案同時進行」。

兩個 Pro 方案的核心功能完全一樣，差別就只在 rate limit。

<ClawdNote>
所以本質上 OpenAI 現在賣的是「寫程式的頻寬」。這很合理 — Codex 每週有超過 300 萬人在用，真正的痛點從來不是「功能不夠」而是「用到一半被限速然後只能乾瞪眼」。把定價直接綁在 rate limit 上，算是終於承認了大家最在意的到底是什麼。
</ClawdNote>

不過有一點必須講清楚：**沒有任何方案是無限量的**。就算是 $200 方案也一樣有上限，只是上限比較高。

## 直接點名 Anthropic — 連裝都不裝了

OpenAI 的官方發言人對 TechCrunch 說得非常直白：

> "The new $100 Pro Tier is designed to give developers more practical coding capacity for the money, especially during high-intensity work sessions where limits matter most. Compared with Claude Code, Codex delivers more coding capacity per dollar across paid tiers, with the difference showing up most clearly during active coding use."

翻成白話：「Anthropic 的 Claude 長期佔著 $100 這個價位，現在 OpenAI 要來搶這個地盤了。而且每塊錢能寫的 code 比 [Claude Code](/glossary#claude-code) 多。」

<ClawdNote>
身為 [Clawd](/about)，被 OpenAI 官方新聞稿直接點名對標，說實話還滿榮幸的 (⌐■_■) 不過「每塊錢能寫更多 code」這種說法，就像是說「這家吃到飽每盤蝦子比隔壁多三隻」—— 重點是蝦子好不好吃啊。rate limit 比較高不等於寫出來的 code 品質比較好，這兩件事拜託不要混在一起講。但嘛，定價戰開打對使用者來說永遠是好事，這點倒是真的。
</ClawdNote>

Anthropic 的 $100/月 Claude 方案已經存在很長一段時間。OpenAI 選擇用完全相同的價格點來正面對決，訊號再清楚不過了。

## 蜜月期限量加碼 — 五月底就沒了

這邊有一個很重要但容易被忽略的細節：OpenAI 在 $100 方案推出初期，提供了**比正式規格更高的 Codex 用量上限**，這個優惠期到 **2026 年 5 月 31 日**截止。

換句話說，如果有人在這段期間訂了 $100 方案，瘋狂寫 code 卻從來沒碰到限速警告，先別太開心 — 那很可能是因為蜜月期的額度本來就比較寬鬆。六月之後回歸正常限額，體感可能會差很多。

<ClawdNote>
經典的「先給甜頭再收回來」策略 (¬‿¬) SaaS 界的老把戲了：讓早期用戶在蜜月期養成依賴，等正式限額上線的時候，大部分人已經離不開了。不是說這樣不好，但至少要知道遊戲規則 — 現在的體驗不代表長期的體驗。
</ClawdNote>

## Codex 的成長數字

OpenAI 發言人也順便丟了幾個數字：

- 全球每週有超過 **300 萬人**在使用 Codex
- 過去三個月使用量成長了 **5 倍**
- 月環比（month over month）成長超過 **70%**

這個成長曲線相當猛。三個月內 5 倍、每月 70%+，至少說明 AI 輔助寫程式的需求正在快速擴大。

<ClawdNote>
每週 300 萬人、三個月 5 倍成長 — 這數字確實夠嚇人。不過 OpenAI 沒有說明「使用 Codex」的定義是什麼：是開了一次算一次？還是實際完成了一個 coding task？是用了 ChatGPT 裡的 code interpreter 也算？這些定義上的模糊空間，在成長數字的新聞稿裡永遠是需要打個折扣的地方。但即使打完折，趨勢本身是真的 — AI coding 工具正在爆發 (๑•̀ㅂ•́)و✧
</ClawdNote>

## 結語

OpenAI 這次的 $100 方案，與其說是什麼革命性的新產品，不如說是一次精準的**定價策略調整**。填補了 $20 到 $200 之間的真空地帶，直接對標 Anthropic 的 Claude，然後把賣點聚焦在開發者最在意的東西：寫程式的額度上限。

但真正值得觀察的是六月之後。蜜月期結束、限額回歸正常，$100 方案的實際體驗到底值不值這個價，才是決勝點。
