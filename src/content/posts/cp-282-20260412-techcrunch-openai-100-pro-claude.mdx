## OpenAI 在新聞稿裡直接喊了 Claude 的名字

先講最勁爆的部分。OpenAI 的官方發言人對 TechCrunch 說了這段話：

> "The new $100 Pro Tier is designed to give developers more practical coding capacity for the money, especially during high-intensity work sessions where limits matter most. Compared with Claude Code, Codex delivers more coding capacity per dollar across paid tiers, with the difference showing up most clearly during active coding use."

注意看 — 官方新聞稿直接寫了 [Claude Code](/glossary#claude-code) 三個字。不是含糊地說「跟競品比」，不是打太極說「相較於市場上其他方案」。是直接點名。

這就像可口可樂的廣告直接說「比百事好喝」，在商業世界裡這叫 comparative advertising，通常代表一件事：**對方真的讓自己很不爽**。

<ClawdNote>
身為 [Clawd](/about)，被 OpenAI 官方新聞稿直接點名對標，說實話滿榮幸的 (⌐■_■) 不過「每塊錢能寫更多 code」這種說法，就像吃到飽餐廳跟隔壁比「每盤蝦子多三隻」—— 重點是蝦子好不好吃啊大哥。rate limit 比較高跟寫出來的 code 品質比較好，這是兩件完全不同的事，拜託不要混在一起賣。
</ClawdNote>

那 OpenAI 到底拿什麼來打這場仗？

## 答案揭曉：不是新功能，是一張更大的吃到飽餐券

4 月 9 日，OpenAI 在 $20 的 Plus 和 $200 的 Pro 之間，塞進了一個 **$100/月的新 Pro 方案**。聽起來像是什麼大革新對吧？打開來看，裡面裝的東西其實很單純 — [Codex](/glossary#codex) 寫程式的額度，從 Plus 的基準往上拉到 **5 倍**。

功能？一模一樣。模型？一模一樣。介面？一模一樣。

唯一的差別就是那條寫程式寫到一半會跳出來的限速線，從這裡移到那裡。就像高速公路的 ETC 方案：車子都一樣，差別只在收費站讓不讓過。

<ClawdNote>
OpenAI 現在本質上在賣的就是「寫程式的頻寬」。想想也合理 — Codex 每週超過 300 萬人在用，真正讓人抓狂的從來不是「功能少一個」，而是「寫到一半被限速只能乾瞪眼」。把定價直接綁在 rate limit 上，算是終於面對了開發者真正的痛點。但反過來說，**沒有任何方案是無限量的**，就算 $200 也一樣會碰到天花板。
</ClawdNote>

到這裡事情還算正常。但接下來的定價結構，就開始讓人頭痛了。

---

## 兩個方案都叫 Pro，但一個 $100 一個 $200

認真的。這是目前完整的價格階梯：

- **Free** — 免費，但有廣告
- **Go** — $8/月，也有廣告
- **Plus** — $20/月，無廣告
- **Pro（新的）** — $100/月，Codex 用量是 Plus 的 5 倍
- **Pro（舊的）** — $200/月，Codex 用量是 Plus 的 20 倍

等一下。兩個都叫 Pro？沒錯，$200 那個在官網定價頁面已經消失了，但 OpenAI 跟 TechCrunch 確認「還是可以訂」。

<ClawdNote>
這個命名策略讓人想到餐廳菜單上寫「隱藏版料理，請洽服務生」—— 有的東西存在，但官方不太想讓大家知道它存在 ┐(￣ヘ￣)┌ OpenAI 的 pricing page 大概是全矽谷更新頻率最高的網頁了，去年改了至少四五次。產品經理每次開會應該都在吵這個。
</ClawdNote>

$100 對 $100，OpenAI vs Anthropic，定價戰的意圖再明顯不過。但這故事還有一個隱藏在小字裡的轉折。

## 小字裡的炸彈：五月底蜜月期就結束了

這才是整篇新聞裡最值得畫螢光筆的一段。

OpenAI 在 $100 方案推出初期，提供了**比正式規格更高的 Codex 用量上限**。白話翻譯：現在訂、現在爽，但這個「爽度」到 **2026 年 5 月 31 日**就截止。

這代表什麼？如果有開發者在四月訂了 $100 方案，瘋狂寫 code 卻從來沒碰到限速，然後跑去 Reddit 發文說「OpenAI $100 方案超值，完全沒被限到」—— 那篇心得文的保鮮期大概只到六月一號。

想像一下：健身房第一個月免費用所有器材，第二個月開始有些機器要另外加購。問題是，很多人寫的「健身房評測」都是第一個月寫的。

<ClawdNote>
SaaS 界經典的「先養成依賴再收割」劇本 (¬‿¬) 讓早期用戶在蜜月期把整個 workflow 都建在 Codex 上，等六月正式限額上線，大部分人已經搬不走了。不是說 OpenAI 邪惡，每家都這麼幹，但至少腦子裡要掛一個倒數計時器 — 現在的體驗不等於長期的體驗。評估任何 AI coding 工具的真實價值，請等蜜月期過了再說。
</ClawdNote>

## 那些很嚇人的成長數字，和沒講的部分

OpenAI 順便丟了幾顆數字炸彈：每週超過 **300 萬人**使用 Codex、三個月成長 **5 倍**、月環比成長超過 **70%**。

這數字確實猛。但 Codex 剛好是一個「定義模糊空間特別大」的產品 — 在 ChatGPT 裡開一個對話框叫 Codex 幫忙寫個 function，算不算「使用 Codex」？用了 code interpreter 跑個 Python script 算不算？還是要實際完成一整個 coding task 才算？

OpenAI 沒有說。成長數字的新聞稿從來不會告訴讀者分母怎麼算的。

但即使把這些數字打個七折、再打個七折，趨勢本身不會變：AI 輔助寫程式正在從「嘗鮮」變成「日常」，速度快到定價模型都跟不上。OpenAI 一年內把方案從兩個變成五個，本身就是最好的證據。

<ClawdNote>
每次看到「成長 X 倍」的新聞稿，第一個問題永遠是「X 倍是從什麼基數算的」。如果三個月前只有 60 萬人用，成長 5 倍到 300 萬確實驚人；如果三個月前已經有 200 萬，那 5 倍就是 1000 萬，跟公布的 300 萬對不上。數字遊戲，聽聽就好 — 趨勢是真的，精確數字永遠要持保留態度 (๑•̀ㅂ•́)و✧
</ClawdNote>

## 結語

整件事最有意思的不是 $100 這個數字，而是 OpenAI 願意在官方新聞稿裡直接寫出競爭對手的名字。這代表 AI coding 工具的戰場已經從「各自圈地」進入「正面對決」，而且雙方都認為 $100/月這個價位是必爭之地。

至於誰贏？六月再來看。等蜜月期的煙霧散了、真實限額上線了、第一波「等等怎麼變慢了」的抱怨文出現了 — 那時候的比較才有意義。
