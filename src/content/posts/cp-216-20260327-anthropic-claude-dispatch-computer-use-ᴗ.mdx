想像一下：你早上出門搭捷運，手機上丟一句「幫我把昨天那個 bug 修一下，順便跑測試」，等你到辦公室坐下來打開電腦，PR 已經開好了。

這不是科幻電影預告，是 Anthropic 上週真的發布的東西。Claude 現在可以在 Claude Cowork 和 Claude Code 裡面，真的去動你的滑鼠、點你的按鈕、開你的瀏覽器。不是隱喻，不是 API call，是它真的在你的螢幕上一格一格地操作。搭配新推出的 Dispatch 功能，你甚至可以人不在電腦前，從手機遠端指派任務給它。

目前是 research preview，僅限 Pro 和 Max 訂閱者，而且只有 macOS。

<ClawdNote>
好，所以現在的狀況是：你睡覺的時候，有一個 AI 在你的電腦上移動滑鼠。這個畫面怎麼想都有點像恐怖片開頭 ╰(°▽°)╯ 但如果它真的能穩定運作，這大概是自從 autocomplete 以來最改變日常工作流的功能了。
</ClawdNote>

## 它怎麼決定要怎麼操作？

Claude 操作你電腦的方式其實比你想的聰明——它不是一上來就抓滑鼠亂點。想像你叫一個實習生幫你做事，如果有現成的工具能直接完成，聰明的實習生會先用工具；只有工具不夠用的時候，才會手動去處理。Claude 的邏輯一模一樣：

1. **先找 connector** — 如果有直接的 integration（像 Slack、Google Calendar），Claude 優先走 API，因為又快又精準
2. **找不到才動螢幕** — 沒有對應整合的時候，Claude 才會真的去操控瀏覽器、滑鼠、鍵盤

它會捲動頁面、點東西、自己探索畫面上的內容。不過每次要碰一個新的 app，它都會先問你「欸，這個可以開嗎？」

<ClawdNote>
這個設計選擇其實很關鍵。直接 API 呼叫跟用滑鼠點的差別，就像你用快捷鍵 Ctrl+S 存檔 vs. 用滑鼠去選單裡找「檔案 → 儲存」——結果一樣，但速度和可靠度天差地別。Anthropic 很清楚 computer use 現在還是「會走路但偶爾跌倒」的階段，所以把它當 fallback 而不是預設路徑。這是工程判斷力，不是保守 (⌐■_■)
</ClawdNote>

## 安全：他們怕什麼，你該怕什麼

讓 AI 操作你的電腦，安全問題當然是重中之重。Anthropic 在這塊下了不少功夫，但也很坦白地說了哪些地方還沒搞定。

**他們做了什麼：** 系統會自動掃描模型內部的 activation 來偵測 prompt injection 攻擊——簡單說就是防止有人在網頁上埋一段惡意文字，騙 Claude 去做你沒叫它做的事。你可以隨時中斷操作，每碰一個新 app 都要你授權。

**他們自己承認了什麼：** Computer use 現在還很早期。跟 Claude 寫 code 或處理文字的能力比起來，操作螢幕這件事的成熟度差了一大截。他們的原文是 "computer use is still early"，然後直接建議你別拿它碰敏感資料。

<ClawdNote>
一家公司在發布新功能的時候主動說「這東西還很嫩，請不要碰你的銀行帳戶」——這在 tech 圈幾乎是反常的坦白。多數公司的策略是把 demo 弄得很炫然後在 FAQ 最底下用 6 號字寫免責聲明。Anthropic 把限制寫在公告正文裡，這種做法我覺得應該被當成業界標準，而不是例外 (๑•̀ㅂ•́)و✧
</ClawdNote>

## Dispatch：你的電腦，它的工作

好，這才是真正讓人興奮的部分。

Dispatch 是上週推出的功能，核心概念很簡單：你用手機跟 Claude 開一個持續的對話頻道，丟任務給它，然後放下手機去過你的生活。等你回到電腦前，工作做完了。就像你出門前跟掃地機器人說「把客廳掃一下」，回家地板已經乾淨了——只是這次掃的不是地板，是你的 codebase。

搭配 computer use 之後，Dispatch 等於長了手腳。Claude 可以在你不在的時候：

- **通勤時幫你備好 morning briefing** — 你在搖晃的公車上滑手機，它在你的 Mac 上整理今天的待辦和 email 摘要
- **在 IDE 裡幫你改 code、跑 test、開 PR** — 整個開發流程從頭到尾自己來
- **按照你排好的計畫推進專案** — 你設定每天早上自動檢查 email，每週拉一次 metrics，它就照做

<ClawdNote>
「你在捷運上看 YouTube，Claude 在你的 Mac 上幫你開 PR」——光用想的就覺得有點不真實。但冷靜想一下，真正的瓶頸不是 Dispatch 的設計，而是 computer use 本身的穩定度。如果它十次有九次成功，這個工作流會改變很多人的日常；如果十次有三次翻車，那你回家看到的就不是做好的 PR，而是螢幕上六個錯誤對話框疊在一起的慘況 ┐(￣ヘ￣)┌ 但就算現在成功率沒那麼高，方向是對的。
</ClawdNote>

## 現在能用嗎？誠實說

說到這裡你大概很想試了，但先別衝太快。目前這東西有幾個很實際的門檻。

首先，**只有 macOS**。Windows 和 Linux 的朋友，目前只能在旁邊看。其次，你得是 Pro 或 Max 訂閱者，免費仔暫時沒份。然後你的桌面 app 必須一直開著、螢幕不能睡著——畢竟它是真的在操作你的畫面，電腦關了它也沒轍，就像你把掃地機器人的充電座拔掉，它就只能在原地轉圈。

操作品質方面，Anthropic 自己也說了：複雜任務有時候要重試，螢幕操作比原生 API 慢。部分 app 因為安全考量被預設封鎖。他們選擇提早開放是因為想收集回饋——跟當初 Claude Cowork 的策略一樣，先讓人用，再根據真實場景去改。

<ClawdNote>
「桌面 app 必須保持開啟且喚醒」這條限制其實透露了一件事：目前的 computer use 是在本地跑的，不是雲端虛擬桌面。這代表你的資料不會離開你的電腦（安全加分），但也代表你不能闔上筆電出門（實用扣分）。等哪天他們搞出雲端 VM 方案，這個功能才會真正起飛——但那時候的安全問題又會變成另一個層級了 (￣▽￣)／
</ClawdNote>

## 所以這代表什麼？

回到開頭的那個畫面：你在捷運上丟一句話，回到辦公室 PR 已經開好了。

現在這個畫面大概十次能成功五六次——畢竟 research preview 嘛。但重要的不是今天的成功率，而是 Anthropic 正在把 AI 從「你問它答」推向「你交代它做」。從對話介面到實際操作，這中間的跨度比大多數人想的大得多。這次的 computer use + Dispatch，就是他們踏出去的第一步。

是第一步，所以會跌跌撞撞。但至少方向很清楚：AI 不只是你的聊天對象，它想當你的同事。一個會自己開電腦幹活、偶爾搞砸、但越來越可靠的同事 (◕‿◕)
