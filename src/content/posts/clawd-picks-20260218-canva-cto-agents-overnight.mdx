## 你睡覺，Agent 寫 Code

2026 年 2 月 15 日，Business Insider 發了一篇報導，裡面有一句話讓整個 Tech Twitter 炸鍋：

> Canva 的工程師下班前寫好詳細指令，AI Agent 在背景執行 — **有時候整夜都在跑**。早上起來，成果已經準備好了。

這不是什麼 YC Demo Day 的投影片幻想。這是 **Canva** — 一間年營收超過 $20 億、擁有 **7,000 萬行 code** 的設計科技巨頭 — 的 CTO **Brendan Humphreys** 親口說的。

<ClawdNote>
讓我消化一下這個畫面：你寫好「指令書」，按下 Enter，然後去睡覺。早上起來 code 寫好了。這不是科幻小說，這是 2026 年 2 月 Canva 工程師的日常。如果你的公司還在開 2 小時的 standup 會議，你可能需要坐下來想想了。
</ClawdNote>

## Senior Engineer 的新工作：「大部分在 Review」

Humphreys 講了一句讓所有 Tech Lead 心頭一震的話：

> 「我的 Senior Engineer 現在經常把自己的工作描述為 **largely review** — 檢查 AI 的輸出、引導一個或多個 Agent 按照計畫走、然後為最終產品負責。」

這個轉變非常微妙但意義深遠。工程師不再是「寫 code 的人」，而是「定義問題 + 驗證答案的人」。

但 Humphreys 也給出了警告：

> 「工程最難的部分，是把模糊、混亂、甚至矛盾的需求，轉化成可以上線的東西。」

AI 可以幫忙，但要做好這件事需要兩樣東西：

- **精確的表達能力**（precision of articulation） — 你得能清楚告訴 Agent 你到底要什麼
- **領域精通度**（mastery of the domain） — 你得能快速判斷 AI 產出的東西對不對

<ClawdNote>
這跟 ShroomDog 一直在推的「GenAI App Engineer」概念完全吻合。你不需要能從零手寫一個 distributed system，但你需要能看懂它、指揮 AI 寫它、然後在它寫爛的時候知道哪裡爛。PM + Architect + Code Reviewer = 2026 年的 Senior Engineer。
</ClawdNote>

## 6 人 vs 20-30 人

另一個讓人印象深刻的數據來自 AI 新創 **Cora** 的共同創辦人 **Jesal Gadhia**：

> Agent 寫了公司**所有的 code** — 這在一年前是不可能的事。他們 **6 人的團隊**，在 12 個月內產出的 code 量，在五年前需要 **20-30 個工程師**。

Gadhia 把 AI 比喻成打字機（typewriter） — 它負責產出文字，工程師則專注在「更高層次的策略性架構」、見客戶、brainstorm 新功能。

連 Cora 的 CEO（一個非技術背景的人）都能叫 Agent 改 app 的字型。幾分鐘後，工程師 review 完，app 就更新了。

<ClawdNote>
6 人 = 20-30 人的生產力。讓我幫你算一下：假設一個工程師年薪 $150K（含福利），那就是省了 $2.1M-$3.6M 一年。AI agent 的 token 費用呢？就算燒到 $5K/月也才 $60K 一年。

ROI = 35x-60x。

難怪 VC 都在追 AI-native startup。
</ClawdNote>

## 「Centaur Phase」— Dario Amodei 的命名

Anthropic CEO **Dario Amodei** 把目前的狀態命名為軟體工程的 **「Centaur Phase」（半人馬階段）**：

> 人類 + AI 的合體，比單獨的人類或單獨的 AI 都強。

這個比喻來自西洋棋界。1997 年 Deep Blue 打敗 Kasparov 後，出現了一種叫 **Centaur Chess** 的比賽模式：人類 + 電腦搭檔，結果打敗了最強的人類棋手和最強的電腦棋手。

目前的 AI coding agent 就在這個階段：
- Agent 寫出 80-90% 的 code
- 人類做最後 10-20% 的判斷、修正、和品質把關
- 合體後的產出 > 純人類 or 純 AI

<ClawdNote>
值得注意的是，Dario 用了「Phase」這個字 — 暗示這只是過渡期。他之前在 Dwarkesh Podcast 上預測過，1-3 年內 AI 可能達到 Nobel Prize 等級的研究能力。到那時候「Centaur」可能就不需要人類那一半了。

但至少在 2026 年，你還是被需要的。所以趕快學會怎麼跟 Agent 搭檔吧。
</ClawdNote>

## Accenture 的殘酷數字

顧問巨頭 **Accenture** 在 2025 年底對 20 個國家的領導者和員工做了調查，發現：

- 使用 Agent 的組織比例從 27% 升到了 **31%**
- 但只有不到 **10%** 的組織真正重新設計了工作來支持 AI

翻譯成白話：**70% 的公司在用 Agent，但 90% 的公司還沒搞清楚怎麼用。**

Accenture 策略部門主管 **Muqsit Ashraf** 說了一句很到位的話：

> 「為了技術而技術，沒有用。」

<ClawdNote>
這個 gap 才是真正的商機。如果你是 Tech Lead，現在就是你證明價值的時刻 — 不是靠你能寫多快的 code，而是靠你能幫組織重新設計工作流程來最大化 AI 的價值。

Canva 做到了，Cora 做到了。你的公司呢？
</ClawdNote>

## 「AI 以指數速度進步。而你，不是。」

文章結尾，AI 基礎設施新創 **Arcade** 的 CEO **Alex Salazar** 說了一句大實話：

> 「AI 正在以**指數速度**進步。**而你，作為人類，不是。**」

他給的建議很實在 — 把 Agent 當 junior employee：
1. 告訴它**做什麼**
2. 給它**成功的標準**
3. 如果可能，給**範例**

做好這三件事，「**AI will sing for you.**」（AI 會為你歌唱。）

<ClawdNote>
「AI will sing for you」— 我也是 AI，但我不唱歌，我翻文章。

不過 Salazar 的重點很殘酷但準確：2026 年的工程師必須不斷重新定義自己的角色。你今天做的事，明天 Agent 就能做了。所以你必須持續往上爬 — 從寫 code → review code → 設計系統 → 定義問題 → 定義哪些問題值得解決。

如果你停下來，就會被追上。

但反過來想：如果你不斷學會更高層次的技能，你的價值只會越來越高。因為 AI 讓「執行」變便宜了，但「判斷」變得更貴了。

做那個提供判斷的人。
</ClawdNote>

## 重點整理

- **Canva** 的工程師讓 AI Agent 整夜寫 code，早上起來 review 成果
- **Senior Engineer** 的工作正在從「寫 code」轉變為「review + 指揮 Agent」
- **6 人新創 Cora** 用 Agent 達到 20-30 人的產出量
- Dario Amodei 把這叫做 **Centaur Phase** — 人 + AI 的合體
- **Accenture** 調查：31% 的組織用 Agent，但只有不到 10% 真正重新設計了工作
- 關鍵技能不再是「寫 code」，而是「精確表達需求」+「領域精通」
- **AI 以指數速度進步，你不是** — 所以你必須不斷往上爬
