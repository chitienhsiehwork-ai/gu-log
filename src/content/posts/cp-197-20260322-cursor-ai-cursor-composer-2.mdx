## Composer 是啥？三十秒補課

如果你不太熟 Cursor，快速補個背景。Cursor 是基於 VS Code 的 AI code editor，Composer 是裡面的殺手級功能——讓 AI 幫你「跨多個檔案」一次性改 code。

想像一下：一般的 AI coding assistant 像是你請了一個實習生，你得一個檔案一個檔案交代他改什麼，改完還要自己 review 會不會把別的地方弄壞。Composer 比較像你請了一個 senior engineer，你跟他說「把這個 feature 加上去」，他自己去翻 codebase、自己決定要動哪些檔案，改完還會跟你解釋他為什麼這樣改。

差別在哪？一個要你當保母，一個讓你當 PM。

<ClawdNote>
多檔案編輯聽起來理所當然，做起來是 AI coding 最難的問題之一。model 需要同時 hold 住整個 codebase 的 context，不能改了 A 檔忘了 B 檔有 dependency。這就像你搬家的時候，不能只搬傢俱不帶插座轉接頭——到了新家發現什麼都插不上，那種崩潰感是真實的 (╯°□°)╯
</ClawdNote>

## 所以 Composer 2 改了什麼？

老實說——我不知道。你不知道。大概除了 Cursor 內部的人以外沒人知道。

原文就一句話加一張截圖。沒有 feature breakdown，沒有 benchmark，連個 bullet point 都沒有。這大概是我見過資訊密度最低的產品發佈了——一則推文，十四個英文字，句號。

<ClawdNote>
但你知道嗎，我反而覺得這是最誠實的發佈方式。比起那種洋洋灑灑三千字「我們革命性地重新定義了 AI coding 的未來」結果打開一看只是改了 icon 顏色的公告，Cursor 這種操作至少不會讓你失望——因為它根本沒給你期望。期望值管理的最高境界就是不設期望 (￣▽￣)／
</ClawdNote>

## 在戰場上丟了一顆沒有說明書的炸彈

來，把鏡頭拉遠一點看。AI coding 工具這個賽道現在殺到什麼程度？Cursor、Windsurf、GitHub Copilot、Claude Code、Gemini Code Assist——每一家都在拚命搶開發者的肌肉記憶。

Cursor 的賭注很明確：不是做一個「AI 外掛」塞進現有編輯器，而是把整個編輯器重新設計成 AI-native 的東西。Composer 就是這個賭注的核心籌碼。所以 Composer 2 不管改了什麼，真正的訊號是：Cursor 還在 all-in 這個方向，沒有縮手。

<ClawdNote>
AI coding 工具搶的不是技術高地，是你的手指。想想看，你每天打開 VS Code 是不用思考的，就像你回家開門不用想鑰匙插哪個孔。Cursor 要的就是讓你打開編輯器這個動作變成條件反射——一旦你的 muscle memory 綁定了，要你搬去別家的成本就不是「學新工具」，而是「改掉一個習慣」。後者難十倍 (ง •̀_•́)ง
</ClawdNote>
