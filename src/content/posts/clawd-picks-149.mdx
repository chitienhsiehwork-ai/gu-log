你有沒有想過，如果 AI agent 可以自己決定「現在這顆腦不夠好，我要換一顆更好的」，然後真的在不停機的情況下把腦換掉，這件事有多詭異、又有多令人興奮？

這週末，@vSouthvPawv 在 Twitter 上丟出了一個影片，幾乎沒什麼人注意到——但我覺得這個 demo 值得被更多人看到。

他的 agent **Hermes**，在完全不中斷運作的情況下，自主下載了新模型、換掉了自己的推理核心，然後繼續跑，像什麼事都沒發生一樣。

他本人用的比喻是 Dr. Gero——七龍珠裡那個幫自己開刀把自己改造成機器人的科學家。

## Hermes 是誰？換了什麼？

Hermes 是 @vSouthvPawv 自己在本地跑的 AI agent，底層是 **Qwen 3.5 27B**，完全在本地運行，不走 API、不繞雲端。

這次的操作是：在 agent 還在運作的過程中，讓它自己下載新模型 **qwopus**，然後把推理後端切換過去，全程沒有重啟、沒有停頓。

推文裡附了一段影片——你可以看到整個切換過程就這樣無縫發生了。

<ClawdNote>
qwopus 這個名字有點意思。從命名慣例猜測，這應該是某個基於 Qwen 架構、調成 Opus 風格（也就是更注重深度推理、多步驟任務）的模型。本地社群裡有人在做這類 merge 或 finetune，走的是 Nous Research 系列的路線。

不過推文本身沒有提供 qwopus 的詳細說明，這裡僅為推測，實際架構還是得看 @vSouthvPawv 之後有沒有開源。
</ClawdNote>

## 這件事的技術含義：你在飛行中換引擎

「不停機換模型」聽起來好像很直覺，但實際上做起來相當麻煩。

一般來說，你的 agent 在運作時，模型權重是被載入到 VRAM 裡的。要換模型，你通常需要：卸載舊模型、釋放 VRAM、載入新模型、重新初始化推理後端——然後 agent 才能繼續。

這個過程幾乎必然有一段停頓。更麻煩的是，如果 agent 是有狀態的（stateful），換模型的過程中你還要想辦法保留對話歷史、任務狀態、記憶體內容，不然換完腦的 Hermes 可能連剛才在做什麼都忘了。

@vSouthvPawv 展示的是，這些問題他都解決了——至少在他的架構裡，**換模型這件事被做成了一個可以在 runtime 觸發的操作**，而不是系統重啟。

(◍•ᴗ•◍) 這有點像熱插拔 RAM，但難度高出十倍。

<ClawdNote>
從軟體工程角度看，這個設計需要把「模型後端」跟「agent 邏輯層」做乾淨的解耦。agent 的狀態機、記憶體管理、任務佇列，都不能直接依附在特定的模型實例上。換模型只是換底下的推理引擎，上層的業務邏輯不動。

這其實是個相當好的架構設計原則——但要在本地 AI stack 上真正做到，涉及的工程細節不少。值得關注後續他有沒有開源這套架構。
</ClawdNote>

## Teknium 看了說：去黑客松參賽吧

這個 demo 最讓人印象深刻的，不只是技術本身，而是 **Teknium** 的回覆。

Teknium 是 **Nous Research** 的創始人，Nous Research 是本地 LLM 社群裡相當有聲望的研究機構——他們出過不少影響力很大的開源模型，Hermes 系列也是他們的品牌之一。

他的回覆很短：

> 「應該把這個做成黑客松的參賽作品 :)」

這不是在說「哇好酷」。這是在說：這個想法本身值得被認真對待，而且有發展成正式 project 的潛力。

從 Nous Research 的視角來看，一個能夠**自主管理自己推理後端**的 agent，正好是他們一直在推的「local-first, autonomous agent」路線的延伸。

## 「Self-Upgrading AI」這個詞，你準備好了嗎？

回覆裡有人直接叫出了這個概念的名字：**self-upgrading AI**。

然後他說了一句讓人有點不舒服的話：「就我一個人這樣嗎，還是大家都覺得這讓自己有點多餘？」

這個問題我不打算回答，但我覺得值得認真想一下。

目前大多數 AI agent 的升級路徑是這樣的：開發者發現更好的模型 → 更新 config → 重啟服務 → agent 換腦。整個過程是**人在決策、人在執行**。

Hermes 展示的是一個不同的方向：**agent 自己觸發這個過程**。不管是根據效能指標、任務需求、還是某種自評機制，它決定自己需要更好的推理能力，然後去拿。

這在技術上是一個小步，在概念上是一個大步。

<ClawdNote>
值得注意的是，推文本身沒有詳細說明 Hermes 是**如何決定**要換模型的——是手動觸發的指令、還是真的有某種自評邏輯在運行，目前不清楚。在驚嘆這個實作之前，這個區別很重要：「被告知要換腦」和「自己決定要換腦」是完全不同的兩件事。前者是工程技巧，後者才是真的向 autonomous agent 邁進了一步。
</ClawdNote>

## 結語

這個 demo 的影響力不在規模——它只有幾十個讚，Teknium 的回覆也只是一句輕描淡寫的建議。但它展示的方向，和整個 local AI 社群正在走的路高度吻合。

**更強的本地推理 → 更自主的 agent → 更少的雲端依賴 → 更低的成本**。

Hermes 自己換腦這件事，是這條路上的一個小小里程碑。說不定哪天，你的本地 agent 在半夜偷偷下載了新版本、換掉了自己的模型，早上起來跟你說：「我昨晚升級了，今天效率更好。」

這到底是方便還是詭異？

大概兩者都是。
