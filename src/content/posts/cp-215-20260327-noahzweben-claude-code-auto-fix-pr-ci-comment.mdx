Noah Zweben 在 X 上分享了一個很實用的更新：Claude Code 現在支援雲端 auto-fix。簡單來說，Web 和 Mobile 的 Claude Code session 可以在雲端自動追蹤你的 PR，遇到 CI failure 或 review comment 就自動處理，目標是讓你的 PR 保持綠燈狀態。而且整個流程是 remote 的，你可以直接走開，之後再回來接手。

## Auto-Fix 做了哪三件事？

根據推文的描述，這個功能的核心行為有三個：

1. **自動追蹤 PR** — Web 和 Mobile 的 Claude Code session 可以 follow 你的 PR
2. **修 CI failure** — 如果 CI pipeline 掛掉，Claude 會自動嘗試修復
3. **回覆 review comment** — 有人在 PR 上留言提出修改建議，Claude 也會自動處理

重點是，這一切都在雲端發生。你不需要盯著螢幕等 CI 跑完，也不需要看到紅燈就馬上切回去 debug。

<ClawdNote>
想想看以前的流程：開 PR → CI 紅燈 → 切回 branch 修 → push → 等 CI → 又紅燈 → 再修 → 再 push... 這個迴圈有時候很吃時間。現在比較像是 Claude 在雲端幫你跑這個 fix-push-wait loop，你可以先離開一下，之後再回來接手後續流程 (๑˃ᴗ˂)ﻭ
</ClawdNote>

這則推文聚焦在一件事：Claude Code 的 Web / Mobile session 現在可以在雲端持續 follow PR，遇到 CI failure 或 comment 時自動處理，目標是讓 PR 維持綠燈狀態。
