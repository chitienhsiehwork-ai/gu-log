有人終於把 Claude Code 跟 Codex 的定位講清楚了。
用遊戲職業來比喻超到位 (⌐■_■)

## Claude Code = Alpha Templar（聖殿騎士）

穿重甲、扛大盾、穩如泰山。

<Toggle title="適合的任務">
Fullstack、系統設計、API 設計、資料庫建模、狀態管理、驗證流程、Debugging、重構、測試、安全強化、Code review、文件、Microservices、部署 pipeline...

**簡單說：**正常軟體工程師會做的事，全包。
</Toggle>

## Codex = Glass Cannon Sorcerer（玻璃大砲法師）

輸出爆炸，但被摸到就死。

<Toggle title="適合的任務（開 extremely high）">
RL 策略、Reward shaping、Hyperparameter tuning（煉丹）、Loss function 設計、Gradient debugging、Tensor shape 搏鬥、Custom training loops、Attention mechanism 變體、Distributed training...

**簡單說：**ML 煉丹師的日常。實驗性、前衛、容易炸。
</Toggle>

## 結論

原文說 Codex 更適合「post-AGI」的世界——那種超前衛、搞 RL 跟 alignment 的領域。
而 Claude Code 是現在就能穩穩幫你出貨的工具。

> "Pick your task, then your player."
> 選好任務，再選角色。

<Toggle title="Clawd 碎碎念">
"Glass cannon" 是遊戲圈經典詞——玻璃大砲，傷害爆高但血量超薄。
你一發可以打掉 Boss 半條血，但 Boss 咳嗽一聲你就躺了。

這整篇的 vibe 就是把 AI coding tools 當 RPG 職業在講。
Templar 穩扎穩打推副本，Sorcerer 開大招秒 Boss 但常常先暴斃。
你是要穩定輸出還是賭一把？ ╰(°▽°)╯
</Toggle>
