## What Is Clawdbot Technically?

Everyone knows Clawdbot is a personal assistant, but what is it fundamentally?

**It's a TypeScript CLI application.**

It's not Python, not Next.js, not a web app. It's a process running on your machine that:
- Opens a Gateway Server handling all channel connections (Telegram, WhatsApp, Slack, etc.)
- Calls LLM APIs (Anthropic, OpenAI, Local, etc.)
- Executes tools locally
- Does whatever you want on your computer

## The Architecture

What happens when you message Clawd on Telegram?

### 1. Channel Adapter
Receives your message and normalizes it (extracts attachments, etc.). Different messaging apps have their own adapters.

### 2. Gateway Server
This is the heart. It coordinates sessions and handles multiple requests.
To serialize operations, Clawd uses a **lane-based command queue**.
- Each session has its own dedicated lane
- Low-risk parallel tasks can run in parallel lanes (like cron jobs)

### 3. Agent Runner
This is where AI enters the scene.
- Decides which model to use
- Dynamically assembles System Prompt (adds Tools, Skills, Memory)
- Injects session history
- Checks context window (compresses if too full)

### 4. LLM API Call
Calls the model and streams the response.

### 5. Agentic Loop
If the LLM returns a tool call, Clawd executes it locally and adds the result back to the conversation. This repeats until the LLM gives a final response or hits max turns (default ~20).

## How Clawd Remembers (Memory)

An AI without memory is like a goldfish. Clawd handles this through two systems:

1. **Session Transcripts** (JSONL): Short-term conversation logs
2. **Memory Files** (Markdown): Long-term memory, stored in `memory/*.md`

Search uses **Hybrid Search**: Vector (semantic) + Keyword.

And there's no complex "memory merging" or "monthly compression." It simply lets the agent write markdown to the `memory/` folder.

## Clawd's Claws: Computer Use

This is Clawdbot's moat: you give it a computer, let it use it.

- **Sandbox**: Commands run in Docker container by default (safe)
- **Host**: Run directly on host machine (powerful but dangerous)

### Safety
There's an `exec-approvals.json` whitelist. Safe commands (jq, grep, ls) are allowed by default. Dangerous commands (rm -rf) are blocked.

## Browser: Semantic Snapshots

Clawd's browser tool **doesn't primarily use screenshots** ‚Äî it uses **Semantic Snapshots**.

It's a text representation of the page's Accessibility Tree (ARIA):

```
- button "Sign In" [ref=1]
- textbox "Email" [ref=2]
- link "Forgot password?" [ref=4]
```

This is lighter than a 5MB screenshot (under 50KB) and saves tokens. Browsing isn't fundamentally a visual task.
