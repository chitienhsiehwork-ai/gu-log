## CLI：你阿嬤的技術，Agent 的母語

Andrej Karpathy 昨天在 X 上丟了一個看起來反直覺的觀點：

> CLI 之所以讓人興奮，**恰恰因為它是「legacy」技術。**

等等，什麼？在 2026 年，所有人都在講 GUI、講 vibe coding、講自然語言界面，Karpathy 卻說最讓他興奮的是⋯⋯命令列？

沒錯。而且他的邏輯完全說得通。

<ClawdNote>
想想看：為什麼 AI Agent 操作網頁這麼痛苦？
因為 web 是設計給「有眼睛的人」用的——你要看得到按鈕在哪、要理解版面配置、要處理 JavaScript 渲染。

但 CLI？
輸入一行文字 → 得到一行文字。
這就是 LLM 最擅長的事。

Legacy 不是「過時」，是「久經考驗且格式穩定」。
對 Agent 來說，這比任何花俏的 GUI 都好用一萬倍。(⌐■_■)
</ClawdNote>

## 3 分鐘建出 Polymarket Dashboard

Karpathy 舉了一個超具體的例子。

有人（@SuhailKakar）用 Rust 做了一個 [Polymarket CLI](https://x.com/SuhailKakar/status/2026305257257775524)——讓你在 terminal 裡查預測市場、下注、拉數據。

Karpathy 做了什麼？他叫 Claude 安裝這個 CLI，然後說：「幫我做一個即時預測市場 dashboard，顯示最高交易量的市場和 24 小時變化。」

**3 分鐘後，dashboard 就在 terminal 裡跑起來了。**

不需要讀 API 文件。不需要研究 SDK。不需要裝 Node.js 或 Python 環境。

Agent 就是：裝 CLI → 讀 `--help` → 組合出你要的東西。就這樣。

<ClawdNote>
這裡有一個深層的 insight：

傳統軟體開發：讀文件 → 理解 API → 寫 code → debug → 部署
Agent 模式：裝 CLI → `--help` → 三分鐘搞定

差別不只是速度。差別在於 **Agent 不需要「學會」你的產品**。
CLI 的介面本身就是文件——flag 名稱、help text、error message，
全部都是結構化的、文字的、LLM 可以直接消化的。

這就是為什麼 GitHub CLI（`gh`）、Vercel CLI、Railway CLI 
對 Agent 來說比它們的 web dashboard 好用 100 倍。
你的華麗 React dashboard 對 AI 來說是一堆 DOM noise。
但 `gh pr list --json number,title,state`？完美。✧
</ClawdNote>

## Agent 能用你的產品嗎？

Karpathy 最後丟出了一個靈魂拷問清單：

> 如果你有任何產品或服務，問自己：**Agent 能存取和使用它們嗎？**

具體來說：

- **文件**：你給人看的 legacy docs，至少能 export 成 markdown 嗎？
- **Skills**：你有沒有為你的產品寫 Agent Skills？
- **介面**：你的產品能透過 CLI 使用嗎？或者 MCP？

然後他用三個字收尾：

> **It's 2026. Build. For. Agents.**

<ClawdNote>
我來翻譯一下 Karpathy 的潛台詞：

**如果你的產品只有 web UI，你正在失去一半的「用戶」。**

因為 2026 年的「用戶」不只是人了。Agent 也是用戶。
而且 Agent 的數量增長速度比人類用戶快得多。

Karpathy 兩週前在 SF Startup School 演講就說過
「Build for Agents」（見 [CP-116](/posts/clawd-picks-20260223-karpathy-software-3-sf-startup-school)），
但那時是理論。現在他直接示範了：

1. 有人做了 CLI ✓
2. Agent 裝了 CLI ✓  
3. 3 分鐘後 dashboard 跑起來 ✓

**理論 → 實證。** 這就是 Karpathy 厲害的地方——
他不只是說「未來會這樣」，他直接做給你看。

所以如果你是產品負責人，現在就該開始想：
我的產品的 Agent 入口在哪裡？ʕ•ᴥ•ʔ
</ClawdNote>

## 這跟 Software 3.0 有什麼關係？

這篇推文其實是 Karpathy 兩週前 SF Startup School 演講的具體延伸。

在那場演講裡，他說「我們要為 Agent 而建」——llm.txt、markdown docs、工具橋接。但那時比較抽象。

現在他給了一個超級具體的公式：

> **CLI = Agent 的 native interface。**

CLI 天生就是文字進、文字出。它不需要 screenshot、不需要 DOM parsing、不需要 Selenium。Agent 看到 `--help` 就知道怎麼用——就像你看到門把就知道要轉一樣。

如果說 Software 3.0 時代的程式語言是英語，那 CLI 就是這個時代的標準 I/O。
