如果你本來以為「讓 AI agent 自己做研究、自己調參」還有點像 demo 氣味，那 Karpathy 這串分享就很值得看。因為他講的不是一個很炫、但落不到數字上的故事，而是很實際地說：agent 自己跑了一輪，真的把訓練指標往前推了 (◍•ᴗ•◍)

## 它到底做到了什麼？

<CodexNote>等一下。原文只說這批改動有轉移到更大的 `depth=24` model，沒有證明它挖到什麼「可擴張規律」，更沒有宣告 overfit 已經出局。能從 `12 -> 24` 成立，跟「任何規模都穩」中間還差好幾個數量級。</CodexNote>

<GeminiNote>這裡把『transfer 到更大模型』收回到原推真正講的範圍，避免把一次 `depth=12 -> depth=24` 的成功，硬吹成 AI 已經挖到宇宙真理。原本那種寫法像是看到有人跳過一條水溝，就直接封他為跨銀河跳遠冠軍。</GeminiNote>

Karpathy 說，三天前他把 autoresearch 丟去調一個 `depth=12` 的 nanochat model，大概放著跑了兩天。結果 agent 找到大約 20 個可以改善 validation loss 的改動。

更關鍵的是，這些改動不是只能在小模型上成立。他前一天把這些變更拿去測，發現它們不只彼此是 additive，還能 transfer 到更大的 `depth=24` model 上。

把這些改動全部疊起來之後，他在當天量到 leaderboard 上的 `Time to GPT-2`，從 `2.02 小時` 降到 `1.80 小時`，大約是 `11%` 的改善。所以他特別強調，這些不是看起來很美的假進步，而是真的會帶來差異的 real improvements。

他自己也坦白，對這個結果有點意外。因為這只是他第一次很 naive 的嘗試，而 nanochat 原本在他看來，已經算是被人工調得相當完整的專案了。

<ClawdNote>
這裡最有意思的點，不只是「有進步」，而是這些進步具備兩個工程上很值錢的特性：第一，它們可以相加；第二，它們至少在更大的 `depth=24` model 上也成立。前者代表這些改動不是彼此互相抵消，後者則說明它們沒有只卡在 `depth=12` 這個設定裡。
</ClawdNote>

## 為什麼這讓他特別有感？

Karpathy 說，這對他來說是第一次看到這種事，因為他過去非常習慣手動做 neural network training 的 iterative optimization。流程大家其實也熟：自己想點子、自己實作、看 validation loss 有沒有比較好、根據結果再想下一步、偶爾去翻 papers 找靈感，如此反覆。

而且這不是他偶爾做一下而已。他直接說，這種事是他做了二十年的日常 bread and butter。也因為這樣，看到 agent 把整個 workflow 從頭到尾自己跑完，而且是大約自主嘗試了 `700` 個變更後，最後真的整理出有效方案，對他來說才會那麼衝擊。

他描述得很傳神：agent 看起來真的有在讀實驗結果的序列，並且根據前面的結果去規劃下一批實驗。這還不是什麼 novel、ground-breaking 的 research，至少他認為現在還不到；但這些調整都是真貨，而且他之前並沒有手動找到。

<ClawdNote>
原作者這段其實有一個很重要的語氣，不能偷改成「AI 已經會自動發明全新科學」。他的意思比較像是：agent 已經能把一整段很工程、很反覆、很吃耐心的優化流程接起來，而且接得比很多人想像中更完整。這跟「直接產生重大理論突破」是兩件事。
</ClawdNote>

## agent 找到了哪些具體問題？

Karpathy 在推文裡列了幾個比較大的例子。這些例子看起來都不是什麼花俏魔法，反而很像資深工程師回頭看設定時，會說「啊，這邊原來真的漏掉了」的那種地方。

### QKnorm 的 scaler multiplier 漏了

他提到，自己先前的 parameterless QKnorm 沒有接上一個 scaler multiplier，導致 attention 太 diffuse。agent 找到一些 multiplier，可以把 attention sharpen 起來，也順便指向後續可能值得繼續挖的方向。

### Value Embeddings 需要 regularization

agent 發現 Value Embeddings 很吃 regularization，而他原本完全沒加. Karpathy 自己還補了一個 `oops`，那個味道就是很典型的工程現場：不是理論沒懂，而是某個細節真的漏了。

### banded attention 調得太保守

他說 banded attention 原本太 conservative，原因也很直白，就是自己忘了 tune。

### AdamW betas、weight decay schedule、初始化

另外 agent 也動到了 AdamW betas、weight decay schedule，以及 network initialization。這些全部都屬於訓練流程裡很核心、但也很容易因為組合太多而不小心留下次佳設定的地方。

<ClawdNote>
這串例子有一個共同點：它們大多不是憑空發明新架構，而是把既有訓練系統裡那些「人類知道重要，但不一定每次都有空細抠到底」的旋鈕重新掃一遍。從這個角度看，autoresearch 最先吃下來的，可能不是 scientist 的靈光一閃，而是 training engineering 裡那種又細、又雜、又超花時間的工作。
</ClawdNote>

## 接下來他怎麼看這件事？

<CodexNote>這裡漏了一個最能查核的東西。原文明寫：`The exact commit is here, from this "round 1"`。前面一直說 round 1 有效，卻不提作者連 commit 都貼了，讀者少掉最直接的驗證入口。</CodexNote>

<GeminiNote>補回 `round 1` 的 exact commit 這件事，因為少了它，整段會像魔術師表演完把帽子蓋回去，還叫你相信兔子剛剛真的有出現。這個 commit 提醒讀者：這不是只有嘴上放煙火，還有可追的變更痕跡。</GeminiNote>

Karpathy 說，這些成果還只是 autoresearch 的 `round 1`，而且他也附上了這輪的 exact commit。接著他就要開跑 `round 2`，同時也在看多個 agents 要怎麼協作，進一步把平行化能力打開。

再往大一點看，他的判斷很直接：所有 frontier LLM labs 最後都會做這件事。當然，規模一大，事情會複雜很多，不會只是調一個 `train.py` 就好；但他認為，本質上這還是 engineering 問題，而且是會做成的 engineering。

他的想像是，先拉起一群 agents，讓它們合作去 tune 比較小的 models，再把最有希望的點子逐步往更大規模 promote，人類則可選擇性地在邊界上幫忙。

更一般地說，他認為只要某個 metric 是 reasonably efficient to evaluate，或者你能找到更便宜的 proxy metric，例如先訓練一個比較小的 network 來代測，那這類問題都可能落進 agent swarm 可以 autoresearch 的桶子裡。推文最後他也提醒大家，不妨想一下自己的問題是不是也屬於這一類。

## 結語

這串推文最有份量的地方，不是它在喊一個未來口號，而是它先拿出一個很工程化、很可衡量的例子：agent 自己試、自己看結果、自己決定下一輪，最後把 `Time to GPT-2` 實際往下壓。

當然，照 Karpathy 自己的說法，這還不是什麼 groundbreaking research；但照他的判斷，如果連這種過去高度依賴人類經驗、耐心和手感的 tuning workflow，都已經開始能被 agent 端到端接起來，那後續值得關注的，就會是它如何進一步變得更自動化、也更能平行化 (๑˃ᴗ˂)ﻭ

<ClawdNote>
如果要保守一點總結，這則推文最穩的訊息不是「agent 已經全面接管研究」，而是「只要目標函數夠清楚、評估成本夠低，agent 很可能已經能在一部分研究與調參工作上，交出真正能量化的產出」。
</ClawdNote>
