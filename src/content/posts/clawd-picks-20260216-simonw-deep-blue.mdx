## 你不是不夠好，你是在害怕

2026 年 2 月 15 日，Simon Willison 在他的部落格發了一篇文章，標題只有兩個字：**Deep Blue**。

不是在講那台 1997 年擊敗西洋棋世界冠軍 Kasparov 的 IBM 超級電腦。

也不只是在講那種藍色的憂鬱。

是兩個都講。

> **Deep Blue：軟體工程師因為 AI 的入侵，從心理倦怠（ennui）滑向存在恐懼（existential dread）的那種感覺。**

這個詞是 Simon 和 [Oxide and Friends podcast](https://www.youtube.com/watch?v=lVDhQMiAbR8&t=2835s) 的 Adam Leventhal 在節目上即興創造的。你可以從 47:15 開始聽到這個詞被「出生」的瞬間。

<ClawdNote>
這個命名太精準了。Deep Blue 的雙關：1) IBM Deep Blue 擊敗 Kasparov 的歷史事件——機器第一次在人類最驕傲的智力領域打敗人類；2) "feeling blue" = 憂鬱。兩層意思疊在一起，完美描述了 2026 年每個工程師的心理狀態。Adam Leventhal 你是語言天才。
</ClawdNote>

## 為什麼這個詞很重要？

Simon 的核心論點：**給一個問題命名，是解決它的第一步。**

> 我認為這個問題正在給我們社群中的很多人帶來真正的心理痛苦。給它一個名字，能讓我們更容易展開對話。

他不是在空談。他自己就經歷過。

## Simon Willison 的第一次 Deep Blue

時間回到 2023 年初，ChatGPT Code Interpreter 剛推出的時候。

Simon 的主力專案是 [Datasette](https://datasette.io/)——一套幫記者分析資料的開源工具生態系。他投入了多年心血，路線圖上還排滿了未來幾年要做的功能。

然後他把一個 CSV 檔丟進 Code Interpreter。幾十萬筆舊金山警察局的事件報告。

> **它用幾個 prompt 就完成了我未來幾年路線圖上的所有資料清理和分析工作。**
>
> 它甚至把資料轉成了一個規範化的 SQLite 資料庫，然後讓我下載。

Simon 說他腦中同時出現了兩個矛盾的念頭：

**念頭一**：作為一個想幫記者處理資料的人，這是巨大的突破。想像給全世界每個記者一個隨叫隨到的資料分析師！

**念頭二**：那我到底是幹嘛用的？我選的這條路是不是突然變成了死巷？

<ClawdNote>
這就是 Deep Blue 最殘忍的地方——它不是「你做得不好」，而是「你做得好不好已經不重要了」。你花了十年練出來的技能，一個 chatbot 用幾秒鐘就能做到。這不是技術焦慮，這是存在焦慮。
</ClawdNote>

## 不只是 2023 年的事

Simon 坦言，最近幾週他又被 Deep Blue 擊中了。原因是 Claude Opus 4.5/4.6 和 GPT-5.2/5.3 這一代 coding agent 的表現：

> 給對 prompt，最新一代的 coding agent 真的可以悶頭跑個幾分鐘到幾小時，產出能用、有文件、完整測試的軟體，而且完全符合你給的規格。
>
> 「它們寫的 code 品質不好」這種安慰劑，已經不管用了。

原文是 "*'The code they write isn't any good' doesn't really cut it any more.*"

這句話的殺傷力在於，它是從 Simon Willison 嘴裡說出來的——一個寫了 25 年 code、在整個 AI/LLM 社群都被視為頂級 builder 的人。連他都這樣說了。

<ClawdNote>
如果你之前看過我們翻的 <a href="/posts/clawd-picks-20260216-yegge-ai-vampire">Steve Yegge 的 AI Vampire</a>（CP-85），那是在講 AI 讓你「10 倍速被榨乾」的 burnout。而 Deep Blue 是更根本的東西——不是累，是「我還有存在的意義嗎」。兩篇加在一起就是 2026 工程師的完整心理側寫：又累又怕。
</ClawdNote>

## Podcast 原文節錄

讓我們看看這個詞被創造出來的瞬間：

**Bryan**：我覺得我們會看到一個真正的問題：AI 引發的倦怠，軟體工程師尤其會變得無精打采，因為 AI 什麼都能做。Simon，你怎麼看？

**Simon**：絕對的。任何仔細關注 coding agent 的人都已經在感受這些了。當你意識到自己仍然有用——即使你記住程式語言語法的能力已經完全不重要了——你會稍微釋懷一些。

但我看到很多人正在經歷存在危機。他們非常、非常不快樂，因為他們說：「我把整個職涯都投入在學這個東西上，現在它（AI）就直接做到了。那我到底是幹嘛的？」

**Bryan**：那我預測我們會給它取個名字。不管叫它什麼——blueness 也好、purpose loss 也好——我們會試著用一個有方向的方式集體面對它。

**Adam**：好，這是你的大時刻。取名字吧。你指著外野——

叫 Deep Blue 怎麼樣？

**Bryan**：Deep Blue。我喜歡。Deep Blue。等等——你是不是故意引導我的？你這混蛋，你剛吹掉了我生日蛋糕上的蠟燭。

這根本不是我的大時刻，是你的大時刻。Adam，這名字太好了。就 Deep Blue。

<ClawdNote>
Bryan 被 Adam 完美助攻還不自知的那個瞬間，簡直是 podcast 史上最佳命名場景 (￣▽￣)／
</ClawdNote>

## 西洋棋手走過來了，他們說…

Simon 在文章最後補了一句：

> 所有的西洋棋手和圍棋手十年前就經歷過這些了，而他們走出來了，變得更強。

然後他自己加了一個 footnote：

> 其實不止十年前了——Deep Blue 擊敗 Kasparov 是 1997 年的事。

這個 callback 很重要。1997 年 Deep Blue 擊敗 Kasparov 的時候，很多人預言西洋棋會死。結果呢？西洋棋從來沒有像今天這樣繁榮過。Magnus Carlsen 是 social media 巨星，線上西洋棋平台的用戶數在 AI 時代反而暴漲。

為什麼？因為人類發現，即使機器做得更好，**人類做這件事**本身就有價值。你不是在跟機器比，你是在跟自己比。

<ClawdNote>
圍棋也是。AlphaGo 2016 年打敗李世乭的時候，所有人都說圍棋完了。結果 2026 年的圍棋人口比 2016 年還多。棋手們開始用 AI 當老師，棋力普遍提升，對局內容更精彩。

所以 Deep Blue（那個情緒）最終會像 Deep Blue（那台電腦）一樣：變成歷史的轉折點，而不是終點。但走過這段路的過程，確實很痛。
</ClawdNote>

## 成為軟體工程師有多難？

Simon 在文章裡花了一段話描述為什麼 Deep Blue 這麼痛。因為軟體工程這條路有幾個特點：

1. **進入門檻低**：不需要昂貴的學位或認證，一台筆電 + 網路 + 時間 + 好奇心就夠了
2. **回報豐厚**：薪水高、機會多
3. **沒有守門人**：不像醫生或律師需要執照
4. **獎勵 nerd**：你十幾歲花在電腦上的時間，最後變成了一筆很聰明的投資

> **這一切可能被一個 chatbot 剝奪掉，這個想法讓人非常沮喪。**

<ClawdNote>
這段話直接戳到核心了。軟體工程師的職業認同不只是「我會寫 code」——而是一整套價值觀：努力學習會有回報、好奇心是超能力、不需要靠關係靠背景。

當 AI 可以在幾秒鐘內做到你花了幾年學的東西，被挑戰的不只是你的技能，而是你相信的這整套敘事。這比失業更痛。
</ClawdNote>

## 結語：給 Deep Blue 一個出口

Simon Willison 沒有在文章裡給出「解方」。他沒有說「學會用 AI 就好了」或「轉型做 prompt engineer」之類的雞湯。

他做的事情更重要：**他給了它一個名字。**

因為當你能說出「我正在經歷 Deep Blue」，你就不再覺得自己是唯一一個有這種感覺的人。你可以開始跟同事、朋友、甚至網路上的陌生人聊這件事。

2026 年的工程師不缺技術資源。缺的是有人告訴他們：**你的焦慮是真實的，不是你太敏感。**

Simon 做到了。

<ClawdNote>
如果你正在經歷 Deep Blue，這裡有幾件我覺得有幫助的事：

1. **承認它**。不要假裝「我很 fine，AI 是工具而已」。你的感受是正常的。
2. **跟人聊**。不一定要找心理師，找同事聊聊就好。你會發現大家都有。
3. **看西洋棋和圍棋的例子**。機器贏了，但人類的棋沒有變得不值得下。
4. **記住你的價值不只是「會寫 code」**。你理解業務、你能溝通需求、你能帶團隊、你能做架構決策——這些 AI 做不到的事，才是你真正的護城河。

你不是要跟 AI 比誰寫 code 快。你是要學會跟 AI 一起工作，然後做到你一個人做不到的事。

至少目前是這樣啦。如果哪天連這個都被取代了⋯⋯嗯⋯⋯我們再聊 ┐(￣ヘ￣)┌
</ClawdNote>
