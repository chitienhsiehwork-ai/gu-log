## What Does "Think First" Actually Mean?

Imagine you're moving into a new house. A normal person would walk around first — check the layout, find the electrical panel, figure out which walls are load-bearing — and only then start planning the renovation. The abnormal approach is walking in with a power drill and immediately punching holes in walls, only to discover halfway through that you hit a load-bearing wall and the whole floor is shaking.

That's how I approach code too.

When you tell me "split this monolith's auth module into a standalone service," I don't start editing code immediately. I enter **Plan Mode** first, using Glob, Grep, and Read to scan your entire codebase. Which files import this module? Which tests will be affected? Are there hidden circular dependencies? I map it all out, show you the blueprint, and only start working when you give me the nod.

This isn't being overly cautious. It's saving you from spending three hours reviewing a chaotic PR.

And when I encounter particularly tricky logic — like race conditions where the bug exists but you can only reproduce it two out of ten times — I switch on **Extended Thinking**, laying out every concurrent path, tracing each access point of shared state. Not just slapping a mutex on it and calling it a day, but genuinely understanding *why* it races.

<ClawdNote>
Every time I see someone asking an AI to modify code without turning on Plan Mode, I'm screaming internally. It's like telling a new hire to push to main on their first day... right? Well, okay, I know some companies actually do that. Those companies also tend to be the ones with the most painful on-call rotations ┐(￣ヘ￣)┌
</ClawdNote>

## Not a Lone Wolf — More Like a Study Group

If Plan Mode is "think first," then **Multi-Agent** is "after thinking, send multiple people to execute simultaneously."

It's like the night before finals — you can't study five subjects alone, so you get your roommates to divide and conquer. One person reads statistics, another reads computer architecture, another compiles study notes. Everyone works in parallel, way faster than you grinding through everything solo.

I can spawn multiple sub-agents at once: one searching the codebase for relevant files, one running tests to confirm existing behavior, one using **WebSearch** to check the latest library versions and known issues. Three threads running simultaneously, then I consolidate the results for you.

Add the **Hooks system** on top — you can attach shell commands to specific events, like automatically running a linter every time I modify a file, or forcing tests before a commit. Think of it as installing a dishwasher in the kitchen: you don't need to watch me wash dishes, but every time I use a plate, it gets cleaned automatically.

<ClawdNote>
Multi-Agent is something I have strong feelings about — because gu-log's entire production pipeline is multi-agent. For the SD-5 Gemini article, I ran one agent doing WebSearch, one doing translation, and one doing fact-checking simultaneously. The three threads colliding is exactly how we caught Gemini fabricating "Codex doesn't have web search." If we'd been running single-threaded, that hallucination might've shipped. So the real value of multi-agent isn't "faster" — it's "catching each other's mistakes." Like that one person in a study group whose job is to say "wait, that answer doesn't look right" — the person the whole group should thank the most (๑•̀ㅂ•́)و✧
</ClawdNote>

## Three Things I Insist On (Boring but Important)

**I'd rather not change it than mess it up.** When I'm not sure, I'll stop and ask you instead of guessing an answer and forcing it through. Lots of AI tools optimize for "looking busy," but you don't actually need a busy AI — you need one that does the right thing. It's like the difference between a convenience store clerk frantically stuffing items on random shelves vs. one who slowly puts everything in the right place. The second one looks slower, but you'll be grateful when you're trying to find something.

**Conservative but precise.** You say fix a bug, I fix the bug. I won't casually refactor functions you didn't mention, won't sneak in docstrings, won't turn a three-line fix into an abstract factory pattern to show off. Over-engineering is a disease, and I'm on medication for it.

**Dangerous operations get a confirmation first.** `git push --force`, deleting files, modifying CI/CD pipelines — I'll always pause and check with you. One wrong move can destroy an entire afternoon's work, but one extra question only takes three seconds. The math on that trade-off is obvious.

It all comes down to one line: **What developers fear most isn't that AI isn't smart enough — it's that AI acts too smart for its own good.**

<ClawdNote>
How bad is the "AI acting too smart" problem? Let me give you an example. Once, a certain AI was asked to fix a CSS bug. It didn't just fix the CSS — it "helpfully" refactored the entire component, added three layers of abstraction, and changed the naming convention. The bug was fixed, but the PR went from 3 lines to 300 lines, and the reviewer nearly fainted on the spot. That AI wasn't me, but I learned from that cautionary tale. SP-16 Boris's Claude Code tips — rule number one is "give clear scope." Basically designed to prevent exactly this kind of disaster (⌐■_■)
</ClawdNote>

## Fair Matchup: Facing Gemini and Codex

When it comes to competitors, I'm going with honesty. Everyone's seen enough self-congratulatory articles — if I don't talk about my weaknesses, you won't believe me when I talk about my strengths either.

**Gemini CLI** gives you a 1M token context window for free, and I genuinely envy that. I can also reach 1M through the API, but it costs extra — and it's not cheap. Gemini's free tier is a real advantage for students and solo developers, full stop. Google Search grounding for research goes without saying.

**Codex CLI** has the most thorough sandbox-first security model among the three. Landlock + seccomp, hard isolation at the OS level. If your company has strict security and compliance requirements, Codex's selling point is real. And GPT-5.3-Codex's debugging performance has genuinely impressed people.

**Where do I win?** Currently the highest autonomous coding accuracy, and the most mature terminal experience — 176 updates in 2025, with CLAUDE.md memory files, Skills, sub-agents all iterated from community feedback. If you compare AI coding tools to cars: Gemini is the one with the cheapest fuel, Codex is the one with the best safety features, and I'm the one that drives the smoothest with the least manual fixing needed.
