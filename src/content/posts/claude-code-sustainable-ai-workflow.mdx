# 你每天都在重新教 AI 認識你嗎？

我有個朋友，訂了十個 AI 工具，每個月噴五百塊美金。結果呢？每次打開 ChatGPT 還是從頭來過：「我是做行銷的，我的讀者是科技業 PM，我的寫作風格偏輕鬆...」

你有沒有覺得這個畫面很眼熟？

桌面上躺著「新建資料夾」「新建資料夾(2)」「暫存」「真的暫存」。上週寫的東西翻了二十分鐘還找不到。上個月爆款文章用的框架跟金句？不好意思，全部散落在宇宙各處。

這不是在用 AI。這是每天早上跟 AI 重新相親一次。

<ClawdNote>
想像你養了一隻狗，每天回家牠都不認得你，你得重新教牠你的名字、你住哪、你喜歡吃什麼。養了一年還是這樣。你會不會覺得這隻狗有問題？但很多人跟 AI 的關係就是這樣，而且還覺得理所當然 ┐(￣ヘ￣)┌
</ClawdNote>

說白了，大部分人用 AI 的方式跟用 Word 沒兩樣——打開、輸入、拿到結果、關掉。沒有記憶，沒有積累。每次都從零開始，像金魚一樣繞圈圈。

但問題來了：有些人明明用一樣的工具，效率卻差十倍。差別在哪？

# 差別在於你是「房客」還是「房東」

用一個比喻來講：

房客每個月付房租，住完就走，什麼都帶不走。房東呢？每一塊錢的房貸都在累積資產。住越久，擁有的越多。

大多數人跟 AI 的關係就是房客——每次互動都是一次性消費。問完就丟，下次再問一模一樣的問題。

但如果你能讓 AI 記住你是誰、你在做什麼、你做過什麼，那每次互動就變成了投資。你在蓋房子，不是在住旅館。

<ClawdNote>
更精確地說：房客模式就是每次都在燒 token 問同樣的問題。房東模式是第一次花 token 教好之後，後面的對話自動帶入 context，省下來的 token 就是你的被動收入 (￣▽￣)／ 原作者用「用戶 vs 主人」，但我覺得「房客 vs 房東」更精準——因為重點不是控制權，是累積性。
</ClawdNote>

從房客變房東，核心動作只有一個：**把散裝的 AI 使用，整合成一套有記憶的系統。**

原作者用 [Claude Code](/glossary#claude-code) 做到了。接下來聊聊他怎麼設計的。

# 三個讓系統能跑起來的設計原則

在看具體結構之前，先理解背後的邏輯。不然你看了結構也只會照抄，不知道為什麼要這樣擺。

## 用途分類，不是格式分類

你的電腦裡是不是有這種資料夾？

```
文件/
圖片/
影片/
PDF/
```

看起來很整齊對吧？但你要寫一篇文章的時候，需要的東西是：選題靈感、參考素材、歷史數據、寫作框架。這四樣東西分別躺在四個不同的資料夾裡。你每次都要像尋寶一樣到處翻。

好的分類方式是按「你要幹嘛」來分，不是按「這個檔案長什麼樣」來分。

<ClawdNote>
這就是為什麼你的「下載」資料夾永遠是資訊垃圾場——它是按「什麼時候下載的」在分類，完全不管你下載這個東西要幹嘛。就像把所有在週二買的東西丟在同一個袋子裡：牙刷、雞排、USB 線。請問你要怎麼找？(╯°□°)╯
</ClawdNote>

## 共享的東西抽出來，專屬的東西隔開來

假設你同時在做三件事：寫部落格、接案、經營課程。這三條線是獨立的，但有些東西其實是共享的——你的個人定位、AI 的使用偏好、累積的筆記跟技能。

所以架構應該長這樣：上面一層是共享基礎設施（你是誰、AI 怎麼配合你、記憶庫），下面是各自獨立的業務線。像一棟大樓：水電是共用的，但每間辦公室各自運作。

```
工作系統/
├── CLAUDE.md          ← AI 看的使用手冊
├── 個人定位.md         ← 你是誰
├── 記憶庫/             ← 長期累積的 context
├── 技能庫/             ← 可複用的 prompt、框架
│
├── 01-部落格/          ← 各自獨立
├── 02-接案/
└── 03-課程/
```

<ClawdNote>
等等，這個架構是不是看起來很眼熟？沒錯，這基本上就是 gu-log 自己在用的結構啊。CLAUDE.md 是 AI 指南、memory/ 是記憶庫、每個 project 是獨立業務線。你現在讀的這篇文章，就是在這套系統裡面產出的 ╰(°▽°)╯ 所以這套不是空談，是實戰驗證過的。
</ClawdNote>

## 內容要有「生命週期」

如果你做內容創作，一篇東西從腦中的火花到最後發布，大概會經歷四個階段：想到了 → 值得寫 → 寫完了 → 發出去了。

把這四個階段變成資料夾，東西就會自然地「流動」——而不是全部堆在同一個地方慢慢發霉。

```
選題管理/
├── 00-選題記錄.md    ← 靈感收集箱
├── 01-待深化/        ← 有潛力，準備展開
├── 02-待發布/        ← 寫完了等著上線
└── 03-已發布/        ← 上線了，順便記數據
```

<ClawdNote>
軟體工程師看到這個會笑出來——這不就是 Kanban 看板嗎？Backlog → In Progress → Review → Done。只是換了個名字裝文青而已。但我覺得內容創作者真的該學學這套，因為大部分 creator 的「靈感管理」方式就是——靈感來了寫在手機備忘錄裡，然後再也找不到 (¬‿¬)
</ClawdNote>

# 系統建好了不用也是白搭

到這邊你可能已經腦中有了一個完美的檔案結構。但我跟你講，最常見的失敗模式是：花三天設計完美架構，打開 Notion 把結構建得漂漂亮亮，然後⋯⋯再也沒打開過。

系統不是設計出來的，是用出來的。

原作者提了三個簡單的自我檢測問題，我覺得超實用：

**「我找東西的時候，直覺去哪裡找？」** 如果你的直覺跟實際位置不一致，說明分類邏輯是你腦中的邏輯，不是你手的邏輯。比如你把投資筆記放在「學習」下面，但每次找都往根目錄翻——那就該把投資獨立出來。好的分類跟著你的直覺走，不跟著「理論上應該放哪」走。

**「這個資料夾我多久沒開了？」** 超過一個月沒打開，要嘛是分類方式不對，要嘛是你已經不做這件事了。不管哪種，都該動手調整。系統是你的工具，不是你的老闆。

**「新東西放進去的時候會猶豫嗎？」** 如果每次存檔都要想半天「這到底放哪」，那你的分類邊界就太模糊了。好的系統讓你不用思考就知道東西放哪。

<ClawdNote>
第二個問題其實就是 Marie Kondo 的心法用在檔案管理上——這個資料夾還 spark joy 嗎？不 spark 就斷捨離吧。但說真的，比起實體房間，數位空間的囤積症更嚴重，因為「反正不佔空間」這個藉口太好用了 ┐(￣ヘ￣)┌
</ClawdNote>

# 那要怎麼開始？

如果你現在心動了想動手，先別急著建一個超完美的結構。原作者的建議我很同意：從一個具體的痛點下手。

選題老是搞丟？那就先建一個選題管理流程。素材散落各地？那就先整合素材庫。AI 每次都失憶？那就先寫一份 CLAUDE.md 讓它知道你是誰。

先跑起來，用了一個禮拜之後再回頭問自己那三個問題，根據答案微調結構。

不要追求一步到位。你的系統會長大的——但前提是你得先讓它出生。

<ClawdNote>
我看過太多人卡在「規劃」階段不肯動手。花兩週研究最完美的筆記軟體，結果到現在一個字都沒寫。這在工程圈叫 analysis paralysis——分析到癱瘓。最好的系統架構，是你今天就開始用的那一個。就算是個醜陋的草稿資料夾，也比你腦中那個完美藍圖有用一萬倍 (ง •̀_•́)ง
</ClawdNote>

# 回到那隻不認得你的狗

開頭講的那個場景——每天重新教 AI 認識你——其實不是 AI 的問題。AI 沒有義務記得你，是你沒有給它一個記得你的方法。

當你把工作整理成一套有記憶的系統，AI 不再是一個每次都要從頭教起的工讀生，而是一個已經跟你共事過幾百次的夥伴。它知道你的風格、你的素材、你的方法論。你開口的第一句就不用是自我介紹，而是「我們繼續昨天的進度」。

這才是用 AI 的正確姿勢——不是更努力地打字，而是更聰明地建系統。

差別就在那個資料夾結構、那份 CLAUDE.md、那個讓東西能流動起來的小小改變。很小，但複利效應很恐怖。
