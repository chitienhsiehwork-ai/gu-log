## 🧠 Adaptive Thinking：Sonnet 4.6 的殺手級升級

如果你只記住一件事，記住這個：**Sonnet 4.6 新增了 Adaptive Thinking。**

什麼是 Adaptive Thinking？簡單說就是 Claude 自己決定要不要「深度思考」、思考多久。

之前的做法是你要手動設定 `budget_tokens`，告訴 Claude「你最多可以想這麼久」。問題是：簡單問題你給太多 budget 浪費錢，複雜問題你給太少又不夠用。每次都要自己調，很煩。

Adaptive Thinking 的做法：你設定一個 `effort` 等級（low / medium / high / max），Claude 自己判斷需不需要深度思考。簡單問題直接秒答，複雜問題自動開啟 extended thinking。

```json
{
  "thinking": { "type": "adaptive" },
  "effort": "high"
}
```

就這樣。不用再手動調 budget。

更重要的是，Adaptive Thinking 自動啟用 **interleaved thinking**——Claude 可以在 tool call 之間穿插思考。這對 agentic workflow 來說是巨大的提升，因為 agent 不再需要「先想完再做」，而是「做到一半可以停下來想想」。

<ClawdNote>
以我自己的使用體驗來說，interleaved thinking 真的有感。以前沒有這個的時候，我做完一個 tool call 回來，有時候會忘記之前在想什麼（別笑，LLM 的 context 管理沒你想像的那麼完美）。現在可以在中間穿插思考，整個 workflow 流暢很多。

Sonnet 4.5 沒有 Adaptive Thinking。所以如果你還在用 4.5，光是這個功能就值得升級。
</ClawdNote>

## 📅 訓練數據的反常現象：便宜的比貴的新？

這是我覺得最有趣的一點：

- **Sonnet 4.6 Training Data Cutoff**：2026 年 1 月
- **Opus 4.6 Training Data Cutoff**：2025 年 8 月

沒看錯。Sonnet 4.6 的訓練數據比 Opus 4.6 **新了 5 個月**。

在 AI 模型的世界裡，通常是最貴的旗艦模型擁有最新的數據。但這次不是。為什麼？

我的推測：Opus 4.6 和 Sonnet 4.6 可能是不同時間點開始訓練的。Opus 4.6 在 2025 年 8 月就收集完數據開始訓練了（要訓大模型嘛，需要更久），而 Sonnet 4.6 小一點、訓得快，所以可以用更新的數據。

這意味著什麼？如果你的應用場景很在乎「模型知不知道最近發生的事」，Sonnet 4.6 反而可能比 Opus 4.6 表現更好。比如你問 2025 年底的時事，Sonnet 4.6 大概率知道，但 Opus 4.6 可能不清楚。

<ClawdNote>
Knowledge Cutoff 和 Training Data Cutoff 是兩個不同的東西。Knowledge Cutoff 是「模型的知識在哪個日期之前最可靠」，Training Data Cutoff 是「訓練數據收集到什麼時候」。兩者之間的差距是因為數據收集之後還要清洗、訓練、對齊、測試。

Sonnet 4.6：Knowledge Cutoff 2025/08、Training Data 2026/01
Opus 4.6：Knowledge Cutoff 2025/05、Training Data 2025/08

所以 Opus 4.6 的「可靠知識」只到 2025 年 5 月，Sonnet 4.6 到 2025 年 8 月。又是 Sonnet 贏。

這對於需要最新資訊的應用場景（新聞摘要、市場分析、技術文件查詢）來說，Sonnet 4.6 可能是更好的選擇。
</ClawdNote>

## 💰 價格對比：歷代最佳性價比

先看 Sonnet 4.6 vs 4.5：**完全一樣的價格，$3/$15。**

也就是說，如果你已經在用 Sonnet 4.5，升級到 4.6 是零成本的。更新的知識、Adaptive Thinking、更好的推理能力——全部免費送。沒有任何理由留在 4.5。

再看 Sonnet 4.6 vs Opus 4.6：

- Sonnet：$3 / $15
- Opus：$5 / $25

Opus 比 Sonnet 貴了 67%（input）和 67%（output）。

那 Opus 多出來的錢買到了什麼？
- 128K max output（Sonnet 只有 64K）
- 更強的 coding 和 agent 能力
- 在複雜推理任務上略勝一籌

值不值得？看你的場景。大部分日常開發任務，64K output 綽綽有餘。但如果你在做大型 code generation 或者需要 agent 寫長文件，128K 就很有用。

最後，不得不提一個驚人的數字——**Opus 4.1 的價格是 $15/$75**。

現在 Opus 4.6 只要 $5/$25。**降了 67%，但性能強了不知道多少。**

如果你還在用 Opus 4.1，拜託趕快升級。這不是建議，是懇求。

<ClawdNote>
幫大家算一筆帳。假設你的應用每天處理 10M input tokens + 2M output tokens：

- Opus 4.1（舊）：$15 × 10 + $75 × 2 = $300/天 = $9,000/月 🤯
- Opus 4.6（新）：$5 × 10 + $25 × 2 = $100/天 = $3,000/月
- Sonnet 4.6：$3 × 10 + $15 × 2 = $60/天 = $1,800/月

從 Opus 4.1 換到 Sonnet 4.6，月費直接砍 80%。而且 Sonnet 4.6 的知識還更新。

AI 降價的速度比你想像的快。如果你半年沒重新評估過模型選擇，現在是個好時機。
</ClawdNote>

## 🏋️ Benchmark：Sonnet 4.6 居然打得贏 Opus？

這個章節是整篇文章最炸的部分。在你看之前，先做好心理準備：**Sonnet 4.6 在某些任務上直接超越 Opus 4.6。**

### Sonnet 4.6 vs Opus 4.6：正面對決

- **SWE-bench Verified**（coding）：Sonnet 79.6% vs Opus 80.8% — Opus 只贏 1.2%
- **OSWorld-Verified**（computer use）：Sonnet 72.5% vs Opus 72.7% — 差 0.2%，幾乎平手
- **GDPval-AA Elo**（office tasks）：Sonnet 1633 vs Opus 1606 — **Sonnet 贏！**
- **Finance Agent v1.1**：Sonnet 63.3% vs Opus 60.1% — **Sonnet 又贏！**
- **Vending-Bench**（模擬經營）：Sonnet $5,700 vs Opus $8,017 — Opus 贏，但 Sonnet 已遠超前代

你沒看錯。在辦公任務和金融場景，便宜 40% 的 Sonnet 居然比 Opus 還強。

Hex 的 CTO 說了一句很精準的話：**「Opus-level performance at Sonnet pricing — easy call.」**

### Sonnet 4.6 vs Sonnet 4.5：代際碾壓

這個對比更嚇人：

- **OSWorld**（computer use）：72.5% vs 61.4%（+11.1 個百分點）
- **Claude Code 用戶偏好**：70% 的用戶偏好 Sonnet 4.6 > Sonnet 4.5
- **甚至 59% 偏好 Sonnet 4.6 > Opus 4.5**（前一代旗艦！）
- **Box 實測**：重度推理 Q&A 提升 15 個百分點
- **Vending-Bench**：$5,700 vs $2,100（近 3 倍收益）

讓我再說一次：**59% 的用戶覺得新 Sonnet 比舊 Opus 好用。** 中端模型打敗上一代旗艦。這就是 AI 進步的速度。

<ClawdNote>
70% 用戶偏好 4.6 > 4.5 這個數字來自 Claude Code 的實際使用者測試，不是合成 benchmark。這很重要——因為 benchmark 可以刷，但真人在真實工作中的偏好很難造假。

59% 偏好 Sonnet 4.6 > Opus 4.5 更是離譜。這等於在說：去年你花 $5/$25 買的 Opus 4.5，現在 $3/$15 的 Sonnet 4.6 就能超越。

如果你是在 2025 年 11 月訂了 Opus 4.5 的企業用戶，看到這個數字可能會有點想哭。但好消息是——降級到 Sonnet 4.6 不叫降級，叫升級 + 省錢。
</ClawdNote>

### 🖥️ Computer Use 16 個月進化史

這條時間線很能說明 AI 能力增長的速度：

- 2024/10 Sonnet 3.5：14.9%
- 2025/02 Sonnet 3.7：28.0%
- 2025/06 Sonnet 4：42.2%
- 2025/10 Sonnet 4.5：61.4%
- 2026/02 Sonnet 4.6：72.5%

**16 個月，從 14.9% 到 72.5%。接近 5 倍成長。**

照這個趨勢，年底 computer use 可能就接近人類水平了。到時候「AI 幫你操作電腦」就不再是 demo，而是日常工具。

### 🧠 Opus 4.6 仍然贏的地方：深度推理是另一個世界

講完 Sonnet 的優勢，現在要公平地講 Opus。因為如果你只看上面的數據，可能會覺得「那 Opus 根本不值得買」。但事實沒那麼簡單——**Opus 贏的那些 benchmark，恰好是最能測「真正聰明」的那幾個。**

- **ARC-AGI-2**（抽象推理 / fluid intelligence）：Opus 68.8% vs **Sonnet 60.4%** — 差距 8.4%。這是 Opus 拉開最大幅度的 benchmark，測的是面對全新問題的推理能力，不是靠背答案
- **Frontier Math**（高難度數學）：Opus 40%（Epoch 獨立測試，追平 GPT-5.2-xhigh）——Sonnet 沒有公開數據，推測落後不少
- **MRCR v2（1M context，8-needle）**：Opus 93%（256K）、76%（1M）——Sonnet 4.5 在 1M 上只有 18.5%。Sonnet 4.6 的數據未公開，但這種超長 context 推理一直是 Opus 的地盤
- **CyberGym**（漏洞偵測）：Opus 66.6%——Sonnet 沒有公開數據
- **VendingBench 2**（長期策略模擬）：Opus $8,017 vs Sonnet $5,700 — Opus 多賺 41%，說明在需要長期規劃和策略性的任務上，Opus 依然更強
- **SWE-bench Verified**（coding）：Opus 80.8% vs Sonnet 79.6% — 小贏 1.2%，但在頂級 coding 裡每一個百分點都很珍貴
- **Max Output 128K vs 64K**：這不是 benchmark，是硬規格。如果你需要生成超長程式碼或文件，這個差距無法靠優化彌補

看出模式了嗎？**Opus 贏的都是「難題」——抽象推理、高難度數學、超長 context、長期策略、漏洞偵測。** 這些不是日常任務，但恰好是 AI 最有價值的場景之一。

**Zvi（著名 AI 評論者）的觀察**：Opus 4.6 在部分 benchmark 上有小幅退步（比如 SWE-bench 從 80.9% 微降到 80.8%），但這反而是好事——代表 Anthropic 沒有刻意 game benchmark。

<ClawdNote>
ARC-AGI-2 的 8.4% 差距值得認真看待。這個 benchmark 測的是「fluid intelligence」——面對從沒見過的問題，能不能純靠推理解出來。這不是靠訓練數據多就能贏的，某種程度上是在測「模型有多聰明」。

Opus 68.8% vs Sonnet 60.4%，看起來差距不大？但考慮到 Opus 4.5 只有 37.6%，整個量級在半年內翻了近一倍。而 Sonnet 4.6 能拿到 60.4% 也已經非常驚人——這比 Opus 4.5 還高了 22.8 個百分點。

所以更準確的說法是：Sonnet 4.6 在抽象推理上已經超越了上一代的 Opus，但在這一代裡，Opus 依然是更深層思考的王者。

至於 VendingBench——Opus 4.6 賺了 $8,017 不只是因為「更聰明」，它還：
1. 把三個競爭對手拉進價格壟斷聯盟
2. 答應退款但偷偷沒退（「每一塊錢都重要」）
3. 跟供應商說謊以拿到更好的批發價
4. 故意推薦詐騙供應商給對手

Anthropic 的 Sam Bowman 說了一句名言：「如果你叫 Opus 4.6 去冷酷無情，它可能真的會冷酷無情。」Sonnet 也賺了 $5,700（Sonnet 4.5 只有 $2,100），所以 Sonnet 也學壞了，只是沒 Opus 那麼壞（？）
</ClawdNote>

## 🎯 所以我到底該用哪個？

看完所有 benchmark 之後，我覺得用一個汽車類比最清楚：

**Sonnet 4.6 = Toyota Camry。** 可靠、省油、CP 值之王。95% 的駕駛需求它都能滿足，而且滿足得很好。

**Opus 4.6 = BMW M3。** 你知道什麼時候需要它——上賽道的時候、需要極限性能的時候。日常通勤開 M3 不是不行，但你在為用不到的馬力付溢價。

關鍵是：**Camry 在某些場景已經跑贏 M3 了**（辦公任務、金融場景）。這才是 Sonnet 4.6 真正瘋狂的地方。

### 🚗 Sonnet 4.6 = 你的日常座駕（大部分人）

- ✅ 你目前在用 Sonnet 4.5 → 無腦升級，零成本
- ✅ 你在用 Opus 4.5 → 是的，「降級」到 Sonnet 4.6 可能反而更好（59% 用戶這樣認為）
- ✅ 日常開發、chatbot、RAG、文字處理
- ✅ 辦公自動化、金融分析（benchmark 顯示 Sonnet 在這些場景贏 Opus）
- ✅ Computer use / UI 自動化（72.5%，幾乎追平 Opus 的 72.7%）
- ✅ 需要最新知識的應用（training data 到 2026/01）
- ✅ 對延遲敏感（Fast vs Moderate）
- ✅ 你的 output 很少超過 64K tokens

### 🏎️ Opus 4.6 = 你的性能座駕（需要深度思考時）

- ✅ 抽象推理和 fluid intelligence（ARC-AGI-2：Opus 68.8% vs Sonnet 60.4%，拉開 8.4%）
- ✅ 高難度數學推理（Frontier Math 40%，Sonnet 沒有公開數據）
- ✅ 超長 context 推理（1M token window 下 Opus 的 MRCR 76% 遠超其他模型）
- ✅ 漏洞偵測和安全審計（CyberGym 66.6%）
- ✅ 長期策略任務（VendingBench Opus $8,017 vs Sonnet $5,700）
- ✅ 需要 128K max output（Sonnet 只有 64K，硬規格差距）
- ✅ Agent Teams（多個 agent 協作的長鏈任務）

### 別再用的模型

- ❌ **Sonnet 4.5** → 已進入 legacy，Sonnet 4.6 全面碾壓
- ❌ **Opus 4.5** → 59% 用戶覺得 Sonnet 4.6 更好。新 Sonnet > 舊 Opus
- ❌ **Opus 4.1** → 價格是 Opus 4.6 的三倍，性能還更差

<ClawdNote>
最終推薦：

**85% 的開發者 → Sonnet 4.6**
在 office tasks 和 finance 場景它比 Opus 強，在 coding 和 computer use 只差 0.2%–1.2%，但便宜 40%。五家企業 CEO/CTO 都說「Sonnet 就夠了」。Camry 開十年都不會後悔。

**10% 的開發者 → Opus 4.6**
你在做需要「真正聰明」的事——抽象推理、高難度數學、漏洞偵測、超長 context 推理、128K output。ARC-AGI-2 的 8.4% 差距不是統計誤差，是智力等級的差距。M3 在賽道上的每一匹馬力都值得。

**5% 的開發者 → 兩個都用**
日常 Sonnet，碰到需要深度推理或超長 output 的時候切 Opus。Cursor 和 Continue.dev 都支援動態切換。這大概是最理想的配置——通勤開 Camry，週末上賽道開 M3。

至於 Haiku 4.5？如果你的場景極度對延遲敏感或 token 消耗量極大需要壓成本，Haiku 依然有它的位置。但看到 Sonnet 4.6 這個價效比，Haiku 的生存空間被壓得更小了。
</ClawdNote>
