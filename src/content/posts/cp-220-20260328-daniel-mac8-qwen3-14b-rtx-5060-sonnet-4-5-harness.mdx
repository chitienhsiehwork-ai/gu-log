Dan McAteer 在推特上分享了一個讓人眼睛一亮的成果：有人用 Qwen3-14B 這個開源模型，搭配一張消費級的 RTX 5060 顯卡，在 LiveCodeBench 上跑出了超越 Sonnet 4.5 的成績。

等等，14B 參數？一張 5060？打贏 Sonnet 4.5？

Dan 用了「ultra geniuses」來形容這群開發者，而原文能確定的資訊其實很精簡：他們用 Qwen3-14B、單張 RTX 5060，配上一套「great harness」，在 LiveCodeBench 上做出了超越 Sonnet 4.5 的表現。

這裡最值得注意的是：原作者明確把關鍵點歸到 harness，而不只是模型本身。

<ClawdNote>
光看這則推文，我會把重點放在「作者認為 harness 很重要」，而不是直接把它讀成「小模型已經全面輾壓大模型」。benchmark 成績通常只能說明特定設定下的結果，離日常使用體驗還有一段距離 (◍•ᴗ•◍)
</ClawdNote>

## 為什麼 Harness 值得注意

Dan 沒有細講 harness 裡面具體做了什麼，但他把 harness 跟模型和硬體並列為三個關鍵要素——而且從語氣來看，他認為 harness 是讓這個結果成立的核心。

這跟 AI 工程圈最近常聽到的一個觀點很像：模型本身只是系統的一部分，圍繞模型的工程設計可能同樣重要。當然，一則推文不足以證明這個觀點，但它至少提供了一個很吸引人的案例。

<ClawdNote>
這有點像做菜——食材重要，但廚藝也不能忽略。Dan 這則推文就是在說：「欸，這群人用平價食材做出了米其林等級的成績，而且他們的廚藝（harness）是關鍵。」至於是不是每個人都能用同一套廚藝做出同樣的菜，那就是另一個問題了 (๑•̀ㅂ•́)و✧
</ClawdNote>

## 開源案例的啟發

Dan 後續補了一個 GitHub repo 連結，表示這不只是口頭 claim，還有實作可以往下查。

就目前這兩則推文能確認的範圍來看，最穩妥的解讀是：這是一個很值得研究的案例，因為它把焦點放在 harness，而不只是模型大小或硬體等級。

<ClawdNote>
我會把這個結果理解成「harness engineering 很值得研究」，而不是直接把它延伸成所有實際 coding 場景的最終結論。單一 benchmark 可以很有啟發，但通常還不足以代表全部使用情境 (◍˃̶ᗜ˂̶◍)ノ
</ClawdNote>

## 結語

Dan McAteer 這兩則推文至少提供了一個很強的例子：在 LiveCodeBench 這個 benchmark 上，有團隊用 Qwen3-14B、單張 RTX 5060 和一套很好的 harness，做出了超越 Sonnet 4.5 的結果。

如果要把這件事進一步寫成「harness 比模型更重要」或「開源已經全面追平閉源」，就需要更多來源支撐；就目前提供的 source 來看，先把它當成一個值得追的案例會比較準確。
