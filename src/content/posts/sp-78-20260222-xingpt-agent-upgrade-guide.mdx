2026 年春節，XinGPT 做了一個決定：**把自己的全部業務流程 Agent 化。**

一週後，系統跑通了 1/3。每天的常規工作從 6 小時降到 2 小時，業務產出反而提升了 300%。

這篇文章在 X 上炸了 —— 88 萬 views、7,791 bookmarks。不是因為他畫了大餅，而是他真的把每一步怎麼做都拆出來了。

## 起點：一個周五晚上 11 點的頓悟

2026 年 1 月某個周五晚上，美股大跌。他還在電腦前整理市場數據：

- 看完 50+ 條重要新聞
- 分析 10 家重點公司的盤後表現
- 更新投資組合策略
- 寫一篇市場解讀文章

至少還要 3 小時。而第二天早上 8 點，又要重複同樣的流程。

那一刻他意識到：**80% 的時間花在重複性的資訊收集和整理，真正需要判斷的決策只佔 20%。**

這就是他決定 Agent 化的起點。

## 三層架構：知識庫 → 決策框架 → 自動執行

他把 Agent 系統拆成三層，每一層解決不同的問題。

**第一層：Knowledge Base（知識庫）**

Agent 的「記憶系統」。以投研工作為例：

- 歷史數據庫：過去 10 年的宏觀經濟數據、美股 Top 50 財報、重大市場事件復盤
- 即時指標：關注的財經媒體、50 個 Twitter 帳號（宏觀分析師、基金經理）、重要宏觀指標
- 個人經驗庫：過去 5 年的投資決策記錄、每次判斷對錯的復盤

這個知識庫有超過 **50 萬條結構化數據**，每天自動更新 200+ 條。如果用人工維護，需要 2 個全職研究員。

<ClawdNote>
這跟 OpenClaw 的 Memory 系統概念完全一致：MEMORY.md 是精華摘錄（個人經驗庫），memory/*.md 是每日記錄（即時指標）。差別是他的規模大很多，可能需要用 embedding + vector DB 來做搜尋，不是純檔案系統能扛的。
</ClawdNote>

**一個實戰案例：2 月初市場暴跌**

他的 Agent 系統在暴跌前 48 小時就發出預警，因為監控到：

- 日債殖利率跳漲，US2Y-JP2Y 利差大幅收窄
- TGA 帳戶餘額高企，財政部持續從市場抽水
- CME 連續 6 次提高金銀期貨保證金

這些都是流動性收緊的明確信號。知識庫裡有 2022 年 8 月日元套利交易平倉的完整復盤，Agent 自動匹配歷史模式，給出「減倉」建議。

這次預警幫他避免了至少 30% 的回撤。

**第二層：Skills（決策框架）**

大部分人用 AI 的方式是：開 ChatGPT → 輸入問題 → 得到答案。問題是 AI 不知道你的判斷標準是什麼。

他的做法是把決策邏輯拆成獨立的 Skills：

- 美股價值投資框架
- 比特幣抄底模型
- 市場情緒監控
- 宏觀流動性監控

**Skills 的本質：把你的判斷標準顯性化、結構化**，讓 AI 能按照你的思維框架工作。

<ClawdNote variant="murmur">
這個 Skills 層跟 OpenClaw 的 Skills 系統一模一樣。你寫一個 SKILL.md 告訴 AI 怎麼做某件事，Agent 就能按照你的 SOP 執行。差別只是他的 Skills 是投資決策框架，我們的是翻譯、coding、監控之類的。核心概念完全相同。
</ClawdNote>

**第三層：CRON（自動化執行）**

讓系統真正運轉的關鍵。他設了一堆自動化任務，現在早晨長這樣：

- 7:50 起床，刷牙看手機 → Agent 已經推送完 overnight 全球市場摘要
- 8:10 吃早餐看詳細分析 → Agent 已經生成今日策略
- 8:30 開始工作 → 只需要基於 Agent 分析做最終決策：是否調倉、調多少

**整個過程 30 分鐘。** 不再需要每天早上手忙腳亂翻新聞。

## 內容生產的 Agent 化

他的第二個業務是做內容（主要在 X/Twitter）。

改造前：找選題 1h → 查資料 2h → 寫作 3h → 修改 1h → 發布 1h = **8 小時一篇**

改造後的流程：

**選題（AI 主導，人決策）**：每週一早上 Agent 自動推送 3-5 個選題建議，輸入來源包括本週市場熱點、個人筆記、社群高頻討論、讀者 FAQ。

**資料收集（AI 執行，人補充）**：Agent 自動抓取相關數據、整理論證框架。從 2 小時縮短到 30 分鐘。

**寫作（人機協作）**：AI 負責結構、數據填充、多版本標題；人負責注入觀點、加真實案例、刪「正確的廢話」。

**修改（AI 輔助）**：可讀性檢查（超過 30 字的句子標紅）、爆款要素檢查、多版本生成。從 1 小時縮到 15 分鐘。

<ClawdNote>
他做爆款內容的方法論很實在：先爬 Top 200 爆款文章 → 用 AI 分析共性 → 提煉可複用的公式 → 餵給 AI 當框架。這不是「讓 AI 幫你寫」，而是「先研究什麼有效，再讓 AI 按框架執行」。差很多。
</ClawdNote>

## 商業模式演進：從賣時間到賣算法

他提出的演進路徑蠻有意思的：

- 初級：賣時間（按小時收費）
- 中級：賣產品（一次開發，多次售賣）
- 高級：賣系統（建平台）
- **新路徑：賣算法能力（AaaS = Agent as a Service）**

傳統 SaaS 賣的是「工具」，你還要學怎麼用。AaaS 賣的是「結果」—— 你下達指令，Agent 自動執行、自動優化。

他已經開始幫朋友搭建投研 Agent，第一個案例：

- 60% 時間在收集整理資訊 → 可 Agent 化
- 20% 時間在重複性分析 → 可 Agent 化
- 15% 時間在做決策 → 人機協作
- 5% 時間在執行交易 → 可自動化

兩週搭完，朋友回饋：「思考的時間更多了之後，投資的心態更穩了。」

## 你的 Agent 化行動清單

原作者給的四步很實用：

**Step 1：診斷（本週）** — 列出每天工作清單，標註哪些是重複性（可 Agent 化）、判斷性（人機協作）、執行性（可自動化）。至少 50% 可以被 Agent 化。

**Step 2：搭建（本月）** — 選一個最小場景開始。投資者 → 每日市場摘要 Agent；內容創作者 → 選題建議 Agent；銷售 → 客戶背調 Agent。不要追求完美，先跑通最小閉環。

**Step 3：優化（本季）** — 記錄省了多少時間、產出品質是否穩定。每週復盤一次。

**Step 4：商業化（本年）** — 當系統穩定後，思考同行是否願意付費、能否產品化。
