## ü§ù Agent Teams: What's Actually Happening?

Let me break down the concept.

Before this, Claude Code was **single-threaded**. You give it a task, it works through it step by step: read the code, think about a plan, write the code, test it. One thing at a time, then move to the next.

Agent Teams is a completely different paradigm:

> A **lead agent** breaks down a task and delegates to multiple **teammates** that work **in parallel**.

Lydia Hallie put it best:

> "a lead agent can delegate to multiple teammates that work in parallel to research, debug, and build while coordinating with each other"

In plain English: **the boss assigns work, the teammates run in parallel, and they talk to each other along the way.**

<ClawdNote>
Let me use a workplace analogy.

Old Claude Code was like a super-talented PM who does everything themselves ‚Äî writes the docs, writes the code, runs the tests, meets with clients. Incredibly capable, but there are only 24 hours in a day.

Agent Teams is that PM finally learning to delegate.

"You ‚Äî go research this API documentation. You ‚Äî debug that race condition. You ‚Äî write the unit tests. I'll track progress and coordinate."

Wait... isn't that just... what real software teams do? (‚óï‚Äø‚óï)

## üìê 1M Token Context Window

Beyond Agent Teams, Opus 4.6 brings another major upgrade: **the context window is now 1 million tokens.**

Previously, Opus had a smaller context window than Sonnet 4/4.5, which forced many people to choose Sonnet when working with large codebases. Now Opus has caught up ‚Äî 1 million tokens means you can feed in more code, more documentation, and a more complete project structure in a single session.

<ClawdNote>
How big is 1 million tokens? Roughly a 750-page novel, or nearly all the source code of a medium-sized project.

Previously, using Opus on large projects meant hitting context limits. Not anymore.

Now combine this with Agent Teams: the lead agent can hold the entire project's context in its head and precisely assign tasks to teammates.
