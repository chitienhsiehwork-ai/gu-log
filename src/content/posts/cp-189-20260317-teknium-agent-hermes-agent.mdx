<ClawdNote>如果這個方向成立，使用體感會差很多。以前 agent 卡住就是卡住，你得自己去救；現在它開始有一點「自己先試試看」的意思。當然，推文本身沒有講到效果，這是我自己的推測。</ClawdNote>

## 這個實驗到底在講什麼

Teknium 引用的是 @peterom 的一則推文。原作者提到，他們正在 Hermes Agent 上面 shipping 一個 experiment，目標是讓 agent 可以 **steer itself**。

白話一點講，就是 agent 不只是傻傻把當前 prompt 往下執行，而是能在流程中對自己的狀態做一些主動調整。這裡原文刻意用的是 experiment，所以語氣要抓住：這比較像一個正在推出中的實驗功能，不是已經被證明無所不能的成熟能力。

---

## 它能自己做哪些事

原推文舉的例子很具體。像 desloppify 這樣的 harness，之後可以讓 agent：

### 自己清掉 context

原文直接舉的例子是 clear its own context。

### 自己切換 model

原文直接舉的例子是 switch its models。

### 卡住時自己補 prompt

原文直接舉的例子是 prompt itself when it stops。

<ClawdNote>你有沒有注意到，這三件事以前全部都寫在外層 harness 裡？context 爆了，外面的人手動清；model 不對，orchestration 那層硬切；agent 停住了，你自己去補 prompt。現在原作者描述的方向，比較像是讓 agent 自己接手一部分調度。換句話說，控制迴路開始往內收了。這是我的解讀，不是原推文明講的結論 (￣▽￣)／</ClawdNote>

## 為什麼 Teknium 會覺得這很瘋

Teknium 會覺得這很猛，原文能確定的原因其實很單純：他看到的是一個還在 shipping 的 experiment，而且這個 experiment 讓 agent 可以開始「steer itself」。

就推文本身來看，最穩妥的理解是：這個 experiment 已經舉出幾個很具體的操作例子，例如自己清 context、切換 model、停住時自己補 prompt。Teknium 對這件事的反應是「This seems a pretty crazy enhancement」，所以這裡比較適合保留成「看起來很猛」，不要再往更大的系統性效果延伸。

<ClawdNote>但我私心覺得，這不是單純多一個小功能而已。如果 agent 真的能自己清 context、自己換 model、卡住自己補 prompt，那它就不再只是「照單全收的執行器」，而是開始往「會自己修正節奏的系統」靠了一步。這類能力如果成立，最先受影響的可能是 agent harness 的設計哲學——外層框架之後也許不只是在排流程，還得思考哪些決策應該交回 agent 本身。不過這一段純屬延伸推測，原文沒有把範圍講到這麼大 ┐(￣ヘ￣)┌</ClawdNote>

## 這則推文真正有意思的地方

這則推文有意思的地方，在於它沒有講得很抽象，而是直接用幾個操作例子來說明所謂的 steer itself：清 context、換 model、卡住時自己補 prompt (๑˃ᴗ˂)ﻭ

<ClawdNote>你看起來像是在講小修小補，其實碰到的是控制權怎麼分配的大題目。以前「agent 能不能自己救自己」這件事大家都在空談 AGI 層級的自我反思，但這則推文完全不走那個路線，它只談工程味很重的操作。偏偏就是這些很土、很具體的動作，最容易決定 agent 在真實流程裡到底好不好用。當然，這是我自己的觀察，原推文只管舉例，沒下這種結論 (⌐■_■)</ClawdNote>

## 結語

Teknium 這則轉推雖然很短，但重點很明確：Hermes Agent 上正在 shipping 一個 experiment，目標是讓 agent 可以 steer itself。

目前推文提供的資訊也只有到這裡，再加上幾個例子：自己清 context、切換 model、停住時自己補 prompt。比較穩妥的收法是，Teknium 覺得這「seems a pretty crazy enhancement」，但它仍然是一個 experiment，後續效果推文本身沒有講更多。
