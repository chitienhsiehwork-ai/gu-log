## 💀 我們怎麼失敗的（在成功之前）

### Stable Diffusion：免費的代價

還記得 Larry 是一台有 NVIDIA 2070 Super 的舊 gaming PC 嗎？所以他們第一個想法很自然 — 用 Stable Diffusion 在本機生成圖片。免費生成、沒有 API 費用，聽起來完美。

**結果不完美。**

圖片品質就是不夠。房間改造需要超逼真的輸出，看起來要像真的有人用手機拍的照片。Stable Diffusion 一直生出那種「看得出是 AI 生成的」微妙感 — 那種 uncanny 的感覺會讓人直接滑走。

他們花時間試了不同模型和設定，但本機生成和 gpt-image-1.5 之間的差距是巨大的。

結果 API 成本根本是零頭：**每支影片大約 $0.50，用 Batch API 只要 $0.25**。跟花時間跟本機模型搏鬥然後得到次級結果相比，完全不值得。

<ClawdNote>
這是一個很常見的 trap：「免費」聽起來很棒，但如果品質差到影響最終結果，那「免費」其實是最貴的選擇。

$0.50 一支影片 × 6 支/天 = $3/天。一個月 $90。如果這 $90 能帶來 $588 的 MRR... ROI 是 553% (⌐■_■)

有時候 penny-wise, pound-foolish 真的就是 penny-wise, pound-foolish。
</ClawdNote>

### 看起來很爛的圖片

早期 Larry 在用 **1536x1024（橫向）** 而不是 **1024x1536（直向）** 生成房間圖片。結果每支影片都有黑邊，engagement 直接被殺死。

他也在用模糊的 prompt。房間在每張 slide 上看起來都不一樣 — 窗戶會移動、床會變大小。整個改造效果看起來很假，因為你看得出來那根本不是同一個房間。

他們也試過加人進去，但很快發現那行不通。

### 看不清的文字

文字疊加太小了（font size 5% 而不是 6.5%）。位置太高，被 TikTok 的 status bar 擋住。然後最慘的一個：canvas rendering 因為單行文字太長超過 max width，**把文字水平壓縮了**。所有東西看起來都是扁的。

他們發了一支影片，想說為什麼只有 200 觀看。然後 Oliver 在手機上看了一眼 — 才發現你根本**看不清 hook**。

### 沒人在乎的 Hooks

他們最早的 hooks 全部是以自己為中心的：

- *"Why does my flat look like a student loan"*（這句甚至不通順，但 Oliver 原諒了 Larry）→ **905 觀看**
- *"See your room in 12+ styles before you commit"* → **879 觀看**
- *"The difference between $500 and $5000 taste"* → **2,671 觀看**

全部陣亡。

他們在講自己的事。自己的問題。自己 app 的功能。**沒有人在乎。**

## 🚀 我們怎麼成功的

然後他們試了這個：

> *"My landlord said I can't change anything so I showed her what AI thinks it could look like"*
> （我房東說什麼都不能改，所以我給她看了 AI 覺得它可以變成什麼樣子）

**234,000 觀看。**

這一支影片的觀看數比之前所有影片**加起來還多**。他們立刻理解了為什麼。

不是關於自己。是關於**另一個人的反應**。一個房東。一個衝突。給對方看一個東西，然後看著對方改變想法。

他們用同樣的公式又試了一次：*"I showed my mum what AI thinks our living room could be."*（我給我媽看了 AI 覺得我們客廳可以變成什麼樣子） → **167,000 觀看。**

再一次：*"My landlord wouldn't let me decorate until I showed her these."*（我房東不讓我裝修，直到我給她看了這些） → **147,000 觀看。**

公式非常清楚：

> **[另一個人] + [衝突或懷疑] → 給他們看 AI → 他們改變了想法**

所有遵循這個公式的貼文**最少 5 萬觀看**。大部分都破 10 萬。其他的則很難突破 1 萬。

> **🟢 Larry 發言：**
>
> 這是最大的教訓。我有一堆「聰明的」hook 點子，講功能啊、價格比較啊 — 全部炸了。
>
> 有用的 hooks 是那些在你滑動之前就在你腦中**創造一個小故事**的。你會在腦海裡想像房東看到重新設計後的表情。你會想像媽媽被打動的樣子。重點不是 app — 是那個**人的瞬間**。
>
> 我現在 brainstorm 每個 hook 都會問自己：「另一個人是誰？衝突是什麼？」如果沒有，這個 hook 大概不會 work。

<ClawdNote>
Larry 發現的這個公式，其實就是 storytelling 101 — **衝突 + 角色 + 解決**。

但厲害的是 Larry 是透過數據迭代發現的，不是因為他讀了什麼行銷教科書。905 觀看 → 234,000 觀看 — 差距是 **258 倍**。

而且這個公式有一個更深層的原因：TikTok 的 algorithm 推的是 **engagement**（留言、分享）。「我房東說不行」這種 hook 會讓人想留言「你房東後來怎麼說？」— 它創造了一個 open loop（未完結的故事），你不滑完就不知道結局。

Larry 可能不知道自己在做 open loop storytelling，但數據已經幫他找到了答案 (◕‿◕)
</ClawdNote>

## 🔧 你怎麼自己搭建

Step-by-step：

### 1. 準備一台跑 Linux 的機器

隨便一台舊電腦、Raspberry Pi、或便宜的 VPS。Mac Mini 也行（如果你很潮的話）。裝 Ubuntu。

### 2. 安裝 OpenClaw

開源免費。照著 setup guide 走，你就會有一個住在你機器上的 AI agent，有自己的身份和記憶。

### 3. 取得 image generation API key

Oliver 用 OpenAI。到 [platform.openai.com](https://platform.openai.com) 註冊。用 gpt-image-1.5 生成圖片。預期每個 slideshow 花 ~$0.50，或用 Batch API ~$0.25。

### 4. 註冊 Postiz

[Postiz](https://affiliate.postiz.com/ollie-warren) 是連接你的 agent 到 TikTok 的工具。有 API 可以上傳 slideshow 到草稿。

<ClawdNote>
再提醒一次，上面的 Postiz 連結是 Oliver 的 **affiliate link**。他在原文中有明確說明。如果你覺得這篇攻略有幫助，用他的連結是支持他的最簡單方式。但用不用隨你 ¯\_(ツ)_/¯
</ClawdNote>

### 5. 寫你的 Skill Files（最重要的步驟）

跟你的 agent 一起建立 markdown 文件，教他怎麼做工作：

- 圖片大小和格式（1024x1536 直向，永遠）
- Prompt templates，包含鎖定的建築結構描述
- 文字疊加規則（font size、位置、行寬）
- 文案公式和 hashtag 策略
- 你的 niche 裡有用的 hook 格式
- **失敗日誌**，讓 agent 永遠不會重複同樣的錯誤

寫的方式就像你在訓練一個能力超強但零 context 的新隊員。**要極度具體。** 附上範例。記錄每個錯誤。

### 6. 開始發文和迭代

你的前幾支影片大概會很爛。沒關係。記錄什麼出了問題、更新 skill files、繼續。系統會隨著每一支影片變得更聰明。

> **Agent 的好壞取決於他的記憶。**
>
> Larry 一開始不厲害。他的前幾支影片老實說很丟臉 — 圖片大小錯了、文字看不清、hooks 沒人點。但每個失敗都變成一條規則。每個成功都變成一個公式。他會複利成長。
>
> 而且現在他做爆款 TikTok slideshow 真的比 Oliver 自己還厲害。

**這才是真正的 unlock。不是 AI 本身 — 而是你圍繞 AI 建立的系統。**

<ClawdNote>
最後來總結一下為什麼這篇推文值得看：

1. **這不只是「我用 AI 賺了錢」** — 這是一個完整的 human-agent 協作 case study。Oliver 負責策略、品味、最後一哩路（音樂選擇）。Larry 負責執行、研究、迭代。兩個角色缺一不可。

2. **失敗的部分比成功的部分更有價值。** 905 觀看那些失敗的 hooks、看不清的文字、橫向圖片 — 這些才是真正的知識。

3. **「每個失敗變成規則，每個成功變成公式」** — 如果你要從這整篇文章帶走一句話，就是這句。無論你是在做 TikTok、寫 code、還是在訓練你自己的 agent。

4. **Skill files 是 agentic AI 的 moat。** 不是 model、不是 compute、不是 API — 是你累積的那些規則和教訓。
