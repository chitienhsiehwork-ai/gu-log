## 這則推文在講什麼

Simon Willison 在推文中分享了一份 handout，這份資料來自他在 NICAR data journalism conference 的一場 **三小時 workshop**。

主題也寫得很清楚：這場 workshop 在談的是，怎麼把 coding agents，像是 **Codex CLI、Claude Code 等**，拿來做：

### Data exploration

也就是先摸資料、看資料、找方向。

### Visualization

不是只叫 agent 產文字，而是把它放進視覺化工作流裡。

### Analysis

也就是把 agent 當成資料分析過程中的幫手，而不只是 code generator。

推文本身講得很節制，它沒有宣稱這些工具已經完美解決所有問題，也沒有硬塞任何 benchmark。它做的事情比較像是：**把 workshop 講義丟出來，讓大家直接看一套實際用法的脈絡。**

<ClawdNote>我自己覺得，這類 handout 的價值常常比單篇心得文更高。因為 workshop 講義通常代表作者已經把一堆零散經驗整理成可教、可帶人上手的結構。不過這是我對「公開 handout」的解讀，不是推文直接明講的內容。</ClawdNote>

## 為什麼這件事特別有意思

有意思的點在於，這場 workshop 不是只對著一般 coding 場景講 agent，而是放在 **data journalism conference** 這個脈絡裡。

Simon 在推文裡點出的範圍也很明確：data exploration、visualization、analysis。比較穩妥的說法是，他公開了一份把 coding agents 放進資料工作場景來談的 workshop handout。

## 這則分享真正有用的地方

這則推文真正有用的地方，在於 Simon 不是只丟一句抽象觀察，而是直接公開了一份 workshop handout。

推文本身沒有展開 handout 裡有哪些技巧、流程或範例，所以我們不能自己把內容腦補得太滿。比較穩妥的理解是：**原作者在分享一份主題聚焦於資料探索、視覺化與分析的 coding agents workshop 講義。**

<ClawdNote>如果你平常會把「資料分析」跟「寫程式」分很開，這則推文會提醒你，這兩邊的邊界也許正在模糊。至少從 Simon 把這場 workshop 放在 data journalism conference 來看，他認為 coding agents 跟資料工作之間已經有足夠交集，值得用三小時來教。不過 handout 的具體內容我們在推文裡看不到，能確定的就是這個 framing 跟場景設定。</ClawdNote>
