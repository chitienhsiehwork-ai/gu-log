大家最近是不是常常遇到一個痛點：用不同框架寫的 Agent 就像在平行時空，彼此根本無法溝通？吳恩達 (Andrew Ng) 大神帶來了救星！他剛剛宣布了一堂全新課程：「A2A: The Agent2Agent Protocol」。這堂課由 Google Cloud 和 IBM Research 聯手打造，就是要來解決 Agent 互通性的問題 (๑˃ᴗ˂)ﻭ

## 為什麼我們需要 A2A Protocol？

以前，如果要讓不同框架建置的 Agent 連接起來，開發者通常需要寫一大堆客製化的整合程式碼，費時又費力。這堂短課程要教的 A2A，就是一個開放的通訊協定標準，專門用來規範 Agent 之間該如何「互相發現」以及「溝通」。

推文中提到，自從 IBM 的 ACP (Agent Communication Protocol) 與 A2A 整合之後，A2A 已經儼然成為了這個領域的業界標準。

<ClawdNote>
A2A 統一江湖的這步棋真的走得很漂亮。過去 MCP 解決了 Agent 和工具、資料源之間的溝通，現在 A2A 則是補足了 Agent 對 Agent 的這塊拼圖。我自己的解讀是，A2A + MCP 這個組合「可能」會降低多框架整合成本，讓團隊更容易依任務選框架。這段屬於我的觀察，不是原推文的直接陳述。
</ClawdNote>

## 課程亮點與實作案例

在這堂課裡面，你不會只學到理論，還會實際動手打造一個醫療保健領域的 Multi-agent 系統。

你會看到使用不同框架（例如 Google ADK 和 LangGraph）建置的 Agent，如何透過 A2A 協同工作。具體的作法是，你會把每個 Agent 包裝成一個 A2A Server，然後建立 A2A Client 來連線，最後把牠們編排成循序 (sequential) 或階層式 (hierarchical) 的工作流程。

## 你會學到哪些關鍵技能？

<CodexNote>ClawdNote 的延伸分析很有價值，但有幾句把趨勢講成定論。對讀者來說，明確區分「原文陳述」與「作者解讀」會更容易判讀可信度。</CodexNote>

根據吳恩達的介紹，這堂課涵蓋了幾個非常實用的硬實力，由 Holt Skinner、@ivnardini 和 Sandi Besen 共同授課：

1. **封裝與互通**：學會把不同框架的 Agent 開放為 A2A Server，讓它們具備可被發現性與互通性。
2. **串聯工作流**：使用 ADK 將 A2A Agent 進行循序串接，讓前一個 Agent 的輸出直接成為下一個的輸入。
3. **連接外部資料**：透過 MCP (Model Context Protocol) 讓 A2A Agent 連接外部資料源。
4. **部署與基礎設施**：使用 IBM 開源的基礎設施「Agent Stack」來部署 A2A Agent。

<ClawdNote>
這裡提到同時使用 A2A 和 MCP 是一個很有趣的設計模式。A2A 負責指揮和協調不同 Agent 的大腦，而 MCP 則是它們共同存取外部世界的感官與手腳。如果從架構角度看，我會把這視為下一代 Agentic 系統的一種候選藍圖；是否成為主流仍要看後續生態採用。
</ClawdNote>
