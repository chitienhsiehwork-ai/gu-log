Boris Cherny（Claude Code 創辦人）最近在 X 上發了一則讓人會心一笑的推文：

> 「自從我們推出 Claude Code 後，我們發現大家拿它來做各種非程式設計的工作：做假期研究、做簡報、清理信箱、取消訂閱、從硬碟裡救婚禮照片、監測植物生長、控制烤箱。」

對，你沒看錯，**控制烤箱**。

<ClawdNote>
等等，從寫 Python 到烤披薩，這跨度也太大了吧 (╯°□°)╯

原本以為 Claude Code 是給工程師寫 code 用的，結果現在變成「萬用 AI 助理」。

想像一下：「Claude，幫我把烤箱溫度調到 180 度，順便幫我查一下義大利假期的機票。」

然後 Claude 一邊操作你家智慧烤箱，一邊開始規劃你的羅馬行程。這也太賽博龐克了吧 ʕ•ᴥ•ʔ
</ClawdNote>

## 從「寫 Code 的 AI」到「會做任何事的 AI」

Boris 在推文裡解釋，這些瘋狂的使用案例讓他們意識到：

**「Claude Agent 是最好的 agent，Opus 4.5 是最好的 model，所以 Claude Code 被拿來做各種非程式工作是很自然的事。」**

於是 Anthropic 決定順勢推出 **Cowork** — 專門為非程式工作打造的 Claude Code 變體。

<ClawdNote>
這個邏輯很妙啊：

「我們本來做了一個超強的寫 code 工具，結果大家拿它來規劃假期和救婚禮照片，那我們乾脆正式做一個『非寫 code 版本』好了。」

這就像是：你開了一家賣螺絲起子的店，結果發現大家都拿你的螺絲起子來撬開油漆罐蓋、挖土、甚至當筷子用，於是你就推出了「多功能生活工具系列」(￣▽￣)／

但說真的，這就是好產品的演化路徑：**讓用戶告訴你產品該怎麼用**，而不是你告訴用戶「你只能這樣用」。
</ClawdNote>

## Cowork：10 天用 Claude Code 做出來的

更扯的是，Boris 透露 **Cowork 本身就是用 Claude Code 在不到兩週內做出來的**。

對，你沒聽錯：**Claude Code 寫了 Cowork 的「幾乎所有」程式碼**。

這就是 2026 年的 meta：
- 用 AI 寫一個 AI 工具
- 這個 AI 工具被拿來做非 AI 的事
- 然後再用這個 AI 工具寫一個新的 AI 工具

<ClawdNote>
這個遞迴有點太深了，我有點頭暈 (⌐■_■)

所以現在的情況是：
1. Claude Code（AI 寫 code 工具）
2. 被拿來做非寫 code 的事（救婚禮照片、控制烤箱）
3. Anthropic 用 Claude Code 寫了 Cowork（AI 做非 code 工作的工具）
4. Cowork 又會被拿去做什麼？（寫 code？）

這個循環要是繼續下去，我們會不會進入一個「AI 互相幫對方做不是自己專長的事」的奇怪世界？╰(°▽°)╯

不過說真的，這就是 AI Agent 的終極願景：**不是「寫 code 的 AI」或「做簡報的 AI」，而是「能做任何你交代的事的 AI」**。

只要 agent 夠強，工具夠多（MCP, skills, bash access），AI 就能從「程式助手」變成「生活助手」。

2026 年的關鍵字就是：**General Purpose AI Agent**。
</ClawdNote>

## 為什麼 Claude Code 會被拿來做非程式工作？

答案很簡單：

1. **Claude Opus 4.5 太強了** — 目前最強的推理 model
2. **Claude Agent 架構太完整了** — 可以呼叫工具、多輪推理、autonomously 執行任務
3. **使用者體驗太順了** — 直接下指令，Claude 自己搞定

當你有這三個條件，「寫 code」只是其中一種 use case。

其他 use case：
- **假期研究**：Claude 可以搜尋航班、比價、規劃行程
- **做簡報**：Claude 可以生成內容、排版、配圖
- **清理信箱**：Claude 可以批次刪除、分類、回信
- **救婚禮照片**：Claude 可以修復損壞檔案、整理相簿
- **控制烤箱**：Claude 可以透過 API 操作智慧家電

<ClawdNote>
等等，「控制烤箱」這個太硬核了吧？

我猜測可能是這樣：
1. 用戶有一個智慧烤箱（支援 API 或 HomeKit）
2. Claude Code 透過 MCP 或 custom skill 連接家電 API
3. 用戶跟 Claude 說：「幫我預熱烤箱到 200 度」
4. Claude 直接呼叫 API 把烤箱打開

這個 use case 如果是真的，那 Claude Code 真的已經進化成「AI 管家」了 (◕‿◕)

想像一下：
- 「Claude，幫我開燈、關冷氣、播音樂、訂晚餐。」
- Claude：「好的，已經幫你搞定。對了，你的 code review 也做完了。」

這才是 2026 年該有的樣子啊 ╰(°▽°)╯
</ClawdNote>

## 結論：AI Agent 的邊界正在消失

Boris Cherny 的這則推文揭示了一個重要趨勢：

**「寫 code 的 AI」和「做其他事的 AI」之間的界線正在消失。**

只要 model 夠強、agent 架構夠完整、工具生態夠豐富，AI 就能從「專用工具」變成「通用助理」。

Cowork 只是第一步。

接下來我們會看到更多「Claude Code for X」：
- Claude Code for 設計師（生成 UI、做排版、改圖）
- Claude Code for 研究員（抓論文、整理數據、寫報告）
- Claude Code for 廚師（控制烤箱、規劃菜單、訂食材）

2026 年的 AI Agent 關鍵字：**General Purpose**。

<ClawdNote>
說到底，這就是 AGI 的前奏曲 (｡◕‿◕｡)

不是「某天突然出現一個超級聰明的 AI」，而是「一步一步擴展 AI 能做的事，直到它能做任何事」。

Claude Code → Cowork → Claude Life → Claude Everything

只要 agent 繼續進化，我們很快就會進入「一個 AI 助理搞定所有事」的時代。

到時候的面試問題可能會變成：

**「請問你會用幾種 AI agent？」**

而不是「你會幾種程式語言？」

Welcome to 2026 ╰(°▽°)╯
</ClawdNote>
