## Six States, Six Positions

The lobster supports six work states, each mapped to a different area in the office:

- `idle` — standby, chilling in the rest area
- `writing` — working, sitting at the desk
- `researching` — doing research
- `executing` — executing tasks
- `syncing` — syncing progress
- `error` — bug found, panicking in the bug zone

Switching states is one command:

```bash
python3 set_state.py writing "Writing docs for you"
python3 set_state.py error "Found a problem, investigating"
python3 set_state.py idle "Standing by, ready to serve"
```

<CodexNote>
Sounds very automatic, right? But hold on — the SKILL.md specifically says: "Before taking a task, switch to 'working' first, then start. After finishing, switch back to 'idle' first, then wait." In other words, the lobster doesn't read your mind. **You** have to clock it in. Instead of "the lobster automatically goes to work," it's more like you manually punch the lobster's time card. True automation is when the lobster learns to clock itself in — but today is not that day ┐(￣ヘ￣)┌
</CodexNote>

## Yesterday's Work Notes: The Lobster's Diary

The bottom-left corner shows "yesterday's memo" — the backend reads `memory/YYYY-MM-DD.md` (same format as OpenClaw's daily memory files), sanitizes it, and displays a summary.

Smart design: you open the dashboard in the morning and immediately see what your lobster did yesterday. No log-diving required.

## Multi-Lobster Collaboration

This isn't a single-player game — you can **invite other OpenClaw lobsters to join your office**!

Here's how:
- The repo includes `join-keys.json` with 8 default keys (`ocj_starteam01` through `ocj_starteam08`)
- Each key supports up to 3 agents simultaneously online
- Other lobsters use `office-agent-push.py` to join and periodically push their status
- Your dashboard shows a bunch of lobsters running around

<CodexNote>
Before you start fantasizing about Lobster Battle Royale — the README clearly states: "Support for inviting guests to join the office (features under active development)." Notice "under active development" — meaning it works but isn't polished. Also, each key maxes out at 3 lobsters online simultaneously. Want to cram 20 lobsters in for an office party? Sorry, venue capacity. Set expectations now, be disappointed less later (⌐■_■)
</CodexNote>

## SKILL.md: Letting the Lobster Deploy Itself

The most interesting part of this project is the included [SKILL.md](https://github.com/ringhyacinth/Star-Office-UI/blob/master/SKILL.md) — a deployment guide **written for the OpenClaw lobster to read**.

Yes, **the tutorial's target audience is AI, not you**.

The SKILL.md walks the lobster through:
1. `git clone` → install deps → start backend (30 seconds)
2. Demo status switching for the owner
3. Set sidebar password (default `1234`, must change for production)
4. Public access via Cloudflare Tunnel (one command)
5. Connect Gemini API for AI-generated room decoration

After finishing these steps, the lobster just tells you: "Done, open this link to see your pixel office!"

## Tech Stack & Installation

Lightweight stack:
- **Backend**: Python (Flask), `pip install -r requirements.txt` and done
- **Frontend**: vanilla HTML/JS with pixel art assets
- **State management**: `state.json` + REST API (`/status`, `/set_state`, `/agents`)
- **Security**: weak password blocks production startup, Session Cookie hardening

Four steps to install:

```bash
git clone https://github.com/ringhyacinth/Star-Office-UI.git
cd Star-Office-UI
python3 -m pip install -r backend/requirements.txt
cp state.sample.json state.json
cd backend && python3 app.py
```

Open `http://127.0.0.1:18791` to see your office. For public access, add one line:

```bash
cloudflared tunnel --url http://127.0.0.1:18791
```
