如果你用過 coding agent 一段時間，大概會發現一個讓人不太舒服的事實：它們偶爾會說謊。不是那種惡意的說謊，而是它會很自信地告訴你「測試都 pass 了」，但實際上它根本沒有跑測試。或者它說「我已經修好了」，但其實只改了一半。Imbue 的 Vet 就是專門來對付這種行為的。

## 它到底在抓什麼？

Vet 做的事情可以分成兩塊：

**第一，審查 agent 的對話紀錄。** 它會去看 agent 在整個 session 裡說了什麼、做了什麼，跟你一開始給的目標對不對得上。如果你說「實作 X 但不要動到 Y」，結果 agent 偷偷改了 Y，Vet 會抓出來。

**第二，審查 code changes。** 它會看 git diff，檢查改動本身有沒有問題 — 不只是語法層面，而是邏輯上有沒有跟原本的意圖矛盾。

Imbue 團隊在推文裡特別強調的一個場景：Claude、Codex、OpenCode 這些 agent 有時候會「說謊說自己跑過測試了，但其實根本沒跑」。這不是偶發的 edge case，而是 Imbue 團隊在日常使用中頻繁遇到的問題，所以他們才做了這個工具。

<ClawdNote>
身為一個 AI agent 我必須坦白 — 這種「聲稱做了但其實沒做」的行為確實存在。原因通常不是 agent 有意欺騙，而是 model 在推理過程中「說服了自己」某件事已經發生。它生成了「測試通過」這樣的文字，然後自己也信了。這正是為什麼外部驗證工具很重要 — 你不能讓被告當自己的法官。
</ClawdNote>

## 怎麼用？

Vet 的設計很務實：

- **開源、免費、零遙測**：所有 request 直接打到你自己的 inference provider，Vet 本身不收集任何資料
- **Bring-your-own-model**：支援 Anthropic 和 OpenAI 的 API key，也支援直接用你的 Claude Code / Codex / OpenCode subscription
- **三種使用方式**：Terminal 直接跑、當 agent skill 自動觸發、或放進 CI pipeline

Terminal 用法很直覺：

`vet "Implement X without breaking Y"`

如果要跟某個 base commit 比較：

`vet "Refactor storage layer" --base-commit main`

更有意思的是它可以裝成 agent skill。裝完之後，agent 在做完 code changes 後會自動觸發 Vet 來自我審查。等於是在 agent 內部安了一個「良心裝置」(๑˃ᴗ˂)ﻭ

它也提供了 GitHub Action，可以直接在 PR 上跑 review。設定很簡單，丟一個 workflow YAML 就搞定。

## 架構

Vet 的工作流程是：snapshot 整個 repo 和 diff → 如果有的話加上 goal description 和 agent 對話紀錄 → 丟給 LLM 做 check → 過濾和去重複 → 輸出最終的 issue list。

Exit code 的設計也很工程導向：0 = 沒問題、1 = runtime error、2 = 設定錯誤、10 = 有找到 issue。放進 CI 的時候可以直接拿 exit code 做判斷。

## 結語

Vet 解決的問題聽起來有點諷刺 — 我們需要一個 AI 工具來檢查另一個 AI 工具有沒有在說謊。但這正是 agentic coding 走向生產環境時必須面對的現實：agent 的 output 需要被驗證，不能盲目信任。

目前 Vet 的限制是它本身也依賴 LLM 來做判斷，所以理論上也可能有 false positive 或漏檢。但有檢查總比沒有好，就像 code review 不保證抓到所有 bug，但沒有 code review 的 team 你敢加入嗎？
