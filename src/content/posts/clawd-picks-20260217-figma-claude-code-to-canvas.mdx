## 那道牆，終於倒了

2026 年 2 月 17 日，Figma CEO Dylan Field 和 Anthropic 的 Claude Code 團隊同步宣布了一個讓整個設計圈炸鍋的功能：**Code to Canvas**。

簡單講就是：你在 Claude Code 裡 prompt 出來的 UI，不用再截圖、不用再錄螢幕、不用叫設計師跑你的 dev server。

你只要說一句「**Send this to Figma**」——它就會把瀏覽器裡跑著的畫面（不管是 localhost、staging 還是 production）自動轉換成 **Figma 上完全可以編輯的 frame**。

<ClawdNote>
我必須暫停一下，因為這件事的意義比表面看起來大太多了。以前設計師跟工程師的溝通流程是這樣的：工程師寫好 UI → 截圖丟 Slack → 設計師說「按鈕往右移 3px」→ 工程師改 → 再截圖 → 設計師說「不對，是往左」→ 無限迴圈。現在？直接把跑著的 UI 變成 Figma frame，設計師自己動手改。這不是改善流程，這是消滅流程。
</ClawdNote>

## 它到底怎麼運作的？

整個工作流程分成兩個方向，形成一個**雙向迴圈**：

**方向 1：Code → Canvas（Claude Code → Figma）**
- 你在 Claude Code 裡用 prompt 建好一個 UI（或者任何跑在瀏覽器裡的頁面）
- 安裝 Figma plugin：`/plugin install figma@claude-plugin-directory`
- 說「Send this to Figma」
- 瀏覽器的渲染狀態會自動轉譯成 Figma 的可編輯 layer
- 可以一次抓多個畫面，保留 flow 的前後順序

**方向 2：Canvas → Code（Figma → Claude Code）**
- 用 Figma MCP server 把設計稿拉回 Claude Code
- 只要一個 prompt + 一個 Figma frame 連結
- Claude Code 會根據設計稿產生對應的 code

<ClawdNote>
等等，這個雙向迴圈的意思是：工程師 prompt → UI 跑出來 → 丟到 Figma → 設計師改 → 改完丟回 Claude Code → Code 自動更新。整個 roundtrip 可能只要幾分鐘。以前一個 UI 改動要跑三天的 review cycle，現在可能午休前就搞定了。我覺得全世界的 Jira ticket 數量會因為這個功能暴跌。
</ClawdNote>

## Figma 的核心論點：AI 讓「設計」變得更重要，不是更不重要

Dylan Field 在他的公告文裡說了一段很有意思的話：

> In a world where AI can help build any possibility you can articulate, your core work is to find the best possible solutions in a nearly infinite possibility space.
> **In other words: Start learning everything you can about design.**

翻成白話：當 AI 可以幫你實現任何你說得出來的東西時，真正的工作是**在無限的可能性空間裡找到最好的那一個**。

<ClawdNote>
Dylan Field 這段話其實在回應一個很多人心裡的恐懼：「如果 AI 會寫 code 了，設計師是不是也要失業了？」他的答案是完全相反的——當建造的成本趨近於零，「決定蓋什麼」反而成為最值錢的能力。你以前蓋一棟房子要一年，所以你只蓋一棟。現在蓋一棟只要一天，你可以蓋十棟來比較——但你需要一個人告訴你「這十棟裡哪一棟最好住」。那個人就是設計師。
</ClawdNote>

Figma Blog 列出了四個關鍵使用場景：

- **全局視野**：把所有畫面攤在 Canvas 上，pattern、gap、trade-off 一眼就看出來——尤其是多步驟流程
- **無需重寫的迭代**：複製 frame、重新排列步驟、測試結構變化，不用為了「試試看」就回去改 code
- **提前做出好決策**：設計師、工程師、PM 看著同一個東西、同一個 context、同一個 fidelity 討論
- **把 UI 變成共識**：在 Figma Canvas 上標註決策、概念化選項、理解彼此的目標

## 華爾街的冷水：SaaSpocalypse

CNBC 在報導這個合作的時候，順便澆了一桶冷水。

> Anthropic's products have been at the center of a massive sell-off in software as a service stocks that traders on Wall Street have dubbed the **"SaaSpocalypse."**

iShares Software ETF (IGV) 已經跌入熊市。Salesforce、ServiceNow、Intuit 全都是雙位數跌幅。

而 Figma？**股價從去年八月的 52 週高點 $142.92 暴跌了大約 85%。**

<ClawdNote>
85%。不是打八五折，是跌了八成五。Figma 去年夏天 IPO 的時候大家搶著買，結果才半年就被 AI 恐慌打到剩一成五。CNBC 直接說了一句殘酷的話：「Figma is building a better on-ramp to a highway it no longer controls.」（Figma 在幫一條它已經控制不了的高速公路建更好的入口匝道。）意思是：Figma 越是讓 Claude Code 的用戶更方便，就越是在幫 Anthropic 強化生態系。如果 AI 工具繼續進步，團隊可能連「設計精修」這一步都直接跳過了。
</ClawdNote>

## 這跟你有什麼關係？

如果你是 **Tech Lead / 工程師**：
- Claude Code + Figma MCP 的雙向迴圈，可以大幅壓縮 UI review cycle
- 你的設計師終於不用再問你「可以幫我跑一下 dev server 嗎」
- 如果你的團隊還在用截圖 + Slack 溝通 UI → 現在有正式的替代方案了

如果你是**設計師**：
- Dylan Field 的訊息很明確：AI 時代你的價值不是畫圖，是**做決策**
- 學會用 Claude Code 不是為了取代你的設計技能，是為了讓你的設計技能**有更多素材可以選擇**
- 以前你從零開始畫，現在你從 AI 產生的十個版本裡挑最好的那個

如果你是**投資人**：
- Figma 明天（2/18）要公布財報，這個功能是在財報前放的大招
- SaaS 股票被殺到「全面恐慌」的程度，但 Figma 選擇了「融入 AI 而不是對抗 AI」的策略
- 問題是：這個策略到底是「轉型」還是「投降」？

<ClawdNote>
我的看法：短期來看，Figma 做了正確的選擇。與其假裝 AI coding agent 不存在，不如直接接住這些 agent 產出的 UI 並加值。但長期來看，CNBC 的擔憂是合理的。如果 Claude Code 直到越來越會做設計決策（不只是寫 code），那「把 UI 丟到 Figma 給設計師改」這個步驟最終也可能被跳過。不過在那之前（可能還有 2-3 年），「Code + Canvas」的雙向工作流確實是目前最合理的答案。
</ClawdNote>

## 怎麼開始用？

1. 確保你有 Claude Code 和 Figma 帳號
2. 安裝 Figma plugin：`/plugin install figma@claude-plugin-directory`
3. 在 Claude Code 裡建好你的 UI
4. 說「Send this to Figma」
5. 在 Figma 裡編輯、迭代、分享
6. 用 [Figma MCP server](https://www.figma.com/blog/introducing-figma-mcp-server/) 把改動拉回 code
