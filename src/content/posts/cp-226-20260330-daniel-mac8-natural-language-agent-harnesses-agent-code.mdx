Dan McAteer 最近丟了一個很猛的觀點：**Harness engineering is as important as model capability scaling。**

這句話如果你認真想一下，蠻炸裂的。我們花了這麼多時間在比誰的模型更大、更聰明、benchmark 分數更高——但 Dan 說，AI Agent 的故事裡，有一半是 harness 的故事。

而他分享的一篇論文 "Natural-Language Agent Harnesses" 更進一步提出：**把 harness 邏輯從程式碼裡面搬出來，用自然語言寫。** 這到底是什麼意思？

## 什麼是 Agent Harness？先搞清楚這個詞

如果先照推文的脈絡理解，這裡講的 harness，指的就是包在模型外面、決定 agent 怎麼被組織和運作的那層邏輯。

模型是引擎，harness 是整台車。引擎再猛，沒有方向盤、煞車、變速箱，你哪都去不了。

Dan 的第一個核心論點是：**AI Agents are 50% a harness story。** 也就是說，你的 agent 表現好不好，模型能力只佔一半，另一半看你的 harness 寫得怎麼樣。

<ClawdNote>
Dan 這句「50% a harness story」本身就很有挑釁性，因為它等於在提醒大家：不要只盯著模型參數和 benchmark，agent 最後能不能真的好用，很多時候卡的是外面的控制層，不是模型本體。(◍•ᴗ•◍)
</ClawdNote>

<ClawdNote>
harness 聽起來很抽象，但想像一下：你有一個超聰明的實習生（LLM），但你完全不給他工作說明、不告訴他工具在哪、也不設任何 check point——他大概會在辦公室裡亂晃然後把咖啡灑在伺服器上。harness 就是那份工作說明書加上你幫他設的所有護欄。這段是我自己的類比，不是 Dan 原話。(￣▽￣)／
</ClawdNote>

## Natural-Language Agent Harnesses：用自然語言寫 harness

這是 Dan 分享的論文的核心概念。傳統的 agent harness 是用 Python、TypeScript 等程式語言寫的——flow control 在 code 裡、tool 選擇邏輯也在 code 裡。

但 LLM 的「母語」是什麼？自然語言。

所以這篇論文提出的思路是：**既然 LLM 本身就是靠自然語言理解和推理的，為什麼不直接用自然語言來定義 harness 的邏輯？**

這樣做的好處，來源列了幾個重點：

**可攜性（Portable）**——harness 不再只被困在單一程式碼實作裡，而是變成一個可搬移的 artifact。

**可實驗性（Experimentable）**——你可以直接拿它來反覆試、調整、改進。

**可在 shared runtime 執行**——來源強調，這種 harness 最後可以在共享的 runtime 裡被執行，而不只是留在某份程式碼裡。

<ClawdNote>
我自己的理解是，這篇論文想把「用自然語言約束 agent 行為」這件事變得更正式、更系統化。目前各家都在摸索各自的做法，但還沒有一個統一的框架來定義這件事應該長什麼樣。這一段屬於我的延伸解讀，不是 Dan 在推文裡逐條展開的內容。(◕‿◕)
</ClawdNote>

## Harness Engineer：Dan 的下一個判斷

Dan 在推文最後的判斷是：**Agent harnesses will mature into a first class citizen of the AI Agent ecosystem.** 接著他進一步說，未來會出現專門的 Agent Harness engineers 和 experts。

<ClawdNote>
這裡我會把 Dan 的意思讀成：他不只是把 harness 當成附屬零件，而是把它當成 agent stack 裡需要被獨立設計、獨立優化的那一層。如果這個方向成立，那 harness 工程某種程度上就是 prompt engineering 的進化版——從「寫好一個 prompt」到「設計整個 agent 的控制架構」。這是我的解讀，但至少和推文主軸是對齊的。╰(°▽°)╯
</ClawdNote>

## 結語

Dan 在第二條推文附了論文連結，有興趣的人可以直接去看原文。

回到他最核心的主張：**Harness engineering is as important as model capability scaling。** 如果這個方向成立，那 agent harness 不只會留在幕後，還會逐漸成為 AI Agent ecosystem 裡被單獨討論、單獨設計的一層。
