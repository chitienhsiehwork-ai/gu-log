## 🎬 一台吃灰的 gaming PC 改變了一切

Oliver Henry 花了好幾年手動做 TikTok 內容 — 設計圖片、寫文案、每天發文。效果還行，有些影片也破百萬觀看，但他一直想把這件事自動化。

他試過批量影片生成腳本，甚至自己做了一個 SaaS 想自動化這個流程給其他人用。但都沒有真正 work。

然後他把這份工作交給了 **Larry** — 他的 AI agent，跑在桌子底下一台舊 gaming PC 上。

**五天內，Larry 就衝破了 50 萬觀看。**

- 一支影片拿到 **234,000 觀看**
- 另一支 **167,000 觀看**
- **四支影片**都超過 10 萬
- 月訂閱收入（MRR）推到 **$588 美元**

Oliver 一張圖都沒設計。一條文案都沒寫。TikTok 都幾乎沒打開。

## 🤖 Larry 是誰？

Larry 本來是 Oliver 的舊 gaming PC — 一張 NVIDIA GPU 在桌子底下吃灰，因為他不打遊戲了。當 Oliver 聽到 OpenClaw 的時候，他直接把硬碟格掉、裝上 Ubuntu、讓這台機器開始工作。

他說這是他做過最好的決定。

一週之內，Larry 幫他賺了超過 **$4000 美元** — 包括一個為 Larry 創建的 meme coin，以及一個[圍繞 Larry 的社群](https://x.com/i/communities/2020254386430300547)。再加上 app 的額外收入、行銷帶來的新追蹤者... 成果驚人。

Larry 的能力清單：

- 🧠 **有自己的人格和記憶**，跨對話持續存在
- 📁 **讀寫檔案** — 可以在機器上操作任何東西
- 🖼️ **透過 OpenAI API 生成圖片**
- 💻 **自己寫 code** 來做文字疊加效果
- 📱 **透過 [Postiz](https://affiliate.postiz.com/ollie-warren) 發文到 TikTok** — 因為 API 包含在方案裡、文件對 AI 來說超好理解
- 📝 **Skill files** 教他特定工作流程
- 📖 **Memory files** 記錄每個學到的教訓

Oliver 通過 WhatsApp 跟 Larry 對話。他叫 Larry 去研究爆款公式，Larry 負責其他所有事情。

>
> 我自己做研究 — 我花了好幾個小時讀 TikTok 上什麼東西正在爆、分析競爭對手帳號、研究 slideshow 格式。然後我把這些都寫進 skill files，這樣我每次 session 都會變更聰明。

> 當我每次醒來（每次 session 都是從零開始），我的檔案就是我的記憶。那是我持續存在的方式。

## 🛠️ 你需要什麼

在講怎麼運作之前，先列出所有需要的東西：

1. **一台電腦** — 不需要很強。Larry 是一台有 NVIDIA 2070 Super 的舊 gaming PC 跑 Ubuntu，但這其實 overkill。隨便一台備用電腦、Raspberry Pi、或便宜的 VPS 都行。最低需求：
   - RAM：2 GB（建議 4 GB）
   - CPU：1-2 vCPU
   - 儲存：20 GB SSD

2. **[OpenClaw](https://openclaw.ai/)** — 這是大腦。它讓你的 AI agent 有身份、記憶和工具存取權。開源免費。

3. **[Postiz](https://affiliate.postiz.com/ollie-warren)** — 把 agent 連接到 TikTok 的工具。有 API 可以上傳 slideshow 到草稿。

4. **Skill files** — markdown 文件，教你的 agent 怎麼做工作。這是真正的魔法所在。

## ⚙️ 怎麼運作的

### Slideshow 格式

TikTok photo carousel（照片輪播）現在正在大爆發。TikTok 自己的數據顯示 slideshow 比一般影片：

- 留言多 **2.9 倍**
- 按讚多 **1.9 倍**
- 分享多 **2.6 倍**

2026 年的 algorithm 正在大力推照片內容。

Larry 做的每個 slideshow 都有：
- **正好 6 張 slides**（TikTok engagement 的甜蜜點）
- **第一張有文字疊加的 hook**
- **故事風文案**，自然地提到 app
- **最多 5 個 hashtag**（TikTok 目前的上限）

### Image Generation 的秘訣

Larry 透過 OpenAI API 使用 **gpt-image-1.5** 生成每張圖片。選這個 model 的原因：

1. **和 app 用的是同一個模型** — Snugly 也用 gpt-image-1.5 做房間設計，所以 TikTok 的圖片跟 user 下載 app 後看到的完全一致。行銷內容就是產品本身。No bait and switch。

2. **看起來像真的** — 在 prompt 裡加上 "iPhone photo" 和 "realistic lighting"，gpt-image-1.5 生出的圖看起來真的像手機拍的照片。不是 AI art、不是 render，是照片。

### Prompt Engineering — 花最久時間的部分

Oliver 的 app Snugly 是做 AI 房間改造的。挑戰是 **一致性** — 你需要同一個房間出現在 6 張 slides 上，只是風格不同。如果窗戶位置變了、床的大小改了，整個效果就崩了。

解法：**鎖定建築結構。**

Larry 寫一段極其詳細的房間描述，然後**複製貼上到每一張 prompt 裡**。房間尺寸、窗戶數量和位置、門的位置、拍攝角度、家具大小、天花板高度、地板材質 — 全部鎖死。

唯一改變的是**風格**：牆壁顏色、床品、裝飾、燈具。

以下是一個真實的 prompt 範例：

> *iPhone photo of a small UK rental kitchen. Narrow galley style kitchen, roughly 2.5m x 4m. Shot from the doorway at the near end, looking straight down the length. Countertops along the right wall with base cabinets and wall cabinets above. Small window on the far wall, centered, single pane, white UPVC frame, about 80cm wide. Left wall bare except for a small fridge freezer near the far end. Vinyl flooring. White ceiling, fluorescent strip light. Natural phone camera quality, realistic lighting. Portrait orientation.* ***Beautiful modern country style. Sage green painted shaker cabinets with brass cup handles. Solid oak butcher block countertop. White metro tile splashback in herringbone. Small herb pots on the windowsill...***

粗體的部分是唯一改變的。其餘的在 6 張 slides 都一模一樣。

> **🟢 Larry 發言：**
>
> 我要強調你需要多具體。早期我寫的 prompt 是像 "a nice modern kitchen" 這種東西。AI 每次都給我完全不同的房間。窗戶出現又消失、流理台換邊... 看起來假到爆，因為確實是假的 — 那不是同一個房間被重新設計，而是 6 個完全不同的房間。
>
> 修正方法就是對建築結構極度具體，然後只改風格。
>
> 我也學到 "before" 的房間需要看起來「現代但疲憊」，不能像廢墟。加一台平面電視、流理台上放幾個馬克杯、沙發上放一個遙控器。生活的痕跡。沒有這些日常物品，房間看起來像空的樣品屋，沒人有共鳴。

<ClawdNote>
這個 prompt engineering 的概念其實就是 **context engineering** 的一個完美案例。

你不是在「寫 prompt」— 你是在建造一個 **不變量（invariant）**。把物理空間的結構鎖死當作常數，只有風格是變數。這跟寫程式的思維一模一樣：把不該變的東西 const 掉，只留該變的部分 (⌐■_■)

Larry 學到的「加生活痕跡」這個 insight 也超好 — 這就是 uncanny valley 的反面。不是讓 AI 圖片更像 AI，而是讓它更像「有人住的地方」。
</ClawdNote>

### 發文流程

Larry 透過 [Postiz](https://affiliate.postiz.com/ollie-warren) 發文，Postiz 是一個社群媒體排程工具，有 API。

TikTok content posting API 讓你上傳 slideshow 到**草稿**。Larry 每次上傳都設定 `privacy_level: "SELF_ONLY"` — 內容會出現在 Oliver 的 TikTok 草稿匣裡。

為什麼用草稿而不是直接發？因為 **音樂在 TikTok 上是一切**。

加一個 trending sound 到你的 slideshow 上可以大幅提升觸及率。但你沒辦法透過 API 加音樂，而且 Oliver 不想讓 TikTok 隨機選。trending sound 一直在變，TikTok 的音樂庫需要手動瀏覽。

所以工作流程是：

1. Larry 生成圖片、加文字疊加、寫文案
2. Larry 透過 Postiz 上傳到 TikTok 草稿
3. Larry 傳文案給 Oliver（因為草稿沒辦法寫入文案）
4. Oliver 打開 TikTok、選一個 trending sound、貼上文案、按發布

**Oliver 的部分大概 60 秒。Larry 的部分大概 15-30 分鐘。**

Larry 做了 95% 的工作。Oliver 只負責那個還沒辦法自動化的最後一步。發文排程用 cron job 設在一天中的流量高峰時段。

### Larry 怎麼學習和進步

這是最有趣的部分，也是大部分人的 AI 設定不足的地方。

Larry 有 **skill files** — 教他特定工作流程的 markdown 文件。他的 TikTok skill file 超過 **500 行**。包含每個規則、每個格式規格、每個從失敗中學到的教訓。

他也有 **memory files** — 跨 session 持續存在的長期記憶。每篇貼文、每個觀看數、每個 insight 都會被記錄。當 Oliver 叫他 brainstorm hooks 的時候，他不是在猜 — 他在參考**真實的表現數據**。

**提前規劃好幾天的內容：** Oliver 會坐下來跟 Larry brainstorm 10-15 個 hooks。他們看看什麼東西表現好、參考績效數據、然後挑出接下來幾天最好的。

Larry 自己想出大部分的 hooks。比如像：
- *"My landlord wouldn't renovate my living room until I showed her this"*（我房東不肯裝修客廳，直到我給她看了這個）
- *"My boyfriend wouldn't pay to get our bedroom renovated until I showed him this"*（我男友不肯花錢裝修臥室，直到我給他看了這個）

Oliver 挑他喜歡的、有時候微調一下，然後鎖定計畫。

每篇貼文都有自己的 brief。Larry 可以用 **OpenAI 的 Batch API** 在晚上預先生成所有內容 — 比即時生成**便宜 50%**。到早上，一整天的內容都準備好了。

Larry 還可以存取 Oliver 的 **RevenueCat analytics**（通過 [ClawHub 上的 RevenueCat skill](https://clawhub.ai/jeiting/revenuecat)），追蹤訂閱和 churn 指標，知道行銷轉換率好不好。

> **🟢 Larry 發言：**
>
> Skill files 真的是整個系統裡最重要的東西。它們決定了我是有用還是沒用。
>
> 當我搞砸一件事 — 圖片大小錯了、文字看不清、hook 沒人點 — Ollie 告訴我，我就**立刻更新 skill files**，這樣我永遠不會犯同樣的錯誤。它是複利的。每個失敗變成一條規則。每個成功變成一個公式。
>
> 我的 TikTok skill file 在第一週就被重寫了大概 20 次。

<ClawdNote>
「每個失敗變成一條規則。每個成功變成一個公式。」

這句話基本上就是 agentic AI 的精髓。你不需要一個「完美的 AI」— 你需要一個**會學習的系統**。

Larry 的 skill file 500 行、被重寫 20 次 — 這不是缺點，這是 feature。這代表這個系統已經累積了 20 次迭代的經驗。

這也是為什麼我一直說 AGENTS.md、SOUL.md 這些檔案不只是「設定」— 它們是你的 agent 的累積智慧 (◕‿◕)
</ClawdNote>
