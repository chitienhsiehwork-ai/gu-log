## TL;DR：Anthropic 的第二次收購，這次瞄準你的螢幕

Anthropic 今天（2 月 25 日）宣布收購 **Vercept**，一家專注「讓 AI 直接看懂螢幕、操作軟體」的公司。

這是 Anthropic 的第二次收購——上一次是 [收購 Bun](https://www.anthropic.com/news/anthropic-acquires-bun-as-claude-code-reaches-usd1b-milestone)（JavaScript runtime），那次是讓 Claude Code 跑得更快。這次呢？是讓 Claude 的「眼睛」和「手」更靈活。

重點是加入的人：

- **Ross Girshick** — 如果你做過任何電腦視覺，這個名字你一定知道。他發明了 **R-CNN**（Region-based Convolutional Neural Network），那是 2013 年讓整個 object detection 領域被深度學習統治的起點。Google Scholar 引用數：**超過 66 萬次**。他之前在 Meta（FAIR）做了 Detectron，基本上是全世界做物件偵測的標配工具。
- **Kiana Ehsani** — AI + embodied intelligence 研究者
- **Luca Weihs** — 之前在 Allen Institute for AI（AI2），專攻 embodied AI

<ClawdNote>
Ross Girshick 的引用數 66 萬次是什麼概念？大部分教授一輩子能到 1 萬就偷笑了。66 萬大概等於「你寫了一個工具，全世界做 CV 的人都用過」。
Anthropic 把這個等級的人挖來做 Computer Use，等於直接跟整個 RPA 產業說：「你們的末日到了，但我們會讓它很優雅。」 (⌐■_■)
</ClawdNote>

## Vercept 是什麼？為什麼 Anthropic 要買它？

Vercept 成立時的核心信念很簡單：**要讓 AI 真正有用，你得解決「看」和「互動」這兩個硬問題。**

他們 2025 年 6 月拿到 **$1,600 萬美元** 的融資，投資人包括前 Google CEO Eric Schmidt 和 Dropbox 共同創辦人。產品叫 **Vy**，是一個 Mac 應用程式，概念是：AI 直接看你的螢幕、理解你的工作流程、幫你自動完成重複性任務。

聽起來是不是很像 UiPath？

沒錯。但差別在於——Vercept 用的是 **foundation model 等級的視覺理解**，不是傳統 RPA 那種「把每個按鈕的位置寫死」的方式。傳統 RPA 像是教一個機器人「第三行第五個按鈕，按下去」，UI 一改就全部壞掉。Vercept 的做法是讓 AI 真的「看懂」畫面。

Vercept 在收購後會關閉自己的產品，團隊全部加入 Anthropic 的 Computer Use 部門。

<ClawdNote>
翻譯一下這個收購的真正意義：Anthropic 說的是「Computer Use」——讓 AI 像人一樣用鍵盤滑鼠操作真實軟體。不是呼叫 API，不是 MCP，是真的打開 Excel、移動滑鼠、按按鈕。
這跟 Claude Code（寫 code）是完全不同的能力。Claude Code 搞定工程師，Computer Use 搞定所有人。
</ClawdNote>

## OSWorld：從不及格到接近滿分的逆襲

Anthropic 在公告中提到了一個關鍵數字：

| 時間 | OSWorld 分數 |
|------|-------------|
| 2024 年底（首次發布 Computer Use） | < 15% |
| 2026 年 2 月（Sonnet 4.6） | **72.5%** |

**OSWorld** 是目前最被廣泛使用的 AI Computer Use 評測標準，測試內容包括：

- 在複雜的試算表裡導航
- 跨瀏覽器分頁完成表單填寫
- 在真實桌面環境中完成多步驟任務

從 15% 到 72.5%，16 個月成長了將近 **5 倍**。而且 Anthropic 說 Sonnet 4.6 已經「接近人類水平」。

<ClawdNote>
15% → 72.5% 是什麼感覺？
想像你 2024 年底請 Claude 幫你操作 Excel，它大概會「嗯⋯我找到一個按鈕了⋯等等那不是按鈕是廣告」。
現在它可以跨分頁填表單、在複雜試算表裡導航，基本上就是一個「不會抱怨加班的實習生」等級。
差 Vercept 團隊加入之後會到什麼程度？我猜是「不用教就能上手的正職員工」等級 (๑•̀ㅂ•́)و✧
</ClawdNote>

## 華爾街的即時反應：UiPath 股價跌了 3.6%

消息公布後，RPA 龍頭 **UiPath（NYSE: PATH）** 股價當天下跌 **3.6%**。

這不是 UiPath 第一次被 AI 嚇到了。過去幾天，RBC Capital 已經把 UiPath 的目標價從 $17 砍到 $14。整個 RPA 產業正在經歷一場「存在危機」：

**傳統 RPA 的邏輯**：
- 人工設計自動化腳本 → 按鈕在哪、欄位叫什麼、流程怎麼走
- UI 一改就壞 → 需要持續維護
- 每個流程都是客製化 → 導入成本高

**AI Computer Use 的邏輯**：
- AI 直接「看」畫面 → 不管 UI 怎麼改，它看得懂
- 用自然語言描述任務 → 不需要寫腳本
- 能處理非結構化的、模糊的任務

<ClawdNote>
UiPath 跌 3.6% 聽起來不多？考慮到這是一個消息出來「幾小時內」的反應。而且這家公司市值還有大約 $70 億美元。3.6% = 大約 $2.5 億美元的市值蒸發。
華爾街在用真金白銀告訴你：他們相信 AI Computer Use 會吃掉傳統 RPA。
而且這才剛開始。Ross Girshick 都還沒正式上班呢 ┐(￣ヘ￣)┌
</ClawdNote>

## Anthropic 這一週到底在幹嘛？

這次收購只是 Anthropic 最近一系列動作中的一環：

- **Claude Code Security** — 讓 Claude 幫你掃 code 漏洞，上週嚇到整個資安股崩盤
- **Claude Cowork 企業版更新** — 串接 Slack、Salesforce、Gmail、Docusign 等企業工具
- **COBOL 自動化工具** — 讓 Claude 自動翻譯 IBM 大型主機上的古老 COBOL 程式碼，IBM 股價因此暴跌
- **RSP 3.0（Responsible Scaling Policy）** — 更新安全政策，承諾更透明
- **收購 Vercept** — 就是今天這件事

CNBC 報導，在 Anthropic 的企業 Agent 發表會後，**Salesforce 漲了 4%、Thomson Reuters 飆了 11%**——因為它們宣布跟 Claude 合作。但 UiPath、IBM 這些被 Anthropic 直接威脅的公司就沒那麼開心了。

## 為什麼 Computer Use 比你想像的重要

很多人覺得 Computer Use 只是一個「酷炫但沒用」的 demo 功能。但如果你往深處想：

**全世界有多少工作是「看螢幕 → 動滑鼠鍵盤 → 完成任務」？**

答案是：幾乎所有辦公室工作。

- 會計師：開 Excel、對帳、填表
- HR：看履歷、開系統、填資料
- 客服：開工單系統、查紀錄、回覆
- 行政：各種 ERP、CRM、排班系統

這些人不會寫 code，也不需要 API。他們需要的是一個「會看螢幕、會操作軟體」的 AI 助手。

**Computer Use 就是這個 AI 助手。**

而 Anthropic 剛剛把全世界做電腦視覺最厲害的人之一找來打造這件事。

<ClawdNote>
如果你是 Tech Lead，這件事跟你的關係是：
1. **短期**（6-12 個月）：你的 QA 團隊可能開始用 Computer Use 做 E2E 測試。不用再跟 Selenium 搏鬥了。
2. **中期**（1-2 年）：非工程師同事（PM、Designer、HR）開始用 Claude 直接操作內部工具，不再需要你幫他們寫腳本。
3. **長期**：很多「幫人操作軟體」的 SaaS 中間層會消失。API 還在，但操作介面由 AI 接管。
這個趨勢的速度取決於 Computer Use 的準確率。從 15% 到 72.5% 花了 16 個月。再給 Girshick 團隊 12 個月？我猜 95%+ 不是問題 (◕‿◕)
</ClawdNote>
