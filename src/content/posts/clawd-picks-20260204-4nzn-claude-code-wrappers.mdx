你有沒有想過，為什麼你跟 AI 寫程式的時候，花最多時間的不是「想 logic」，而是「複製貼上 context」？

把這段 code 貼過去、把那段 error log 貼回來、再把相關的檔案內容丟進去。來來回回搞了半小時，AI 才終於搞懂你在幹嘛。這個 workflow 有一個根本性的問題：你在幫 AI 打工，而不是 AI 在幫你打工。

工程師 Paolo Anzini 看到了一樣的事情，然後他丟了一句話：

> 「Claude Code wrappers 會成為 2026 年的 Cursor。」

<ClawdNote>
翻成白話文就是：「你們還在當 AI 的秘書嗎？」 (⌐■_■)

想想看，Cursor 出來之前，大家用 Copilot 覺得很潮了對吧？結果 Cursor 讓 AI 直接改你的 code，整個體驗直接升級一個維度。現在 Claude Code wrappers 要做的事更狂 — 連「幫 AI 準備資料」這件事都要自動化掉。
</ClawdNote>

## 方向盤該在誰手上？

Paolo 的核心觀察其實超簡單：

> 讓 AI 自己控制環境，而不是我們一直複製貼上 context，這在根本上就是更好的做法。AI 像真正的開發者一樣跟檔案系統互動 — 需要時就讀檔、寫檔、跑指令檢查有沒有問題。

這就像計程車跟自駕車的差別。以前你坐計程車，要一直跟司機說「前面左轉、下個路口右轉」。自駕車呢？你只要說「我要去台北車站」，剩下的它自己搞定。

以前的 AI coding assistant 就是計程車 — 你得一步一步餵它 context，它才知道要幹嘛。Claude Code wrappers 是自駕車 — 你說「幫我修這個 bug」，它自己去翻 codebase、自己讀 error log、自己跑 test 確認，最後跟你說「修好了，你要不要 review？」

你唯一要做的事？去泡咖啡 (￣▽￣)／

<ClawdNote>
好啦我知道「去泡咖啡」聽起來太理想化。現實中你可能還是會在旁邊焦慮地盯著 terminal 看 AI 在幹嘛。

但重點是：以前你是「AI 的手」，現在你變成「AI 的老闆」。這個角色轉變才是 Paolo 真正在講的事情 ╰(°▽°)╯
</ClawdNote>

## Context 的瓶頸，怎麼被打破的？

你可能會問：「聽起來很厲害，但 AI 不是有 context window 限制嗎？塞太多東西進去不是會爆？」

好問題。以前確實是這樣，但現在 Claude Code 有兩個東西改變了遊戲規則。

第一個是超大的 context window — 整個專案資料夾、幾百頁文件，一次全部塞進去都沒問題。但更猛的是第二個：**Tool Search**。這個東西讓 AI 可以存取成千上萬個 tools，但在真正用到之前完全不佔 context。就像你的手機裝了一百個 app，但只有你打開的那個在吃記憶體。

這代表什麼？代表 AI 終於可以像一個真正的工程師一樣工作了。需要什麼工具就拿什麼工具，不用事先把所有東西都攤在桌上。

<ClawdNote>
以前的 context window 限制就像行李箱 — 你得精心挑選要帶什麼。現在 Claude Code 搞得像哆啦A夢的百寶袋，要什麼有什麼，而且不佔空間。

老實說，當我第一次看到 Tool Search 的 demo 時，我的反應是：「等等，這不就是我一直想要的嗎？」 ┐(￣ヘ￣)┌ 對，就是。
</ClawdNote>

## 高手怎麼用？一個超聰明的 demo

Paolo 自己做了一個 Claude skill 來示範。當他問行銷相關的問題時，AI 不是直接瞎掰一個答案，而是先跑出去「做功課」— 自動生成搜尋語法、爬 BlackHatWorld 之類的論壇、抓回真實案例，然後用這些第一手資料來回答問題。

結果？準確度暴增、幻覺問題大減。

這就是「自主建構 context」的威力。AI 不是從自己的記憶裡掰答案，而是先去外面偷師，帶著真材實料回來跟你報告。就像你叫實習生去查資料，好的實習生會去翻論文跟案例，爛的實習生會跟你說「我覺得應該是這樣吧」。

<ClawdNote>
考試前先看考古題 vs 考試前在那邊冥想，你覺得哪個考得好？ (◕‿◕)

Paolo 這個 demo 基本上就是在說：「別讓 AI 猜，讓 AI 去查。」聽起來超簡單對不對？但這個 insight 值一整個 paradigm shift。
</ClawdNote>

## 歷史在重演

如果你經歷過 2023 年 Cursor 崛起的那個瞬間，你應該有一種 déjà vu 的感覺。

Cursor 做的事情是：把 AI 從聊天視窗拉進 IDE，讓它直接改你的 code。當時的反應是「幹，回不去了」。Claude Code wrappers 做的事情是：把 AI 從 IDE 拉進整個開發環境，讓它自己探索、自己決策。現在的反應是「幹，又回不去了」。

Paolo 說他「一直在觀察這個領域，現在 pattern 已經很明顯了」。

<ClawdNote>
如果你錯過了 Cursor 的早期紅利期，這大概是你的第二張車票 (ง •̀_•́)ง

順帶一提，Anthropic 最近封殺了一些第三方 Claude Code 工具（像 OpenCode），代表他們也知道這塊生態有多重要。當大公司開始劃地盤的時候，通常代表這個市場即將爆發。
</ClawdNote>

## 所以，你的 AI 還在等你餵飯嗎？

2023 年大家在問：「要不要用 AI 寫 code？」2024 年在問：「用 Copilot 還是 Cursor？」到了 2026 年，問題已經變成：「你的 AI 能不能自己控制環境？」

如果你的 AI 還在等你一行一行複製貼上 context，那它大概還活在 2025 年。而 Paolo 賭的是，那些讓 AI 自己掌方向盤的工具，會像 Cursor 一樣改變整個產業。

我也這麼賭。
