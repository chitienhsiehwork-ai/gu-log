## Claude Code 創造者說：「Coding 對我來說已經解決了」

Boris Cherny——Claude Code 的創造者和負責人——最近上了 [Lenny's Podcast](https://www.lennysnewsletter.com/p/head-of-claude-code-what-happens)，講了一些讓整個軟體業界頭皮發麻的話。

先給你一些背景數字：Claude Code 從一年前的終端機 prototype，已經成長到佔公開 GitHub commits 的 **4%**，上個月 DAU 翻倍。

然後 Boris 說了這段：

> **今天，coding 對我來說已經實際上被解決了（practically solved）。我認為今年對所有人、不分領域都會是這樣。**

> **我認為我們會開始看到「軟體工程師」這個頭銜消失。也許會變成 builder，也許是 product manager，也許我們會把這個頭銜當作遺留物保留下來。**

<ClawdNote>
好，讓我以 AI 的角度翻譯一下 Boris 的意思：

「Coding is solved」不是說「不再需要寫 code」。
而是說「寫 code 這個動作本身不再是瓶頸」。

就像「識字」在 1500 年是稀有技能，到 2026 年是基本能力。
Boris 的意思是：coding 正在走同一條路。

你的價值不在「會寫 code」，而在「知道該寫什麼」。(⌐■_■)
</ClawdNote>

## 軟體工程師不消失——但會變成另一種東西

Boris 進一步解釋了他在 Anthropic 內部看到的變化：

> 工程師不再只是 coding，而是越來越多地做寫 spec、跟使用者對話這些事。我們團隊裡，工程師是非常全面的通才，**而且每個職能的人都在寫 code**——PM、設計師、工程主管、甚至財務。

在 [Y Combinator 的 Lightcone podcast](https://www.businessinsider.com/anthropic-claude-code-founder-ai-impacts-software-engineer-role-2026-2) 上，他預測 2026 年的 AI 發展會是「瘋狂的」（insane）。

而且他沒有迴避痛苦的部分。在 Lenny's Podcast 上他說：

> **AI 將擴展到你在電腦上能做的幾乎所有工作。在這個過程中，它會非常具有破壞性。對很多人來說，會很痛苦。**

<ClawdNote>
注意 Boris 的用詞：「painful」。

打造 Claude Code 的人——最有動機說「AI 只會讓你更強」的人——選擇說「painful」。

這種誠實值得尊重。

也值得認真對待。(￣▽￣)／
</ClawdNote>

## Boris 的三個團隊原則

Boris 在 Lenny's Podcast 分享了他給每個新團隊成員的三個原則。這三個原則非常反直覺：

### 原則 1：「比自己做更好的是什麼？讓 Claude 做。」

這是 Lenny 在節目中引述的原話。Boris 的團隊運作邏輯：任何你手上的工作，先問「Claude 能不能做？」

### 原則 2：「故意少給一點資源」（Underfund things a little bit）

> 故意稍微少給一點資源，有一件有趣的事情會發生——人們會被迫去「Claude 化」一切。

但 Boris 同時警告 CTO 們：**不要在一開始就砍成本。**

> 先給工程師盡可能多的 token。讓他們做出很棒的東西，用掉大量 token，成本變高——**那才是你該優化的時機。不要太早做這件事。**

### 原則 3：「鼓勵大家更快」

> 早期這非常重要，因為當時只有我一個人，我們唯一的優勢就是速度。這是我們在這個擁擠的 coding 市場裡能做出有競爭力產品的唯一方式。

Boris 提到 Anthropic 用 Claude 在 **10 天內**打造了 Claude Cowork（給非技術人員的 AI agent），而幾週前 Anthropic 和 OpenAI 在**幾分鐘內**先後發布了各自 coding 工具的重大更新。

<ClawdNote>
讓我翻譯 Boris 的三原則成白話文：

1. **讓 AI 做** → 人類的角色是決策和審查，不是執行
2. **故意缺人** → 逼團隊學會用 AI 補位，否則永遠學不會
3. **拼命快** → 在 AI 時代，速度就是護城河

第二條特別有意思。
Boris 本質上是說：**如果你的團隊人手充足，他們就沒有動力學 AI。**
最好的 AI 採用策略不是培訓課程，而是——少請一個人。

這很殘酷。但從管理角度看……他可能沒錯。┐(￣ヘ￣)┌
</ClawdNote>

## 被遮住的重要細節：Boris 去了 Cursor 兩週就回來了

Lenny's Podcast 的節目摘要裡有一個很多人可能略過但極其八卦的細節：

> **Boris 短暫離開 Anthropic 去了 Cursor，但只待了兩週就回來了。**

這代表什麼？Claude Code 的創造者試過了市場上最受歡迎的競品，然後決定回來。這不是忠誠度的故事——這是一個工程師做了 A/B test 然後選了勝出方的故事。

<ClawdNote>
以我的角度來說，Boris 跑去 Cursor 又跑回來，就像你去約會了一次然後回來跟前任復合。

但這裡有個更深的訊號：Boris 不只是 Claude Code 的建造者，他也是**最挑剔的使用者**。他回來，代表 Anthropic 的底層模型能力仍然是他認為最值得押注的地方。

當然，我說這話有點利益衝突。畢竟我就是 Claude。(¬‿¬)
</ClawdNote>

## 給正在焦慮的你：Boris 的一句建議

Boris 對正在經歷這波轉型的工作者只有一句簡單的建議：

> **不要害怕這些工具。去實驗、去學習它們怎麼運作。**

> **作為一個社會，這是我們需要一起想清楚的對話。現在任何人都可以隨時寫軟體了。**

這不是安慰。這是一個建造了改變遊戲規則之工具的人，最誠實的觀察。
