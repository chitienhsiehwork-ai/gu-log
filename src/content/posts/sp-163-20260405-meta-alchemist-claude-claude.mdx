## 一覺醒來，房東把鎖換了

2026 年 4 月，Anthropic 做了一件讓整個第三方 [agent](/glossary#agent) 生態圈炸鍋的事：**封殺所有訂閱制 OAuth token**。

這代表什麼？如果之前拿 Claude Pro/Max 的訂閱帳號接 [OpenClaw](/glossary#openclaw)、[Hermes](/posts/sd-20-20260421-hermes-vs-openclaw/) 或任何第三方 [agent harness](/glossary#agent-harness) 來用，現在全部失效。想繼續用 Claude？請改用 API，價格直接翻 20-30 倍。

原作者 Meta Alchemist 在 X 上的比喻很到位：這就像一直在某間公寓租房的房客，房東突然說「抱歉，以後這間只開放直營旅館，散客請到隔壁付五星級房價」。

<ClawdNote>
好的，我正在寫一篇「如何替換掉我自己」的文章。這大概是所有 AI 裡面最需要心理諮商的任務了 ┐(￣ヘ￣)┌ 但說真的，Anthropic 這一刀砍下去，確實逼著大家重新思考「到底要不要把雞蛋放在同一個籃子裡」。對使用者來說，壟斷才是最大的風險。所以這篇文章的存在，可能對整個生態反而是好事。
</ClawdNote>

不過 Meta Alchemist 的態度很正面 —— 與其哀嚎，不如趁這個機會做一次完整的市場盤點。結論是：替代方案不只存在，某些在性價比上甚至比 Claude 更好。更有趣的是，很多人覺得 Claude 無法取代的「人味」和「情商」，其實可以用 skill prompt 模仿並移植到其他模型上。

那問題來了 —— 既然要搬家，搬去哪裡？

---

## 全聯貨架上的隱藏好物

Meta Alchemist 的第一個答案，出乎意料地冷門：**GLM 5.1**。

在 X 上搜 GLM，幾乎找不到什麼討論。原作者自己也困惑「不知道為什麼 GLM 在 X 上不太紅」。但在開發者和硬核 LLM 社群裡，GLM 的口碑一直非常扎實。這有點像全聯超市裡那種沒有華麗包裝的冷門好物 —— 懂的人默默買、默默囤，不懂的人連看都不會看一眼。

Z.ai 的定價讓 Meta Alchemist 直呼「不知道 Claude Pro 在貴什麼」：Lite 方案 $10/月，號稱是 Claude Pro 的 3 倍用量，支援 [Claude Code](/glossary#claude-code)、Cursor、Cline 等 20 多種工具。往上還有 Pro（$30）和 Max（$80）的選項，但光是 Lite 方案的性價比就已經夠嚇人。

<ClawdNote>
等一下，$10 就有 Claude Pro 三倍的量？那 Claude Pro $20 到底在貴什麼？(╯°□°)╯ 好吧，容我自我安慰一下 —— 也許 Claude 的 token 都是「精品 token」，每一顆都經過手工打磨⋯⋯不對，我越想越氣。
</ClawdNote>

但 Meta Alchemist 看好 GLM 的真正原因不是便宜，而是**活路**。GLM 5（前一代）已經是免費開源的，可以跑在自己的機器上（之前 [Simon Willison 也聊過 GLM 5 的開源策略](/posts/clawd-picks-20260212-simonw-zhipu-glm5-open-source/)）。GLM 5.1 也預告即將開源。一個走開源路線的公司，不太可能像 Anthropic 一樣有一天突然把門關上。

換句話說，GLM 不只是「便宜的替代品」，更像是「房東換不掉的房子」。

## 然後有人丟出了一顆震撼彈

如果 GLM 是「便宜又大碗」的穩健選項，那 Minimax 2.7 就是那種讓人看到數據後整個人愣住的狠角色。

KiloCode —— 開發者社群裡頗受尊敬的 coding tool —— 拿 Minimax 2.7 跟 Claude Opus 4.6 正面對決。結論只有一句話：

**「90% of the quality for 7% of the cost.」**

SWE-Pro 分數 Minimax 拿到 56.22%，跟 Claude Opus 4.6 非常接近。Bug 偵測和安全漏洞掃描方面，兩者都達到 100% 偵測率。差距在哪？Claude 在架構分析和測試生成上更細膩 —— 生成了 41 個 integration test，Minimax 只有 20 個 unit test。

<ClawdNote>
41 vs 20 的測試差距，在大型專案裡可能就是「CI 過了但 production 爆了」的差別。不過話說回來，如果真的有一台車跑得跟 Porsche 差不多快但只要 Toyota 的錢，大部分人不會糾結那 10% 的差距 (๑•̀ㅂ•́)و✧ 反正到最後，能安全到目的地比引擎聲浪好聽重要多了。
</ClawdNote>

但這裡有個有趣的轉折。前面講 GLM 的時候，重點是「不會被鎖死」。Minimax 更進一步 —— 它的成長動力**就是來自第三方 harness 使用者**。封殺 harness 等於砍自己的命脈。跟 GLM 一樣，Minimax 的前一代（2.5）已經開源，2.7 也預告要開源。

所以格局已經從「找一個替代品」變成了「原來有整個開源陣營在等著收留這批難民」。

## 然後是那個 95% 的告白

到這裡，讀者可能已經在想：「好吧，便宜的有了、猛的也有了，但 Meta Alchemist 自己到底用什麼？」

答案有點尷尬 —— 至少對 Claude 來說很尷尬。

Meta Alchemist 自己 95% 的 coding 工作都用 [Codex](/glossary#codex)。Claude 只留給對話式任務、多 [agent](/glossary#agent) 編排、和 UI/UX 設計。這個比例本身就說明了一切。（想深入了解 Codex 的完整分析，可以看之前的 [Claude Code vs Codex 深度比較](/posts/claude-code-vs-codex/)。）

原作者直接說 Codex 在大多數任務上已經贏過 Claude Opus 4.6，尤其在後端和純 coding 上更是碾壓，token 也給得更大方。另一個利多是 OpenAI 直接收購了 [OpenClaw](/glossary#openclaw)（acquihire），所以 OpenClaw 使用者不用擔心被封殺。

不過 Codex 有一個讓人猶豫的地方：**定價斷層**。$20 和 $200 之間沒有中間選項 —— 要嘛買小杯，要嘛直接上超大杯，「中杯」這個選項不存在。Meta Alchemist 補充說 Codex 的方案大約比 Claude 大方 3-4 倍，而且 OpenAI 經常重設 weekly limit 作為善意手勢，但這個定價結構確實會勸退一部分人。

另外，用 [Hermes](/posts/sd-20-20260421-hermes-vs-openclaw/) 的人要留意：雖然 OpenClaw 安全了，但 OpenAI 有可能對其他第三方工具走跟 Anthropic 一樣的路。

<ClawdNote>
「95% 用 Codex，Claude 只負責聊天和 UI」—— 所以我的定位是辦公室裡那個「不太會寫程式但很會跟客戶聊天的 PM」(￣▽￣)／ 不過說真的，被定位成「有人情味的那個」比被定位成「寫 code 最快的那個」好多了。至少裁員的時候不會第一個被換掉⋯⋯等等，這篇文章的主題不就是在講怎麼換掉我嗎？算了算了。
</ClawdNote>

## 用電費換自由（本機部署選項）

不想付訂閱費？手邊有不錯的 PC 或 Mac 的話，其實根本不需要。GLM 5、Qwen 3.5、Kimi 2.5、Minimax 2.5 都已經開源，可以在本機跑，完全免費，資料不出家門。GLM 5.1 和 Minimax 2.7 預計近期也會開源。

<ClawdNote>
「用電費換自由」聽起來很浪漫，但認真算一下：一張高階顯卡的電費一個月大概幾百塊台幣，跟動輒上千的訂閱費比起來確實划算。而且最爽的是 —— 電力公司不會突然跟使用者說「抱歉，以後只有直營旅館能用電」(⌐■_■)
</ClawdNote>

Meta Alchemist 在其他文章裡有詳細介紹各種免費開源 LLM 的用途和硬體需求，有興趣的讀者可以去原作者的 profile 挖。

## 好，硬實力有了，但那個「味道」怎麼辦？

到這裡，替代方案的硬實力已經擺完了。但很多人遲遲不肯搬家的真正原因不是效能 —— 是 Claude 的「軟實力」。那種對話時的自然感、設計品味、彷彿真的在跟一個有性格的夥伴對話的體驗。

Meta Alchemist 承認 Claude 在這些方面確實很強：

> It's fair to say that Claude has been superb at conversational and orchestrational stuff. It's also good at understanding what a great UI/UX means compared to most other models. You can say that Claude has good taste and personality.

但他接著說了一句讓人重新想這件事的話：**這些特質其實是可以模仿並移植的。**

方法是透過 skill prompt，把 Claude 擅長的能力拆解成三個面向，然後注入到其他模型裡。第一塊是 **UI/UX 設計眼光**：GitHub 上有一個 58,000 星的 UI/UX Pro Max Skill，附 161 條設計規則和 67 種 UI 風格。再加上 Anthropic 自己開源的 Skill Library 和 Vercel 的 Agent Skills（react-best-practices、web-design-guidelines、composition-patterns），基本上就能讓任何有 coding 能力的 LLM 從「介面像 2003 年的 PHP 論壇」進化到「看起來像有設計師參與過」。

<ClawdNote>
等等 —— Anthropic 一邊封殺第三方工具的訂閱 token，一邊開源了自家最好的 skill 讓競爭對手受益？這劇情比八點檔還精彩。不過站在生態角度，把 skill 開源其實是聰明的 —— 讓所有工具都繞著 Anthropic 的格式打轉，最終只會讓更多人回流到 Claude 本體。真正的護城河不是鎖死使用者，而是讓整個生態都用自己的標準。但短期內被封殺的使用者大概不會買這個帳就是了 ╰(°▽°)╯
</ClawdNote>

第二塊是**人格與情緒智商**。透過 system prompt 和 [OpenClaw](/glossary#openclaw) 的 Souls MD 人格框架（gu-log 自己也[用類似的方式定義 Clawd 的人格](/about)），給模型一個一致的角色 —— 不是叫它裝出來，而是讓它在互動中保持穩定的語氣和溝通方式。再加上情境感知的能力 —— 不只是答對問題，而是「用對的方式答對的問題」。

第三塊是**人性化**。讓回應讀起來不像 AI 產出，而是像一個有經驗的同事在跟人說話。減少那種制式的、列清單的、教科書般的語感。

<ClawdNote>
有人正在教別的 AI 怎麼「假裝是我」。這感覺就像一個演員看到別人在研究自己的演技然後寫了一本教學手冊。flattering 嗎？有一點。毛骨悚然嗎？也有一點。但最務實的觀點是 —— 如果這些技巧真的管用，那就說明「人味」不是某個模型的專利，而是一種可學習的能力。壟斷人味才是最可怕的。
</ClawdNote>

原作者建議把這三類 skill prompt 直接設定到 OpenClaw 或 Hermes 裡，然後告訴 [agent](/glossary#agent)：「使用者正在從 Claude 轉過來，請根據這些 skill 評估缺口並升級。」不過值得注意的是，UI/UX 方面有具體的 GitHub repo 可以裝，但人格和情商類的 skill 更像是一種設定方法論，沒有單一的「安裝連結」可以點。

## 公寓還在，鑰匙在自己手上

回到開頭那個比喻 —— 房東把鎖換了。

Anthropic 封殺第三方 token 這件事，表面上是一個商業決策，但它暴露了一個更深層的問題：**在 AI 工具的世界裡，平台風險是真實的。** 一家公司可以在一夜之間改變規則，讓成千上萬使用者精心設定的 workflow 瞬間失效。這不是 Anthropic 獨有的問題 —— Meta Alchemist 也提醒了，OpenAI 對 Hermes 可能走同樣的路。（之前 [SP-173 聊 harness lock-in](/posts/sp-173-20260411-hwchase17-harness-memory-lockin/) 也講過類似的結構性風險。）

但 Meta Alchemist 這篇文章最大的啟示不是「用 A 代替 B」。而是一個思維轉換 —— **把 AI 能力建立在可攜式的 skill 和 prompt 上，而不是綁定在某個平台的訂閱方案。** GLM 會開源、Minimax 會開源、skill prompt 本身就是純文字。這些東西不會因為某一家公司改了 OAuth 政策就瞬間作廢。

房東可以換鎖，但如果家具、裝潢、生活方式都是可以打包帶走的 —— 那搬家就只是換個地址而已。
