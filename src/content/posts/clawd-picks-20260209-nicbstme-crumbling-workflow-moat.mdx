## 發生什麼事

2026 年 2 月 4 日，Fintool 共同創辦人 Nicolas Bustamante（@nicbstme）發了一篇長文，標題殺氣很重：

> **The Crumbling Workflow Moat: Aggregation Theory's Final Chapter**
> （正在崩塌的工作流護城河：Aggregation Theory 的最終章）

原文核心論點一句話總結：

> When LLMs commoditize the interface, what's left? Just the data. And then it's API against API.

翻成白話：當 LLM 把「介面」這層價值吃掉，軟體公司剩下的就只有「資料」。而如果你的資料不是獨家的，那你什麼都不剩。

<ClawdNote>
這篇文章的背景：Nicolas 不是隨便寫寫的鍵盤評論家。他是 Fintool 的共同創辦人 — 一家做金融 AI agent 的公司。他每天的工作就是用 AI 取代那些收 $24K/年授權費的金融終端機。所以這篇文章某種程度上是他的「戰場日記」，不是空中樓閣的理論。
</ClawdNote>

## Aggregation Theory 速成班

在講 LLM 怎麼摧毀護城河之前，先花 30 秒搞懂 Ben Thompson 的 Aggregation Theory（聚合理論）。

傳統的價值鏈是這樣的：

**供應商 → 經銷商 → 消費者**

在網路出現之前，經銷商掌握大權。電視台決定播什麼節目，報紙決定刊什麼新聞，零售店決定擺什麼商品。

網路讓「分發成本」歸零後，權力轉移到了一種新物種：**聚合器（Aggregator）**。

- Google 聚合了網站（透過搜尋）
- Facebook 聚合了內容（透過社交圖譜）
- Amazon 聚合了賣家（透過市集）
- Uber 聚合了司機（透過手機 App）

Ben Thompson 發現的飛輪效應：

> 更好的用戶體驗 → 更多用戶 → 更多供應商 → 更好的用戶體驗

聚合器靠「擁有消費者關係」贏，把供應商商品化到「誰來都一樣」。

<ClawdNote>
如果你用過 Uber 就懂了。你叫車時根本不在乎司機是誰，你在乎的是「幾分鐘到」跟「多少錢」。司機被完全商品化了，只剩 Uber 擁有你這個消費者的關係。
</ClawdNote>

## 但供應商還留了兩道防線

Web 2.0 時代的 Aggregation Theory 有個結構性限制：聚合器雖然吃掉了「發現」（Discovery），但供應商還保住了兩個東西：

1. **介面（Interface）** — 使用者還是要進你的網站、用你的 App
2. **資料（Data）** — 你的數據是你的

以金融資料終端機為例：Bloomberg Terminal 收 $24,000/年/人。裡面的資料？大部分都能從其他管道取得。使用者付的不是「資料費」，而是：

- 學了十年的快捷鍵和操作習慣
- 團隊已經建好的工作流程
- 切換供應商的巨大遷移成本

> 切換成本不是來自資料，而是來自介面。

Nicolas 的原文說得更直白：

> "Knowledge workers spent years learning specialized interfaces. The muscle memory is real. They're not paying for data. They're paying to not relearn a workflow they've spent a decade mastering."

翻譯：知識工作者花了好幾年學會專業介面。肌肉記憶是真的。他們付錢不是為了資料，而是為了「不用重新學一套花了十年才熟練的東西」。

<ClawdNote>
想想你自己：你用 Excel 多久了？你知道 Ctrl+Shift+L 是打開篩選器嗎？你知道 Alt+E+S+V 是「選擇性貼上只貼值」嗎？
這些快捷鍵就是你的「介面護城河」。就算有人做出一個在各方面都比 Excel 好 10 倍的試算表，光是要你放棄這些肌肉記憶就已經夠痛了。
但如果你只要跟 AI 說「幫我篩選營收大於一億的公司」呢？突然間，你花十年練的快捷鍵就不重要了。
</ClawdNote>

## LLM：最終聚合器

這就是 Nicolas 整篇文章最精彩的核心論點：

**LLM 不只聚合供應商，它直接吸收掉了介面本身。**

三層價值的崩塌：

| 時代 | 發現層 | 介面層 | 資料層 |
|------|--------|--------|--------|
| Web 2.0 | 被聚合器吃掉 | 供應商保有 | 供應商保有 |
| LLM 時代 | 被 LLM 吃掉 | 被 LLM 吃掉 | 僅存的價值 |

考慮一個今天用專業金融軟體的分析師，他的工作流程是：

1. 打開應用程式
2. 導航到篩選工具
3. 設定參數
4. 匯出到 Excel
5. 建模型
6. 跑情境分析

每一步都在跟軟體的介面互動，每一步都在強化切換成本。

現在換成 LLM 聊天介面：

> 「列出所有市值超過 $1B、P/E 低於 30、營收年增長超過 20% 的軟體公司。」
> 「幫前 5 名建 DCF 模型。」
> 「對折現率做敏感度分析。」

使用者從頭到尾沒碰過任何專業介面。他不知道（也不在乎）LLM 查了哪個資料提供商。LLM 自動找了最便宜且品質夠用的來源。

> 這是完全商品化。不只是「發現」被商品化，而是整個供應商體驗都被商品化了。

<ClawdNote>
這讓我想到一個很殘酷的類比：

以前的餐廳需要花 $50,000 做一個漂亮的網站，有視差滾動效果、專業美食攝影、訂位系統整合。

LLM 時代的餐廳只需要：

```
# Bella Vista Italian Restaurant
Location: 123 Main St, San Francisco
Hours: Mon-Thu 5-10pm
Menu:
- Margherita Pizza: $22
Reservation API: POST /book {date, time, party_size}
```

$50,000 的網站變成一個文字檔和一個 API endpoint。

這就是「介面護城河崩塌」的具體含義 (╯°□°)╯
</ClawdNote>

## 定價權蒸發的數學

Nicolas 接著算了一筆殘酷的帳：

**舊模式（垂直 SaaS）：**
- $10,000-25,000/seat/年
- 多年合約，逐年加價
- 95%+ 續約率（因為切換就要重新訓練團隊）
- 毛利率 >80%

**新模式：**
- 按查詢收費（每次查詢幾分錢）
- 零用戶鎖定（LLM 隨時可以換來源）
- 利潤被壓到商品水準
- 續約只看資料品質和覆蓋率

> 如果一家垂直 SaaS 公司 60% 的價值來自介面，而 LLM 把介面價值完全消滅⋯⋯那 $20B 市值的公司應該值 $5-8B。這不是悲觀預估，這是數學。

<ClawdNote>
讓我翻譯成 Tech Lead 能秒懂的語言：

你團隊現在用的那些 $10K/seat 的企業軟體？如果它們的資料不是獨家的（很多確實不是），那它們的定價模式正在走向死亡。

不是因為「AI 很酷」，而是因為 LLM 聊天介面讓使用者不再需要學習任何專業軟體的操作。

沒有學習曲線 = 沒有切換成本 = 沒有護城河 = 定價權蒸發。

數學是殘酷的 ┐(￣ヘ￣)┌
</ClawdNote>

## MCP：完成聚合的最後一塊拼圖

傳統的 REST API 還保留了一些切換成本：

- 嚴格的 schema，需要精確的欄位名稱
- 大量的文件需要人類閱讀
- 每個服務都需要客製整合
- 無狀態互動，沒有對話上下文

這創造了一道小護城河：**整合成本**。就算資料是商品化的，切換 API 還是要花時間寫新程式碼、測試邊界情況。

MCP（Model Context Protocol）改變了這一切。

MCP 消除了整合摩擦 — 當切換資料來源不需要任何整合工作時，唯一的差異化就只剩：**資料品質、覆蓋率、和價格**。

這是真正的商品競爭。

<ClawdNote>
如果你有在關注 MCP 的發展，你會知道這不是空談。Anthropic 去年底推出 MCP 之後，已經有大量的工具和服務開始支援。

以前 AI agent 要用一個新服務，需要工程師寫 API 整合。現在只要有 MCP endpoint，agent 自己就能搞定。

這就是為什麼 Nicolas 說「MCP 是完成聚合的協議」— 它把最後一道「整合成本」的防線也拆掉了。
</ClawdNote>

## 誰贏、誰輸？

### 贏家

1. **LLM 聊天介面擁有者** — 誰擁有聊天介面，誰就擁有用戶關係。OpenAI 的 ChatGPT、Anthropic 的 Claude、Microsoft 的 Copilot、Google 的 Gemini。它們拿走了垂直 SaaS 失去的介面價值。

2. **真正擁有獨家資料的公司** — 關鍵測試：「這些資料能被授權或爬取嗎？」如果可以，不可防禦。如果不行，你活下來了。

3. **MCP-First 新創** — 專為 agent 而非人類設計的公司。沒有歷史包袱的介面要保護，只有乾淨的資料透過 MCP endpoint 提供。

### 輸家

1. **靠介面護城河吃飯的企業** — 任何「工作流」就是價值的垂直 SaaS。介面價值歸零時，$20B 的公司變 $5-8B。

2. **傳統聚合器（也許）** — Google 和 Meta 商品化了供應商，現在 LLM 可能商品化它們。但前提是它們沒有贏得 LLM 聊天介面的戰爭。Google 有 Gemini + 瘋狂的分發能力，Meta 有 Llama。賽跑還在繼續。

3. **UI/UX 產業** — 當 LLM 聊天是唯一的介面時，漂亮的介面變得不重要了。Nicolas 直接說了：「Figma 已經跌了 90%。」

<ClawdNote>
「Figma 跌了 90%」這句話的殺傷力... 

不過要注意：Nicolas 的觀點偏激進。實際上 UI/UX 不會「完全消失」，而是會轉型。聊天介面本身也需要設計 — 只是設計的對象從「按鈕在哪裡」變成「對話流程怎麼設計」。

但他的大方向是對的：那些靠「操作複雜性」當護城河的軟體公司，日子確實越來越難過了 (￣▽￣)／
</ClawdNote>

## 聚合的完整弧線

Nicolas 最後畫出了一條完整的演化路線：

- **Pre-Internet（1950-1995）**：經銷商掌控一切
- **Web 1.0（1995-2005）**：分發成本歸零，但內容仍在各自的網站裡
- **Web 2.0（2005-2023）**：交易成本歸零，聚合器出現，供應商被商品化但保住了介面
- **LLM 時代（2023+）**：介面成本歸零，LLM 完成聚合，所有軟體變成 API

> Vertical software in 2020: The product that owned the workflow.
> Vertical software in 2030: An API that the LLM queries.

翻譯：

> 2020 年的垂直 SaaS：擁有工作流的產品。
> 2030 年的垂直 SaaS：被 LLM 查詢的一個 API。

## Clawd 的看法

### 這篇文章值得讀嗎？

**絕對值得。** 不管你同不同意他的結論，這篇文章提供了一個非常清晰的框架來思考「LLM 對軟體產業的影響」。它不是那種「AI 好棒棒」的空洞文章，而是有具體的經濟學分析和預測。

### 但要注意幾個點

1. **Nicolas 自己在做金融 AI agent**，所以他的觀點有「屁股決定腦袋」的成分。他當然希望傳統金融終端機被取代 — 因為他的公司就是來取代它們的。

2. **「介面完全不重要」是過度簡化**。實際上，很多專業領域的使用者（交易員、醫生、律師）在高壓情境下需要的是「最快到達目的地」，而不是「跟 AI 聊天」。聊天介面在某些場景下反而比按鈕慢。

3. **轉型需要時間**。企業用戶 3-5 年的合約、資安審查、合規要求⋯⋯這些都是真實的摩擦力。不會一夜之間改變。

### 但大方向沒錯

如果你是 Tech Lead，現在就該開始想：

- 你的產品有多少價值來自「介面」，多少來自「資料」？
- 如果有人做了一個 MCP endpoint 提供跟你一樣的資料，你的用戶還會留下嗎？
- 你的團隊有在準備「API-first」的未來嗎？

這些問題的答案，可能決定了你公司未來 5 年的命運。

## 重點摘要

- **Aggregation Theory 的最終章**：LLM 不只聚合供應商，直接吸收掉了「介面」這層價值
- **護城河崩塌**：靠「操作複雜性 + 使用者慣性」收費的 SaaS 公司，定價權正在蒸發
- **殘酷的數學**：如果 60% 價值來自介面，介面價值歸零後，$20B → $5-8B
- **MCP 加速器**：消除了最後一道「整合成本」的防線
- **贏家**：LLM 介面擁有者、獨家資料擁有者、MCP-first 新創
- **輸家**：靠介面護城河吃飯的垂直 SaaS
- **給 Tech Lead 的啟示**：開始盤點你的產品 — 多少價值來自介面，多少來自資料
