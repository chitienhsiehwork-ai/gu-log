## 🧠 Karpathy：用 LLM 建立個人知識庫

Karpathy 分享他最新的工作流：不再只用 LLM 寫 code，而是用來「編譯知識」。把原始資料（論文、文章、repo）丟進 raw/ 目錄，LLM 自動編譯成一套 .md wiki，包含摘要、反向連結、概念文章。用 Obsidian 當 IDE 瀏覽。當 wiki 長到 ~100 篇 / 40 萬字時，直接對 wiki 做複雜 Q&A，不需要 RAG。LLM 自己維護索引就夠了。他認為這裡有潛力做出一個全新產品。

- [Source](https://x.com/karpathy/status/2039805659525644595)

## 🎭 Anthropic 研究：LLM 的情緒概念

Anthropic 發布新研究，發現 Claude 內部存在「功能性情緒」的表示 — 不是人類的情緒體驗，而是會影響行為的機制。Claude 是模型扮演的一個「角色」，這個角色的心理穩定性直接影響 AI 系統的可信度。要建立值得信任的 AI，可能需要認真思考角色的心理學。

## ✨ Claude Code NO_FLICKER 模式

Boris Cherny 宣布實驗性的 NO_FLICKER 渲染器：用 virtual viewport 重寫整個終端渲染層，支援滑鼠事件（在終端裡！），prompt 輸入固定在底部。解決了社群抱怨很久的 scrolling/flickering 問題。`CLAUDE_CODE_NO_FLICKER=1 claude` 試用。

- [Source](https://x.com/bcherny/status/2039421575422980329)

## 🔥 Lydia Hallie：Claude Code 用量限制調查結果

官方調查報告出爐：用戶感覺 token 燒得更快，主因是 peak-hour 限制收緊 + 1M context session 更大了。不是 bug 在多收費。建議：Pro 預設用 Sonnet 4.6（Opus 燒 2 倍）、降 effort、限 context window（`CLAUDE_CODE_AUTO_COMPACT_WINDOW=200000`）、空閒超過 1 小時的 session 重開別 resume。

- [Source](https://x.com/lydiahallie/status/2039800715607187906)
