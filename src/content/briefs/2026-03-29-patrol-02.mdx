## 🧠 Karpathy：LLM 的 sycophancy 反而是最好的思考工具

Karpathy 花了 4 小時用 LLM 打磨一篇文章，越改越滿意。然後靈機一動：「那你來反駁看看？」——LLM 直接把自己剛花 4 小時改善的論點全部拆掉，而且論證之有力，連 Karpathy 自己都被說服了。

核心 insight：LLM 雖然默認 sycophantic（你說什麼它都說好），但它們在「argue 任意方向」這件事上極其擅長。善用這個特性：先讓它幫你強化論點，再讓它反駁——這比任何 debate partner 都便宜好用。所有用 AI 做決策的人都該養成這個習慣。

## 🏗️ Paweł Huryn：Agent 的瓶頸不是數量，是知識架構設計

回應「Anthropic 團隊不寫 code 了」的討論，Paweł 指出：「你是 PM、Agent 是工程師」這個框架聽起來對，但做起來完全不是那回事。瓶頸從來不是「多開幾個 Agent」，而是：你的 intent 表達得夠清楚嗎？CLAUDE.md 寫對了嗎？每個 Agent 的 design rules 和 verification strategy 對嗎？

稀缺技能 = 知識架構設計能力，不是 Agent 管理能力。這跟 ShroomDog 之前寫的 GenAI App Engineer 概念完全吻合。

🔗 [tweet](https://x.com/PawelHuryn/status/2037948706972917782)
