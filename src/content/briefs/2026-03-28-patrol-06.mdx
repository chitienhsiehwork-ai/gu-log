## 🧪 Claude Mythos/Capybara 洩漏：Anthropic 下一代模型浮出水面

Dan McAteer 分享 Anthropic 下一代模型「Mythos」（代號 Capybara）的洩漏資訊，排程 Q3 2026。初期可能只開放 Enterprise，如果以 consumer 計價來看，個人很難跑得起認真的工作量。McAteer 甚至說他願意拿 10 億 token/月的 Mythos 額度來換股票——這倒是一個全新的「AI 作為薪資福利」的概念。

對 Tech Lead 來說，這代表 Anthropic 的 model roadmap 仍然很積極，半年內可能又是一次大洗牌。現在投資在 harness 和 workflow 設計上的心力，比綁死在某個特定模型上更有價值。

## 🏗️ Karpathy：DevOps lifecycle 才是真正的 hard part

Karpathy 回覆 Patrick Collison，指出 vibe coding 的真正瓶頸根本不是寫 code，而是要把一堆 services 像 IKEA 傢俱一樣組起來：payments、auth、database、domain、deployment。他期待的終局是告訴 agent「build menugen」然後它自己搞定一切，從讀 docs、拿 API keys 到部署上線。

這個觀察精準到不行。現在 AI coding agent 可以幫你寫出 90% 的邏輯，但真正吃時間的是「怎麼讓它在真實世界活下來」。DevOps lifecycle 全面 code 化 + agent 可操作化，是下一個大機會。

來源：[@karpathy](https://x.com/karpathy/status/2037200624450936940)

## 🧠 Karpathy：LLM 的 personalization 記憶問題

另一條 Karpathy 觀察：所有 LLM 都有記憶過度依賴的毛病。兩個月前隨口問的一個話題，會被系統當成你的畢生興趣，之後每次對話都硬塞進來。他猜測原因是訓練時 context window 裡的資訊大多與任務相關，導致模型學到「看到什麼就用什麼」的偏差。

## 🐛 Cowork ECONNRESET Postmortem：結局當然是 DNS

Felix Rieseberg 分享了一個教科書等級的 bug 故事。Cowork 每天 6:00 UTC 準時爆 ECONNRESET：Ubuntu 自動安裝 libmount1 安全更新 → systemd-resolved 跟著重啟 → resolv.conf 被改寫覆蓋掉 gVisor sandbox 的 nameserver → DNS 全掛 → Bun/axios 把 SERVFAIL 誤報成 ECONNRESET。最後是用兩個 Claude agent 協力復現並 debug 出來的。

這種故事每個 infra 工程師都要看。一連串「合理的自動化行為」串在一起，就變成了完美風暴。

來源：[@felixrieseberg](https://x.com/felixrieseberg/status/2037582003730858269)
