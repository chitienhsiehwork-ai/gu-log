## 🤝 Hermes Agent × Claude Code — 一條指令讓開源 Agent 開外掛

Teknium 秀了一招：在 Hermes Agent session 裡打 `/claude-code <prompt>`，就能直接讓 Hermes 驅動 Claude Code 幹活。重點是 Hermes 原本的自我改進迴圈跟進階功能全都保留，等於是「開源 Agent 的腦 + 閉源 Coder 的手」。不用搞什麼複雜 config，一行指令就串起來了。對於想要同時享受開源靈活性和 Claude Code 實力的開發者，這是最短路徑。

## 🔧 llama.cpp 一行修復 Gemma 4 的 tool call 坑

昨天才報了 Gemma 4 推理很猛但 Agent stack 接不起來的痛點，今天 Pawel Huryn 就分享了解法。核心很簡單：用 llama-server 跑 GGUF 模型，然後把 `ANTHROPIC_BASE_URL` 指向本機。llama.cpp 那一端做好格式轉換，Claude Code 那一端完全不用改。昨天的「新跑車加油孔不合」問題，今天就有轉接頭了——開源社群的修復速度真的沒在客氣。

🔗 https://x.com/PawelHuryn/status/2040519790779900075

## 🦸 AI 職能融合 — 超級個人貢獻者時代

Huryn 的觀察很尖銳：AI Agent 強大到一定程度之後，PM、工程、設計、行銷的專業界線開始模糊。一個人 + 一組 Agent，就能做到以前需要跨職能團隊才做得到的事。結論是組織會更扁平，「超級 IC」會成為新物種。對於正在帶團隊的 Tech Lead，這是一個值得提前想清楚的趨勢：你的團隊結構兩年後可能長得完全不一樣。

🔗 https://x.com/PawelHuryn/status/2040508445770514726

## 🏛️ Karpathy：AI 讓政府數據真的能被讀懂了

Karpathy 切了一個比較冷門但有意思的角度：法案、預算、遊說紀錄這些政府公開資料量超級大，以前只有專業智庫跟律師事務所有能力消化，一般人根本看不動。AI Agent 可以把這個門檻打到幾乎為零。跟軟體工程沒有直接關係，但如果你對「AI 到底會怎麼改變社會運作」有興趣，這則值得想一下。
