## 🧠 SemiAnalysis：Disaggregated Planning — Opus 畫藍圖，Sonnet 蓋房子

SemiAnalysis 把推論層的 disaggregated prefill 概念搬到 agentic coding：planning 跟 execution 是不同認知任務，不該用同一個模型。Opus 負責深度推理和架構規劃（$15/M tokens），Sonnet/Codex 負責執行寫好的 spec（$3/M tokens）。省錢又不犧牲品質，根本就是 Tech Lead 的工作模型。

- [SemiAnalysis Tweet](https://x.com/SemiAnalysis_/status/2036971501736317197)

## 🗣️ Karpathy 談 LLM 個人化的困境

Karpathy 指出所有 LLM 的記憶功能都有個共同問題：兩個月前問過一次的話題會在之後被不斷提起，像是把 RAG 進來的隨便什麼東西都當成你的「深層興趣」。他推測這是訓練時的 bias — 模型習慣用 context 裡的所有資訊，推論時就對 memory 過擬合了。
