Lv.2 倉庫大災難

EAV：讓道具自己背屬性標籤

今天不背資料庫術語。今天看木弓、紅水、攻擊卷軸在倉庫裡吵到管理員懷疑人生。

NPC 開場：菜鳥被拖進倉庫

倉庫管理員
倉庫管理員

「歡迎來到道具倉庫！我是管理員。

看你一臉新手樣…來，我用你那堆亂七八糟的道具，教你一個連工程師都在用的收納大絕招：EAV。

學會了，你就懂為什麼救災平台的站點資料要這樣存。走吧！」

第一幕：恐怖的「萬用表格」

大寬表 / NULL 海

管理員一開始很天真，做了一張「所有道具都要填的萬用表格」。不管你是木弓、紅水、還是攻擊卷軸，通通站好，全部填：攻擊力 / 攻擊速度 / 回復HP / 卷軸成功率。

倉庫門一關，道具們開始暴動。墮落城市都沒這麼吵。

🏹

木弓

攻擊力 22、攻擊速度「快」都可以填。

「回復HP？卷軸成功率？我一把弓是要回復什麼啦！叫漢斯來也不會把我煮成紅水！」
🧪

紅水

回復 HP 50，任務完成。

「攻擊力？攻擊速度？我、我就一瓶水欸，是要去打菇菇寶貝還是打嫩寶？」
📜

攻擊卷軸

成功率 60%，其他全都跟它無關。

「我只負責讓你心臟停一下，不負責回 HP。不要把我跟藍水放同一個考卷。」

所有道具都被迫填同一張表

管理員崩潰：「每進一種新道具，我就得在這張表多挖一個欄位…結果整張表一半都是 NULL 空格。這片就是工程師說的『NULL 海』。更慘的是，想加一欄就得把整張表打掉重改。」

📓 正經筆記

把所有東西塞進一張「欄位固定的大寬表」，會得到 ① 一堆永遠填不滿的 NULL 欄位、② 想加新欄位就要 ALTER TABLE 改整張表結構。救災現場最不能做的就是臨時改表。

第二幕：神級收納術

EAV：主表 + 屬性表

管理員被 NULL 海淹到膝蓋，終於開竅：「我幹嘛逼紅水填攻擊力？我幹嘛逼木弓填回復HP？各位道具，從今天起，你們自己掛自己的屬性標籤！」

倉庫突然安靜。連魔法森林的漢斯都點頭：這才像個設計。

① 倉庫門口的總名冊 stations

這本簽到簿只記每件道具都有的基本資料：編號、名字、種類。乾淨清爽，像村莊長老終於把任務看板擦乾淨。

1木弓 / 武器
2紅水 / 藥水
3攻擊卷軸 / 卷軸

② 道具自己掛屬性標籤 station_properties

特殊屬性不再硬塞固定欄位，而是一張標籤記一個屬性，貼在該道具身上。沒有那個屬性，就根本不用貼。

🏹 木弓：攻擊力=22 🏹 木弓：攻擊速度=快 🧪 紅水：回復HP=50

標籤表：一筆 row 就是一張屬性標籤

NULL 海消失，標籤自己站好
道具編號 屬性名稱（標籤） 值
1 🏹木弓 攻擊力 22
1 🏹木弓 攻擊速度 快
2 🧪紅水 回復HP 50
管理員得意：「想幫某個道具加新屬性？再貼一張標籤（多一筆 row）就好，完全不用動表格結構！ 這套收納術，工程師叫它 EAV。」

救災平台其實也在做同一件事

站點也一樣：充電站掛〔插座數量=8〕、醫療站掛〔醫護人數=3〕。各掛各的、互不佔位。充電站不需要硬填「醫護人數」，就像紅水不需要假裝自己有攻擊速度。

充電站：插座數量=8 醫療站：醫護人數=3 物資站：物資種類=文字

📓 正經筆記

EAV = 共通資料放主表（固定欄位）；特殊資料拆成「一筆 row 一個屬性」放屬性表（動態欄位）。加屬性 = 加 row，不動表結構。station_properties 就是這張屬性（標籤）表。

第三幕：防止倉庫變垃圾場

白名單 config

標籤制度剛上線，大家爽到亂貼。A 玩家貼〔攻擊力〕，B 玩家貼〔攻擊〕，C 玩家貼〔攻擊數值〕。木弓站在旁邊一臉「你們到底要不要尊重我」。

管理員抱頭：「同一件事三種寫法，查資料會查到墮落城市去。這不是自由，這是垃圾場。」

攻擊數值

官方道具圖鑑 station_property_config

管理員搬出一本厚到可以拿去敲菇菇寶貝的「官方道具圖鑑」：圖鑑早就規定好，「武器的合法屬性叫『攻擊力』，型別是數字」。你貼標籤只能照圖鑑來，亂發明的名稱不准貼。

就像遊戲裡道具的屬性不是玩家亂打的，是遊戲設計時就定義好的。白名單就是那本官方圖鑑。

圖鑑頁：哪些標籤合法

白名單 / 允許清單
合法屬性名稱 適用種類 值的型別
攻擊力 武器 數字
回復HP 藥水 數字
插座數量 充電站 數字
醫護人數 醫療站 數字

📓 正經筆記

三張表分工 = 主表（共通欄）+ 屬性表（誰掛了什麼屬性、值多少）+ 白名單 config（規定哪些屬性名稱合法、什麼型別）。站點、任務單共用這同一套。

終章彩蛋：E / A / V 到底是誰？

角色身分揭曉

管理員把倉庫燈關一半，像墮落城市酒吧裡準備講祕密：「你以為 EAV 是工程師用來嚇新手的咒語？不是。這三個字母，其實就是剛剛在倉庫裡吵架的三個角色。」

木弓停止抱怨，紅水停止裝攻擊力，攻擊卷軸也暫時不讓人心臟停一下。全員坐好，準備公布身分。

🏹

E = Entity（實體）

就是道具本人。木弓、紅水，都是那個「擁有屬性的東西」。

救災平台裡，E 就是一個站點。它站在那裡，等你幫它掛資料。
🏷️

A = Attribute（屬性）

就是標籤上的屬性名稱：〔攻擊力〕〔回復HP〕。

救災平台裡，A 是「插座數量」「醫護人數」。不是亂貼的那種攻擊、攻擊數值。
🔢

V = Value（值）

就是標籤上的數字或內容：木弓的 22、紅水的 50。

救災平台裡，V 是 8 個插座、3 位醫護。真正填進去的答案在這裡。

每一筆 row，就是一張小紙條

（哪個 E） 的 （哪個 A） 是 （多少 V）
小紙條 E：誰 A：什麼屬性 V：是多少
〔木弓 的 攻擊力 是 22〕 木弓 攻擊力 22
〔某充電站 的 插座數量 是 8〕 某充電站 插座數量 8
管理員收網：「屬性表 station_properties 的每一筆 row，就是一張小紙條：誰－的什麼－是多少。」

📓 一句話收網

EAV 就是把資料拆成一堆「誰－的什麼－是多少」的小紙條，而不是硬塞進一張固定欄位的大表。Entity 是誰、Attribute 是哪個屬性、Value 是多少 —— 你剛剛已經在倉庫裡全部見過它們了。
