gu-log Human Finishability Signals

把「ShroomDog 是否真的讀完、留言罵難看、或想分享」變成 versioned human signal，讓 Tribunal 不再只最佳化 AI 分數。

Review GitHub PROpen PR files

這次要你批准什麼

1
Human feedback 必須綁文章版本。
你的「難看死了」要知道是 v1 還是 v5，不然 Tribunal rewrite 後 feedback 會漂移。
2
Raw unversioned Giscus comments 不得驅動 automation。
現有 Giscus 綁 pathname；未 index 出 version 之前，不能拿來自動重寫或 block publish。
3
負向 human feedback 可以推翻 AI PASS。
AI score PASS 只是 proxy；如果你讀不完或罵難看，系統要進 review/requeue policy。
4
Storage / thresholds / exact UI 延後決定。
這版先批准 contract，不急著選 DB、不急著移除 Giscus、不一次做完整 analytics。

目前系統現況

  • Giscus comments：綁 pathname，不含文章版本。
  • postVersion：來自 post-versions.json，目前是 file-touch count，不是 content-only version。
  • Reading tracker：只存 slugs[]，且目前需要 gu-log-jwt 才會寫入。
  • ShareButton：有分享 UI，沒有 share signal event。
  • Vercel Analytics：只有 pageview telemetry，不是 Tribunal 可查詢 evidence。
  • Tribunal：會重寫低分文章，但目前沒有人類 finishability/comment/share signal。

最重要的產品語意

  • 讀完：可讀 signal。
  • 讀一段後放棄：可能 boring，但要有 confidence，不能單次 quick bounce 就判死刑。
  • 站內留言負評：高價值 negative feedback，不是 positive engagement。
  • 分享：強正向 signal，rewrite 時應 preserve/study。

comment presence ≠ 好。
「這篇難看死了」應歸為 negative / rewrite-needed。
