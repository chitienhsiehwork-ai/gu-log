# 我用 Codex 学了 12 个工具，这是我学到的

大多数人用 Codex 的方式是错的。
他们打开 Codex，输入：「请给我讲讲这个工具。」
Codex 给他们 3000 字。
他们读完。点点头。
还是不知道从哪里开始。
**问题不是 Codex。是你的问题。**
你把 Codex 当老师。
然后纳闷为什么一直在听讲座，从来没学会。
我用 Codex 学了 12 个工具，花了 6 个月。
学到的第一件事是：
**别让 Codex 教你。让它帮你找入口。**
这篇文章我会分享：
- 为什么大多数人问 Codex 的问题是错的
- 5 步把 Codex 从百科全书变成学习教练
- 如何设计最小的练习，真正教会你东西
- 那个区分「Codex 懂」和「你懂」的问题
开始。

## 一、大多数人问错了问题
第一次用 Codex 学东西时，我问：「解释一下这个框架怎么工作的。」
Codex 给了我 2000 字。
架构分解。设计模式。核心概念。最佳实践。
我读完。还是不知道从哪里开始。
**Codex 不知道你需要学什么。你得告诉它从哪里开始。**
想想你上次学新东西的时候。
新框架。新 API。新工具。
你可能做了这几件事之一：
1. 读完整份文档（被淹没了）
1. 直接跳进教程（搞糊涂了）
1. 让 Codex 给完整解释（听了场讲座）
这些都不管用。
因为它们都假设同一件事：
**学习是靠吸收信息发生的。**
不是。
学习是靠找到入口发生的。
一个概念。一个例子。一个动作，让其他所有东西突然说得通。

[media]
当你停止让 Codex 教你，开始让它帮你找入口，一切都变了。

## 二、第一步：定义最小的问题
大多数人这样开始：「我想学某某工具。」
那不是问题。那是愿望。
更好的问题是：
**「我需要理解的最小的东西是什么，才能不再卡住？」**
不是「学完整个东西」。不是「精通它」。只是：不再卡住。
**你不需要学所有东西。你需要学够用的，能走下一步。**
我学那个数据工具时，第一个问题是：
「我想理解这个工具，理解到能跑一个例子就行。不是做项目。就是跑一个东西，看到它工作。我该从哪里开始？」
Codex 给了我：
- 一个要看的文件（不是整个文档）
- 三个要先理解的概念（不是所有概念）
- 一条要运行的命令（不是完整流程）
- 我现在可以忽略什么（这个最重要）
15 分钟后，我跑通了第一个例子。
那之前我卡了一周。
**别让 Codex 教你。让它帮你定义最小的问题。**
我现在用的提示词
**复制这个，直接用：**
```
我想学习【工具/项目/主题】。

目标不是一次性学透，而是先完成入门。

我现在知道一点点但不知从哪里开始。

请帮我判断最适合的切入点。
```
这做了三件事：
1. 设定边界（入门，不是精通）
1. 承认你不知道的（不知从哪里开始）
1. 要判断，不是信息（判断切入点）
试试看。你会得到路线图，不是讲座。
## 三、第二步：画粗糙的地图
有了入口之后，下一步是画地图。
不是完整地图。粗糙地图。
刚好够知道：你在哪里、你要去哪里、你可以跳过什么。
我这样问 Codex
**复制这个，直接用：**
```
基于这个切入点，请给我一个入门地图。

我需要知道：
1. 这个工具解决什么问题（一句话）
2. 最先需要理解的 3 个概念
3. 入门级文档或示例在哪里
4. 现在可以暂时忽略什么
5. 你的判断依据是什么
```
注意我没要的：
- 完整架构
- 所有功能
- 最佳实践
- 高级用法
我要的是最小可行理解。
**目标不是知道所有东西。目标是知道该关注什么、该忽略什么。**
大多数人卡住是因为他们试图一次学完所有东西。
他们读核心概念、边缘情况、配置选项、测试策略、部署模式。然后在写第一行代码之前就累垮了。
我学第三个工具时犯过这个错。
我让 Codex 给我完整的架构说明。它给了。我读了两小时。还是不知道从哪里开始。
后来我换了问题：「只告诉我入门需要的。其他的我之后再学。」
Codex 给了我 3 个概念。我花 20 分钟理解了。然后就能往前走了。
**学习不是覆盖面。是牵引力。**
你不需要理解整个系统。你需要理解够用的，能往前走一步。
Codex 很擅长这个。如果你让它筛选，它会筛选。如果你让它排优先级，它会排。
但你得问。

## 四、第三步：跑最小的例子
现在你有地图了。你知道从哪里开始。你知道该关注什么。
下一步：跑点东西。
不是做点东西。不是发布点东西。就是跑点东西。
**最小的、能给你反馈的东西。**
提示词
**复制这个，直接用：**
大多数教程失败在这里。
```
基于入门地图，请设计一个最小练习。

要求：
- 尽量小（小到不会害怕失败）
- 有明确结果（能看到成功或失败）
- 易回退（失败后容易重来）
- 每步有解释（告诉我为什么这样做）
- 附带检查问题（让我确认自己理解了）
```
它们给你一个「hello world」例子，要么太简单（学不到真东西），要么太复杂（需要同时理解 10 个东西）。
最好的练习在中间：
**小到你不会放弃。真到你能学到有用的东西。**
我学那个数据工具时，我问 Codex：
「设计一个练习，让我处理一个 10 行的 CSV 文件，输出一个数字。就这样。」
不是仪表盘。不是数据管道。就是：CSV 进，数字出。
那一个练习教会了我：
- 怎么加载数据
- 怎么跑基本操作
- 怎么检查输出
- 哪里会出错
花了 15 分钟。
对比官方教程，它想让我：搭建数据库、配置认证、建数据模型、写测试。
我会在第二步放弃。
**另一个例子**
我学第五个工具时，也用了同样的方法。那次是个前端框架。
我让 Codex 设计一个练习：「在页面上显示一行字，点击后变颜色。」
就这样。不是完整应用。不是路由。不是状态管理。
就是：一行字，点击，变颜色。
10 分钟后我跑通了。那之后我知道怎么往下走了。
**练习的目标不是做出厉害的东西。是建立信心。**
一旦你成功跑了一个小东西，你就不卡了。你有牵引力了。

## 五、第四步：讲回来
这是大多数人跳过的测试：
**你能不看文档，解释你刚学的东西吗？**
不是给专家讲。给你自己讲。用你自己的话。
这是你发现自己到底懂没懂的地方，还是只是照着做了一遍。
我用的提示词
**复制这个，直接用：**
```
根据刚才的探索过程，请问我 5 个贴近实际使用的问题。

我回答后，请指出理解偏差并用简单语言解释。
```
Codex 会问你这样的问题：
- 「你刚跑的代码，入口在哪里？」
- 「为什么我们用这个方法而不是那个？」
- 「如果你改这个参数会怎样？」
如果你能答，你懂了。如果你答不了，你只是在复制粘贴。
我学第七个工具时，跳过了这一步。我觉得自己懂了。
一周后要用，发现什么都不记得。又从头学了一遍。浪费了两倍时间。
后来我学第八个工具，强迫自己做这一步。
Codex 问了我 5 个问题。我答对了 2 个。另外 3 个，我以为自己懂了，其实没懂。
Codex 重新解释了。这次我真懂了。
**「Codex 懂」和「你懂」差别很大。**
Codex 能跑代码。Codex 能解释代码。但 Codex 不知道你懂没懂代码。
那是你的工作。唯一的检查方法是试着讲回来。
这一步花 5 分钟。但这是区别：真的学到了东西，还是骗自己以为学到了东西。
别跳过。

## 六、第五步：留面包屑
最后一步是会复利的那个。
**写下你刚学的东西。**
不是完整指南。不是文档。就是面包屑。
够用就行，这样 3 个月后回来，你不用从零开始。
我记录的是
**复制这个，直接用：**
```
1. 当前学习目标
2. 已理解内容
3. 三个最重要概念
4. 完成的最小练习
5. 卡住或误解的地方
6. 下次学习的接续点
```
这花 10 分钟。但之后能省几小时。
我学第四个工具时，没做这一步。三个月后要用，完全忘了。从头学了一遍。浪费了一下午。
后来我学第九个工具，强迫自己记录。三个月后要用，打开笔记，5 分钟就想起来了。省了一下午。
**不记录的学习只是昂贵的娱乐。**
目标不是记住所有东西。目标是给自己留条路，能从上次停的地方继续。
Codex 也能帮这个
**复制这个，直接用：**
```
请帮我总结这次学习过程。

包括：
- 我学会了什么
- 我还不确定什么
- 下次应该从哪里继续
```
它会给你干净的总结。复制。保存。继续。
下次需要时，你会感谢自己。

## 七、为什么大多数人不做这个
说实话。这个框架不复杂。
就五步：定义最小的问题、画粗糙的地图、跑最小的例子、讲回来、留面包屑。
谁都能做。那为什么不做？
**因为这需要承认你不知道。**
大多数人宁愿假装懂，也不愿让 Codex 帮他们找入口。
他们宁愿读完整份文档，也不愿承认自己卡在第一页。
他们宁愿跳过「讲回来」这步，也不愿面对自己只是在复制代码的事实。
我也是这样。
前三个工具，我都假装自己懂了。结果每次要用的时候，都得重新学。
第四个工具开始，我逼自己诚实。
承认自己不知道从哪里开始。承认自己需要最小的练习。承认自己需要测试理解。
从那之后，学习速度快了一倍。
不是因为我变聪明了。是因为我停止骗自己了。
不舒服的真相
**Codex 修不好你的自尊心。**
如果你不愿意说「我不知道从哪里开始」，Codex 只会给你更多你不需要的信息。
如果你不愿意跑一个小例子，Codex 会给你一个你完成不了的复杂教程。
如果你不愿意测试自己的理解，Codex 会让你继续骗自己。
框架管用。但只有在你对自己不知道的东西诚实的时候。

## 转变
传统学习说：吸收信息直到你理解。
Codex 辅助学习说：找到入口，走一步，检查理解，重复。
区别不是工具。是方法。
[media]
Codex 不会让你更聪明。它让你更快找到从哪里开始。
但你还是得开始。
